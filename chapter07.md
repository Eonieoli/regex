## 7.1 `re.sub()` 기본 사용법

> 💡 **한 줄 요약**: `re.sub()`은 정규표현식 패턴에 매칭되는 부분을 지정한 문자열로 바꿔주는 "찾아 바꾸기" 함수입니다.

### 직관적 이해

워드 프로세서의 "찾기 및 바꾸기(Find & Replace)" 기능을 생각해보세요. 문서에서 "colour"를 모두 "color"로 바꾸고 싶다면, 찾기 칸에 "colour"를, 바꾸기 칸에 "color"를 입력하고 "모두 바꾸기"를 누르면 됩니다.

`re.sub()`은 바로 이 "찾아 바꾸기"를 정규표현식의 힘으로 수행합니다. 단순 문자열 치환은 파이썬의 `str.replace()`로도 가능하지만, **패턴 기반 치환**이 필요할 때 `re.sub()`이 빛을 발합니다. 예를 들어 "여러 개의 공백을 하나로 줄이기", "전화번호의 하이픈을 모두 제거하기", "모든 HTML 태그를 삭제하기" 같은 작업은 고정된 문자열이 아니라 *패턴*을 기준으로 바꿔야 하므로 `str.replace()`로는 처리할 수 없습니다.

### 핵심 개념 설명

> 📝 **`re.sub(pattern, repl, string, count=0, flags=0)`**: 정규표현식 `pattern`에 매칭되는 `string` 내의 모든 부분을 `repl`(replacement)로 치환한 새 문자열을 반환하는 함수. 원본 문자열은 변경되지 않는다.

`re.sub()`의 기본 시그니처를 살펴봅시다:

```python
re.sub(pattern, repl, string, count=0, flags=0)
```

| 매개변수 | 설명 |
|---------|------|
| `pattern` | 찾을 정규표현식 패턴 |
| `repl` | 매칭된 부분을 대체할 문자열 (또는 함수 — 7.3에서 다룸) |
| `string` | 치환 대상이 되는 원본 문자열 |
| `count` | 최대 치환 횟수. 기본값 `0`은 **모든 매칭**을 치환 |
| `flags` | 정규표현식 플래그 (선택 사항) |

가장 기본적인 사용법부터 봅시다:

```python
import re

text = "오늘 날씨가 좋다. 내일 날씨도 좋을까?"
result = re.sub(r'날씨', '기온', text)
print(result)
# 출력: 오늘 기온가 좋다. 내일 기온도 좋을까?
```

이 예시에서는 리터럴 문자열 `'날씨'`를 `'기온'`으로 바꿨습니다. 이 정도는 `str.replace()`로도 가능하죠. 정규표현식의 진가는 **패턴**을 사용할 때 드러납니다:

```python
# 여러 개의 연속 공백을 하나의 공백으로 정규화
text = "이름:   홍길동    나이:  25"
result = re.sub(r'\s+', ' ', text)
print(result)
# 출력: 이름: 홍길동 나이: 25
```

> 🔗 **연결**: `\s+`는 Chapter 3에서 배운 사전 정의 문자 클래스 `\s`(공백 문자)와 Chapter 4에서 배운 수량자 `+`(1회 이상)의 결합입니다.

#### `count` 매개변수: 치환 횟수 제한

기본적으로 `re.sub()`은 매칭되는 **모든 부분**을 치환합니다. `count` 매개변수를 지정하면 앞에서부터 지정한 횟수만큼만 치환합니다:

```python
text = "사과, 바나나, 사과, 포도, 사과"
# 모든 '사과'를 치환 (기본)
print(re.sub(r'사과', '딸기', text))
# 출력: 딸기, 바나나, 딸기, 포도, 딸기

# 처음 2개만 치환
print(re.sub(r'사과', '딸기', text, count=2))
# 출력: 딸기, 바나나, 딸기, 포도, 사과
```

`count=1`로 지정하면 가장 처음 매칭되는 것 하나만 치환됩니다. 이는 "첫 번째 등장만 바꾸고 싶다"는 상황에서 유용합니다.

#### 매칭이 없으면?

패턴에 매칭되는 부분이 없으면, `re.sub()`은 원본 문자열을 그대로 반환합니다. 에러가 발생하지 않으므로 안전합니다:

```python
text = "Hello World"
result = re.sub(r'\d+', 'NUM', text)  # 숫자가 없음
print(result)
# 출력: Hello World  (원본 그대로)
```

#### 컴파일된 패턴 객체에서의 사용

> 🔗 **연결**: Chapter 5에서 배운 `re.compile()`로 만든 패턴 객체도 `sub()` 메서드를 가지고 있습니다.

```python
pattern = re.compile(r'\d+')
result = pattern.sub('NUM', "주문번호 12345, 수량 3개")
print(result)
# 출력: 주문번호 NUM, 수량 NUM개
```

패턴 객체의 `sub()` 메서드에서는 `pattern` 매개변수가 빠지고, `repl`과 `string`만 전달합니다.

### 상세 예시

> 🔍 **예시 7.1.1: 여러 종류의 구두점 제거**
>
> **상황**: 텍스트에서 모든 구두점(마침표, 쉼표, 느낌표, 물음표 등)을 제거하고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> text = "안녕하세요! 반갑습니다. 오늘 뭐 하세요?"
> clean = re.sub(r'[.!?,]', '', text)
> print(clean)
> # 출력: 안녕하세요 반갑습니다 오늘 뭐 하세요
> ```
>
> **핵심 포인트**: 문자 클래스 `[.!?,]` 안에서 마침표(`.`)는 메타문자가 아닌 리터럴로 동작합니다. `repl`에 빈 문자열 `''`을 전달하면 매칭된 부분이 **삭제**됩니다.

> 🔍 **예시 7.1.2: 전화번호에서 하이픈 제거**
>
> **상황**: 다양한 형식의 전화번호에서 하이픈(`-`)과 공백을 모두 제거하여 숫자만 남기고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> phones = ["010-1234-5678", "02 - 555 - 1234", "031-555-9999"]
> for phone in phones:
>     clean = re.sub(r'[\s-]+', '', phone)
>     print(f"{phone:20s} → {clean}")
> # 출력:
> # 010-1234-5678        → 01012345678
> # 02 - 555 - 1234      → 025551234
> # 031-555-9999         → 0315559999
> ```
>
> **핵심 포인트**: `[\s-]+` 패턴은 공백 문자와 하이픈이 하나 이상 연속되는 부분을 매칭합니다. 이렇게 하면 하이픈, 공백, 또는 그 조합 모두를 한 번에 처리할 수 있습니다.

### 흔한 실수와 주의사항

> ⚠️ **주의: `re.sub()`은 원본을 변경하지 않습니다**
>
> 파이썬의 문자열은 불변(immutable)이므로, `re.sub()`은 항상 **새로운 문자열을 반환**합니다. 결과를 변수에 저장해야 합니다:
> ```python
> text = "Hello World"
> re.sub(r'World', 'Python', text)  # 결과를 저장하지 않음!
> print(text)  # 여전히 "Hello World"
> 
> # 올바른 사용:
> text = re.sub(r'World', 'Python', text)
> print(text)  # "Hello Python"
> ```

> ⚠️ **주의: `count` 매개변수의 위치**
>
> `count`는 **네 번째** 매개변수입니다. `flags`와 혼동하지 마세요. 키워드 인수로 명시하면 실수를 방지할 수 있습니다:
> ```python
> # 모호할 수 있는 호출
> re.sub(r'\d', 'X', text, 2)  # count=2
> 
> # 명확한 호출 (권장)
> re.sub(r'\d', 'X', text, count=2)
> ```

### Section 요약

- `re.sub(pattern, repl, string)`은 패턴에 매칭되는 모든 부분을 `repl`로 치환한 새 문자열을 반환한다
- `repl`에 빈 문자열 `''`을 전달하면 매칭된 부분을 삭제할 수 있다
- `count` 매개변수로 치환 횟수를 제한할 수 있다 (기본값 `0` = 전부 치환)
- 매칭이 없으면 원본 문자열이 그대로 반환된다
- 컴파일된 패턴 객체에서도 `pattern.sub(repl, string)` 형태로 사용할 수 있다

## 7.2 치환 문자열의 역참조

> 💡 **한 줄 요약**: 치환 문자열에서 `\1`, `\g<1>` 같은 역참조를 사용하면, 매칭된 캡처 그룹의 내용을 치환 결과에 그대로 가져와 쓸 수 있습니다.

### 직관적 이해

지금까지의 `re.sub()`은 매칭된 부분을 **고정된 문자열**로 바꿨습니다. 하지만 실무에서는 "찾은 내용을 살짝 변형해서 다시 넣기"가 훨씬 자주 필요합니다.

예를 들어 날짜 형식을 `2024-03-15`에서 `03/15/2024`로 바꾸고 싶다고 합시다. 이때 연도, 월, 일 자체는 원본에서 가져와야 합니다. 고정 문자열로는 이것이 불가능하죠. 이럴 때 **역참조(backreference)**가 등장합니다.

역참조란, 캡처 그룹이 매칭한 내용을 나중에 다시 참조하는 것입니다. 마치 "3번 서랍에 넣어둔 것을 꺼내서 여기에 놓아라"라고 지시하는 것과 같습니다.

### 핵심 개념 설명

> 📝 **역참조(Backreference, 치환 문자열 내)**: 치환 문자열(`repl`) 안에서 `\1`, `\2` 등의 표기법을 사용하여, 정규표현식의 캡처 그룹이 매칭한 내용을 치환 결과에 삽입하는 기법.

> 🔗 **연결**: Chapter 6에서 배운 캡처 그룹 `()`을 떠올려보세요. `group(1)`, `group(2)` 등으로 그룹별 매칭 결과를 추출했었습니다. 역참조는 이 번호를 치환 문자열 안에서 사용하는 것입니다.

#### 기본 역참조: `\숫자`

치환 문자열에서 `\1`은 첫 번째 캡처 그룹이 매칭한 내용을, `\2`는 두 번째 캡처 그룹이 매칭한 내용을 의미합니다:

```python
import re

# 날짜 형식 변환: YYYY-MM-DD → MM/DD/YYYY
text = "오늘 날짜는 2024-03-15입니다."
result = re.sub(r'(\d{4})-(\d{2})-(\d{2})', r'\2/\3/\1', text)
print(result)
# 출력: 오늘 날짜는 03/15/2024입니다.
```

여기서 무슨 일이 일어났는지 단계별로 봅시다:

1. **패턴** `(\d{4})-(\d{2})-(\d{2})`이 `2024-03-15`를 매칭합니다
   - 그룹 1 `(\d{4})` → `2024`
   - 그룹 2 `(\d{2})` → `03`
   - 그룹 3 `(\d{2})` → `15`
2. **치환 문자열** `r'\2/\3/\1'`에서:
   - `\2` → `03` (그룹 2의 내용)
   - `\3` → `15` (그룹 3의 내용)
   - `\1` → `2024` (그룹 1의 내용)
3. 최종 치환 결과: `03/15/2024`

> ⚠️ **주의: 치환 문자열에도 raw string `r''`을 사용하세요**
>
> `\1`이 파이썬 문자열 이스케이프로 해석되지 않도록 치환 문자열에도 raw string 접두사 `r`을 붙이는 것이 안전합니다. `r'\1'`이 아닌 `'\1'`을 쓰면 파이썬이 이를 ASCII 코드 1번 문자로 해석할 수 있습니다.
> ```python
> # 위험: raw string을 쓰지 않은 경우
> re.sub(r'(\w+)', '\1_suffix', text)  # 의도대로 동작하지 않을 수 있음
> 
> # 안전: raw string 사용
> re.sub(r'(\w+)', r'\1_suffix', text)  # 올바름
> ```

#### `\g<n>` 형식: 모호함 방지

`\1` 뒤에 바로 숫자가 오면 파이썬이 혼동할 수 있습니다. 예를 들어 `\12`는 "그룹 1 + 문자 2"인지, "그룹 12"인지 모호합니다. 이때 `\g<n>` 형식을 사용하면 명확해집니다:

```python
import re

# \g<n> 형식으로 모호함 방지
text = "abc"
# 그룹 1의 내용 뒤에 숫자 '0'을 붙이고 싶을 때
result = re.sub(r'(a)', r'\g<1>0', text)
print(result)
# 출력: a0bc

# 만약 \10이라고 쓰면? → 그룹 10을 찾으려 시도함 (의도와 다름)
# re.sub(r'(a)', r'\10', text)  # 오류 또는 의도와 다른 결과
```

> 📝 **`\g<n>`**: 역참조에서 그룹 번호를 명시적으로 지정하는 표기법. `\g<1>`은 `\1`과 동일하지만, 숫자가 뒤따르는 상황에서 모호함을 방지한다.

#### `\g<0>`: 전체 매칭 참조

`\g<0>` (또는 `\0`이 아닌 `\g<0>`)은 전체 매칭 내용을 참조합니다. 이는 `group(0)`에 해당합니다:

```python
import re

# 각 단어를 대괄호로 감싸기
text = "hello world"
result = re.sub(r'\w+', r'[\g<0>]', text)
print(result)
# 출력: [hello] [world]
```

여기서 `\g<0>`은 `\w+`가 매칭한 전체 문자열(각 단어)을 의미합니다. 단어를 그대로 유지하면서 앞뒤에 대괄호를 추가한 것입니다.

### 상세 예시

> 🔍 **예시 7.2.1: 이름 순서 뒤집기 (성 이름 → 이름 성)**
>
> **상황**: "홍 길동" 형태의 이름을 "길동 홍"으로 바꾸고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> names = "홍 길동, 김 영희, 이 철수"
> result = re.sub(r'(\S+) (\S+)', r'\2 \1', names)
> print(result)
> # 출력: 길동 홍, 영희 김, 철수 이
> ```
> - `(\S+)`는 공백이 아닌 문자를 1개 이상 캡처합니다 (그룹 1: 성, 그룹 2: 이름)
> - `r'\2 \1'`은 그룹 2(이름)를 먼저, 그룹 1(성)을 나중에 배치합니다
>
> **핵심 포인트**: 역참조를 사용하면 매칭된 내용의 **순서를 재배치**할 수 있습니다.

> 🔍 **예시 7.2.2: 강조 마크업 변환 (`*text*` → `<em>text</em>`)**
>
> **상황**: 간단한 마크다운의 `*강조*` 표기를 HTML의 `<em>강조</em>`로 변환하고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> text = "이것은 *중요한* 내용이고 *핵심* 포인트입니다."
> result = re.sub(r'\*([^*]+)\*', r'<em>\1</em>', text)
> print(result)
> # 출력: 이것은 <em>중요한</em> 내용이고 <em>핵심</em> 포인트입니다.
> ```
> - `\*`는 리터럴 별표를 매칭합니다 (이스케이프)
> - `([^*]+)`는 별표가 아닌 문자를 1개 이상 캡처합니다 (강조 내용)
> - `r'<em>\1</em>'`은 캡처된 내용을 `<em>` 태그로 감쌉니다
>
> **핵심 포인트**: 원본의 일부(강조된 텍스트)를 유지하면서 감싸는 형식만 바꿀 수 있습니다.

### 흔한 실수와 주의사항

> ⚠️ **주의: 존재하지 않는 그룹 번호 참조**
>
> 치환 문자열에서 패턴에 존재하지 않는 그룹 번호를 참조하면 에러가 발생합니다:
> ```python
> # 그룹이 1개뿐인데 \2를 참조하면 에러!
> re.sub(r'(\w+)', r'\2', "hello")
> # re.error: invalid group reference 2 at position 1
> ```
> 패턴의 캡처 그룹 수와 치환 문자열의 역참조 번호를 반드시 일치시켜야 합니다.

> ⚠️ **주의: `\0`은 사용할 수 없습니다**
>
> 전체 매칭을 참조할 때 `\0`은 사용할 수 없습니다. 반드시 `\g<0>` 형식을 사용하세요:
> ```python
> # 잘못된 사용
> # re.sub(r'\w+', r'[\0]', "hello")  # 의도대로 동작하지 않음
> 
> # 올바른 사용
> re.sub(r'\w+', r'[\g<0>]', "hello")  # '[hello]'
> ```

### Section 요약

- 치환 문자열에서 `\1`, `\2` 등으로 캡처 그룹의 내용을 참조할 수 있다
- `\g<n>` 형식은 `\n`과 동일하지만, 뒤에 숫자가 올 때 모호함을 방지한다
- `\g<0>`은 전체 매칭 내용을 참조한다
- 역참조를 사용하면 매칭된 내용을 재배치, 감싸기, 변형 등에 활용할 수 있다
- 치환 문자열에도 raw string(`r''`)을 사용하는 것이 안전하다

## 7.3 함수를 이용한 동적 치환

> 💡 **한 줄 요약**: `re.sub()`의 `repl` 자리에 함수를 전달하면, 매칭될 때마다 그 함수가 호출되어 매칭 내용에 따라 치환 결과를 동적으로 결정할 수 있습니다.

### 직관적 이해

역참조를 사용한 치환은 강력하지만, 결국 "캡처한 내용을 그대로 재배치"하는 데 한정됩니다. 만약 매칭된 숫자를 2배로 만들거나, 매칭된 단어를 대문자로 변환하거나, 매칭 내용에 따라 완전히 다른 결과를 반환해야 한다면 어떻게 할까요?

이때 `re.sub()`의 두 번째 인수에 **문자열 대신 함수**를 전달합니다. 이 함수는 매칭이 발생할 때마다 호출되며, 매치 객체를 인수로 받아 치환 결과를 반환합니다. 마치 공장의 컨베이어 벨트에서 각 제품(매칭)이 지나갈 때마다 검사원(함수)이 개별적으로 판단하여 처리하는 것과 같습니다.

### 핵심 개념 설명

`re.sub()`의 `repl` 매개변수에 callable(호출 가능 객체)을 전달하면, 매칭이 발생할 때마다 해당 함수가 호출됩니다:

```python
re.sub(pattern, 함수, string)
```

이때 함수는 다음 규칙을 따라야 합니다:
1. **매개변수**: 매치 객체(Match object) 하나를 받는다
2. **반환값**: 치환할 **문자열**을 반환한다

> 🔗 **연결**: 함수가 받는 매치 객체는 Chapter 6에서 배운 것과 동일합니다. `group()`, `group(1)`, `start()`, `end()` 등 모든 메서드를 사용할 수 있습니다.

가장 기본적인 예부터 봅시다:

```python
import re

def to_upper(match):
    """매칭된 내용을 대문자로 변환"""
    return match.group().upper()

text = "hello world python"
result = re.sub(r'\b\w+\b', to_upper, text)
print(result)
# 출력: HELLO WORLD PYTHON
```

매칭이 발생할 때마다의 흐름:
1. `\b\w+\b`가 `"hello"`를 매칭 → `to_upper`에 매치 객체 전달 → `"HELLO"` 반환
2. `\b\w+\b`가 `"world"`를 매칭 → `to_upper`에 매치 객체 전달 → `"WORLD"` 반환
3. `\b\w+\b`가 `"python"`를 매칭 → `to_upper`에 매치 객체 전달 → `"PYTHON"` 반환

#### 람다 함수 활용

간단한 변환은 `lambda`로 간결하게 표현할 수 있습니다:

```python
import re

# 매칭된 숫자를 2배로
text = "사과 3개, 바나나 5개, 포도 2개"
result = re.sub(r'\d+', lambda m: str(int(m.group()) * 2), text)
print(result)
# 출력: 사과 6개, 바나나 10개, 포도 4개
```

여기서 `lambda m: str(int(m.group()) * 2)`는:
1. `m.group()`으로 매칭된 문자열을 가져오고 (예: `"3"`)
2. `int()`로 정수 변환 → `3`
3. `* 2`로 2배 → `6`
4. `str()`로 다시 문자열 변환 → `"6"` (반환값은 반드시 문자열이어야 함)

> ⚠️ **주의: 함수는 반드시 문자열을 반환해야 합니다**
>
> 함수가 문자열이 아닌 값(예: 정수)을 반환하면 `TypeError`가 발생합니다:
> ```python
> # 잘못된 사용: 정수를 반환
> re.sub(r'\d+', lambda m: int(m.group()) * 2, text)  # TypeError!
> 
> # 올바른 사용: 문자열로 변환하여 반환
> re.sub(r'\d+', lambda m: str(int(m.group()) * 2), text)  # OK
> ```

#### 캡처 그룹과 함수 치환의 결합

함수 치환에서도 캡처 그룹을 활용할 수 있습니다. 함수가 받는 매치 객체에서 `group(n)`으로 각 그룹에 접근하면 됩니다:

```python
import re

# 날짜 형식 변환 + 월 이름 변환
months = {
    '01': 'Jan', '02': 'Feb', '03': 'Mar', '04': 'Apr',
    '05': 'May', '06': 'Jun', '07': 'Jul', '08': 'Aug',
    '09': 'Sep', '10': 'Oct', '11': 'Nov', '12': 'Dec'
}

def format_date(match):
    year = match.group(1)
    month_name = months[match.group(2)]
    day = match.group(3)
    return f"{month_name} {day}, {year}"

text = "회의: 2024-03-15, 마감: 2024-12-31"
result = re.sub(r'(\d{4})-(\d{2})-(\d{2})', format_date, text)
print(result)
# 출력: 회의: Mar 15, 2024, 마감: Dec 31, 2024
```

이 예시는 단순 역참조로는 불가능합니다. 숫자 `"03"`을 `"Mar"`로 변환하는 로직이 필요하기 때문입니다. 함수 치환을 사용하면 이런 **변환 로직**을 자유롭게 구현할 수 있습니다.

### 상세 예시

> 🔍 **예시 7.3.1: 매칭 내용에 따른 조건부 치환**
>
> **상황**: 텍스트에서 HTTP 상태 코드를 찾아, 200번대는 "✅ 성공", 400번대는 "❌ 클라이언트 오류", 500번대는 "💥 서버 오류"로 표시하고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> def status_label(match):
>     code = int(match.group())
>     if 200 <= code < 300:
>         return f"{code}(✅ 성공)"
>     elif 400 <= code < 500:
>         return f"{code}(❌ 클라이언트 오류)"
>     elif 500 <= code < 600:
>         return f"{code}(💥 서버 오류)"
>     return match.group()  # 기타 코드는 그대로
> 
> log = "응답: 200, 리다이렉트: 301, 에러: 404, 서버오류: 503"
> result = re.sub(r'\b\d{3}\b', status_label, log)
> print(result)
> # 출력: 응답: 200(✅ 성공), 리다이렉트: 301, 에러: 404(❌ 클라이언트 오류), 서버오류: 503(💥 서버 오류)
> ```
>
> **핵심 포인트**: 함수 치환을 사용하면 매칭된 값에 따라 **완전히 다른 치환 결과**를 만들 수 있습니다. if/elif 분기는 물론, 딕셔너리 조회, 외부 함수 호출 등 파이썬의 모든 기능을 활용할 수 있습니다.

> 🔍 **예시 7.3.2: 단어의 첫 글자만 대문자로 (Title Case)**
>
> **상황**: 영어 문장에서 각 단어의 첫 글자만 대문자로 변환하고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> text = "the quick brown fox jumps over the lazy dog"
> result = re.sub(r'\b(\w)(\w*)\b', lambda m: m.group(1).upper() + m.group(2), text)
> print(result)
> # 출력: The Quick Brown Fox Jumps Over The Lazy Dog
> ```
> - 그룹 1 `(\w)`: 단어의 첫 글자
> - 그룹 2 `(\w*)`: 나머지 글자들
> - 람다 함수: 첫 글자만 대문자로 변환하고 나머지를 그대로 붙임
>
> **핵심 포인트**: 캡처 그룹으로 매칭의 **일부분**만 선택적으로 변환할 수 있습니다.

### 흔한 실수와 주의사항

> ⚠️ **주의: 함수 호출 `()` vs 함수 객체 전달**
>
> `re.sub()`에는 함수 **객체**를 전달해야 합니다. 함수를 호출한 결과를 전달하면 안 됩니다:
> ```python
> # 잘못된 사용: 함수를 호출해버림
> re.sub(r'\d+', to_upper(), text)  # TypeError!
> 
> # 올바른 사용: 함수 객체를 전달 (괄호 없음)
> re.sub(r'\d+', to_upper, text)
> ```

### Section 요약

- `re.sub()`의 `repl`에 함수를 전달하면, 매칭마다 해당 함수가 호출되어 동적 치환이 가능하다
- 함수는 매치 객체를 인수로 받고, 치환할 **문자열**을 반환해야 한다
- 간단한 변환은 `lambda`로, 복잡한 로직은 일반 함수로 구현한다
- 함수 내에서 `group()`, `group(n)` 등 매치 객체의 모든 메서드를 사용할 수 있다
- 조건부 치환, 값 변환, 포맷 변경 등 역참조만으로는 불가능한 복잡한 치환에 활용한다

## 7.4 `re.subn()`

> 💡 **한 줄 요약**: `re.subn()`은 `re.sub()`과 동일하게 치환을 수행하면서, 치환 결과와 함께 **몇 번 치환했는지**를 튜플로 반환합니다.

### 직관적 이해

`re.sub()`으로 치환을 한 후 "도대체 몇 군데가 바뀐 거지?"라고 궁금할 때가 있습니다. 물론 치환 전후의 문자열을 비교해볼 수 있지만, `re.subn()`을 사용하면 훨씬 간편합니다. **subn**의 `n`은 "number(횟수)"를 의미한다고 기억하면 됩니다.

### 핵심 개념 설명

> 📝 **`re.subn(pattern, repl, string, count=0, flags=0)`**: `re.sub()`과 동일하게 동작하되, 반환값이 `(치환된_문자열, 치환_횟수)` 형태의 **튜플**인 함수.

```python
import re

text = "사과 3개, 바나나 5개, 포도 2개"
result, n = re.subn(r'\d+', 'N', text)
print(f"결과: {result}")
print(f"치환 횟수: {n}")
# 출력:
# 결과: 사과 N개, 바나나 N개, 포도 N개
# 치환 횟수: 3
```

튜플 언패킹으로 결과 문자열과 치환 횟수를 동시에 받는 것이 일반적입니다.

#### 활용 시나리오

`re.subn()`은 다음과 같은 상황에서 특히 유용합니다:

```python
import re

# 1. 치환이 실제로 발생했는지 확인
text = "이미 깨끗한 텍스트"
result, n = re.subn(r'\s{2,}', ' ', text)
if n > 0:
    print(f"{n}건의 공백이 정규화되었습니다.")
else:
    print("정규화할 내용이 없습니다.")
# 출력: 정규화할 내용이 없습니다.

# 2. 처리 보고서 생성
dirty_text = "가격:  1,000원    수량:   3개"
clean_text, count = re.subn(r'\s{2,}', ' ', dirty_text)
print(f"정리 완료: {count}건의 불필요한 공백 제거")
print(f"결과: {clean_text}")
# 출력:
# 정리 완료: 2건의 불필요한 공백 제거
# 결과: 가격: 1,000원 수량: 3개
```

### 상세 예시

> 🔍 **예시 7.4.1: 반복 치환으로 텍스트 완전 정리**
>
> **상황**: 텍스트에서 여러 종류의 불필요한 요소를 단계적으로 제거하면서, 각 단계의 치환 횟수를 기록하고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> text = "  Hello,   World!!  How  are   you???  "
> 
> # 1단계: 여분의 공백 제거
> text, n1 = re.subn(r' {2,}', ' ', text)
> # 2단계: 반복된 구두점 정리
> text, n2 = re.subn(r'([!?.])\1+', r'\1', text)
> # 3단계: 앞뒤 공백 제거 (strip과 동일 효과)
> text, n3 = re.subn(r'^\s+|\s+$', '', text)
> 
> print(f"결과: '{text}'")
> print(f"공백 정리: {n1}건, 구두점 정리: {n2}건, 앞뒤 공백: {n3}건")
> # 출력:
> # 결과: 'Hello, World! How are you?'
> # 공백 정리: 3건, 구두점 정리: 2건, 앞뒤 공백: 2건
> ```
>
> **핵심 포인트**: 다단계 텍스트 정리 작업에서 각 단계의 처리량을 추적할 수 있어 디버깅과 보고에 유용합니다.

> 🔍 **예시 7.4.2: 치환 발생 여부에 따른 분기 처리**
>
> **상황**: 사용자 입력에서 위험한 HTML 태그를 제거하되, 제거가 발생했으면 경고를 표시하고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> def sanitize_input(user_input):
>     clean, n = re.subn(r'<[^>]+>', '', user_input)
>     if n > 0:
>         print(f"⚠️ 경고: {n}개의 HTML 태그가 제거되었습니다.")
>     return clean
> 
> print(sanitize_input("안녕하세요 <b>반갑습니다</b>"))
> # ⚠️ 경고: 2개의 HTML 태그가 제거되었습니다.
> # 안녕하세요 반갑습니다
> 
> print(sanitize_input("일반 텍스트입니다"))
> # 일반 텍스트입니다
> ```
>
> **핵심 포인트**: `re.subn()`의 치환 횟수를 이용해 "무언가 변경되었는지"를 감지할 수 있습니다.

### Section 요약

- `re.subn()`은 `re.sub()`과 동일한 치환을 수행하지만, `(결과_문자열, 치환_횟수)` 튜플을 반환한다
- 치환 횟수를 활용하면 치환 발생 여부 확인, 처리 보고서 생성, 조건부 분기 등에 유용하다
- 매개변수와 동작 방식은 `re.sub()`과 완전히 동일하다

## 7.5 `re.split()`

> 💡 **한 줄 요약**: `re.split()`은 정규표현식 패턴을 구분자로 사용하여 문자열을 분할하는 함수로, 파이썬의 `str.split()`이 처리하지 못하는 복잡한 분할을 수행합니다.

### 직관적 이해

파이썬의 `str.split()`은 **고정된 하나의 문자열**로만 분할할 수 있습니다. 예를 들어 `"a,b,c".split(",")` 처럼요. 하지만 현실의 데이터는 그렇게 깔끔하지 않습니다:

- 쉼표 뒤에 공백이 있을 수도, 없을 수도 있는 경우: `"a, b,c, d"`
- 여러 종류의 구분자가 혼재된 경우: `"a;b,c|d"`
- 구분자가 가변 길이인 경우: `"a---b--c----d"`

이런 상황에서 `re.split()`은 **패턴**을 구분자로 쓸 수 있으므로, 위의 모든 경우를 우아하게 처리합니다.

### 핵심 개념 설명

> 📝 **`re.split(pattern, string, maxsplit=0, flags=0)`**: 정규표현식 `pattern`에 매칭되는 부분을 구분자로 사용하여 `string`을 분할한 **리스트**를 반환하는 함수.

```python
re.split(pattern, string, maxsplit=0, flags=0)
```

| 매개변수 | 설명 |
|---------|------|
| `pattern` | 구분자로 사용할 정규표현식 패턴 |
| `string` | 분할할 문자열 |
| `maxsplit` | 최대 분할 횟수. 기본값 `0`은 **모든 매칭 위치**에서 분할 |
| `flags` | 정규표현식 플래그 (선택 사항) |

기본 사용법:

```python
import re

# 쉼표 뒤 선택적 공백으로 분할
text = "사과, 바나나,포도, 딸기"
result = re.split(r',\s*', text)
print(result)
# 출력: ['사과', '바나나', '포도', '딸기']
```

`str.split(",")`을 사용했다면 `['사과', ' 바나나', '포도', ' 딸기']`처럼 일부 항목에 앞 공백이 포함되었을 것입니다. `re.split()`은 패턴 `,\s*`(쉼표 + 0개 이상의 공백)을 구분자로 사용하여 깔끔하게 분할합니다.

#### 여러 종류의 구분자 처리

```python
import re

# 쉼표, 세미콜론, 파이프 중 어느 것이든 구분자로
text = "a,b;c|d,e"
result = re.split(r'[,;|]', text)
print(result)
# 출력: ['a', 'b', 'c', 'd', 'e']

# 하나 이상의 공백 문자로 분할 (탭, 연속 공백 등 모두)
text = "이름\t나이   직업  도시"
result = re.split(r'\s+', text)
print(result)
# 출력: ['이름', '나이', '직업', '도시']
```

#### `maxsplit` 매개변수

`maxsplit`은 최대 분할 횟수를 지정합니다. 왼쪽에서부터 지정한 횟수만큼 분할하고 나머지는 마지막 요소에 그대로 남습니다:

```python
import re

text = "하나-둘-셋-넷-다섯"

print(re.split(r'-', text, maxsplit=2))
# 출력: ['하나', '둘', '셋-넷-다섯']

print(re.split(r'-', text, maxsplit=1))
# 출력: ['하나', '둘-셋-넷-다섯']
```

이것은 "첫 번째 구분자까지만 분할"하고 싶을 때 유용합니다. 예를 들어 `"key=value=with=equals"` 같은 문자열에서 `maxsplit=1`을 쓰면 `['key', 'value=with=equals']`로 키와 값을 깔끔하게 분리할 수 있습니다.

#### 캡처 그룹이 `re.split()`에 미치는 영향

이것은 `re.split()`의 중요한 특성입니다: **패턴에 캡처 그룹이 포함되면, 구분자로 매칭된 내용도 결과 리스트에 포함됩니다.**

```python
import re

text = "사과-바나나+포도-딸기"

# 캡처 그룹 없음: 구분자는 결과에서 사라짐
print(re.split(r'[-+]', text))
# 출력: ['사과', '바나나', '포도', '딸기']

# 캡처 그룹 있음: 구분자도 결과에 포함됨
print(re.split(r'([-+])', text))
# 출력: ['사과', '-', '바나나', '+', '포도', '-', '딸기']
```

이 특성은 구분자 정보를 보존해야 할 때 유용합니다. 예를 들어 수학 식 `"3+5-2"`를 분할할 때 연산자도 함께 유지하고 싶다면 캡처 그룹을 사용하면 됩니다.

> 💬 **참고**: 구분자를 캡처하고 싶지 않은데 패턴에 그룹핑이 필요하다면, 비캡처 그룹 `(?:...)`을 사용하면 됩니다. 이 개념은 Chapter 8에서 자세히 다룹니다. 지금은 "캡처 그룹이 `re.split()`의 결과에 영향을 준다"는 사실만 기억해두세요.

#### 빈 문자열 처리

패턴이 문자열의 시작이나 끝에서 매칭되면 빈 문자열이 결과에 포함될 수 있습니다:

```python
import re

text = ",사과,바나나,포도,"
result = re.split(r',', text)
print(result)
# 출력: ['', '사과', '바나나', '포도', '']
```

맨 앞과 맨 뒤의 빈 문자열은 리스트 컴프리헨션이나 `filter()`로 제거할 수 있습니다:

```python
result = [item for item in re.split(r',', text) if item]
print(result)
# 출력: ['사과', '바나나', '포도']
```

### 상세 예시

> 🔍 **예시 7.5.1: 문장 분리기**
>
> **상황**: 텍스트를 문장 단위로 분리하고 싶다. 문장은 마침표, 느낌표, 물음표로 끝나며, 뒤에 공백이 올 수 있다.

> 🔍 **예시 7.5.2: 키-값 쌍 파싱**

### 흔한 실수와 주의사항

> ⚠️ **주의: 캡처 그룹과 `re.split()`의 상호작용**
>
> 캡처 그룹이 결과에 포함되는 것을 모르면 예상과 다른 결과를 얻을 수 있습니다:
> ```python
> # 의도: 숫자 제거하고 분할
> text = "abc123def456ghi"
> print(re.split(r'(\d+)', text))

> # 의도대로 숫자 없이 분할
> print(re.split(r'\d+', text))

> ```
> 불필요한 캡처 그룹은 제거하거나, Chapter 8에서 배울 비캡처 그룹 `(?:...)`을 활용하세요.

> ⚠️ **주의: 빈 패턴 매칭**
>
> 빈 문자열에 매칭될 수 있는 패턴(`*` 사용 등)을 구분자로 쓸 때는 주의가 필요합니다:
> ```python
> # 모든 문자 사이에서 분할됨 (파이썬 3.7+에서 경고 또는 에러)
> # re.split(r'x*', text)  # 사용 주의!
> ```

### Section 요약

- `re.split(pattern, string)`은 정규표현식 패턴을 구분자로 문자열을 분할한다
- 여러 종류의 구분자, 가변 길이 구분자 등 `str.split()`이 처리하지 못하는 분할이 가능하다
- `maxsplit`으로 분할 횟수를 제한할 수 있다
- 패턴에 **캡처 그룹**이 포함되면, 매칭된 구분자도 결과 리스트에 포함된다
- 문자열의 시작/끝에서 매칭되면 빈 문자열이 결과에 포함될 수 있다

## 7.6 치환과 분할의 실전 패턴

> 💡 **한 줄 요약**: 이 Section에서는 `re.sub()`과 `re.split()`을 실전 시나리오에 적용하여, 데이터 포맷 변환, 텍스트 정리, 간단한 템플릿 처리 등을 수행합니다.

### 직관적 이해

이전 Section들에서 `re.sub()`, `re.subn()`, `re.split()`의 문법과 동작을 배웠습니다. 이제 이 도구들을 **결합**하여 실제로 유용한 텍스트 처리 작업을 수행해봅시다. 요리를 배울 때 칼질, 불 조절, 양념 기법을 따로 배운 후에 실제 요리를 만들어보는 것처럼, 이번 Section은 배운 기법들을 종합 적용하는 시간입니다.

### 핵심 개념 설명

이 Section에서는 새로운 함수나 문법을 도입하지 않습니다. 대신 지금까지 배운 도구들을 다양한 실전 시나리오에 적용합니다.

#### 패턴 1: 데이터 포맷 변환

서로 다른 시스템 간에 데이터를 주고받을 때 포맷 변환은 매우 흔한 작업입니다:

```python
import re

# 전화번호 형식 통일: 다양한 형식 → 표준 형식
phones = [
    "01012345678",
    "010-1234-5678",
    "010 1234 5678",
    "(010) 1234-5678",
]

for phone in phones:
    # 1단계: 숫자만 추출
    digits = re.sub(r'\D', '', phone)
    # 2단계: 표준 형식으로 포맷
    formatted = re.sub(r'(\d{3})(\d{4})(\d{4})', r'\1-\2-\3', digits)
    print(f"{phone:20s} → {formatted}")
# 출력:
# 01012345678          → 010-1234-5678
# 010-1234-5678        → 010-1234-5678
# 010 1234 5678        → 010-1234-5678
# (010) 1234-5678      → 010-1234-5678
```

이 예시는 2단계 처리입니다: 먼저 `re.sub(r'\D', '', phone)`으로 숫자가 아닌 문자를 모두 제거한 후, `re.sub()`으로 역참조를 사용해 표준 형식으로 재구성합니다.

#### 패턴 2: 텍스트 정규화

텍스트 데이터를 처리하기 전 "정규화(normalization)" 단계는 거의 필수입니다:

```python
import re

def normalize_text(text):
    """텍스트 정규화: 불필요한 공백과 구두점 정리"""
    # 연속된 공백을 하나로
    text = re.sub(r'[ \t]+', ' ', text)
    # 줄바꿈 정규화 (연속 줄바꿈은 최대 2개로)
    text = re.sub(r'\n{3,}', '\n\n', text)
    # 구두점 앞의 불필요한 공백 제거
    text = re.sub(r'\s+([.!?,;:])', r'\1', text)
    # 앞뒤 공백 제거
    text = text.strip()
    return text

messy = "  안녕하세요 .   반갑습니다 !  \n\n\n\n  잘 부탁합니다 .  "
print(normalize_text(messy))
# 출력:
# 안녕하세요. 반갑습니다!
#
# 잘 부탁합니다.
```

두 번째 `re.sub()`에서 `r'\1'`을 사용한 부분을 주목하세요. 구두점 앞의 공백을 제거하면서, 구두점 자체는 역참조를 통해 보존합니다.

#### 패턴 3: 구조화된 텍스트 파싱

`re.split()`과 `re.sub()`을 결합하면 구조화된 텍스트를 파싱할 수 있습니다:

```python
import re

# 간단한 CSV 라인 파싱 (따옴표 없는 경우)
csv_line = "홍길동, 25, 서울, 개발자"
fields = re.split(r',\s*', csv_line)
print(fields)
# 출력: ['홍길동', '25', '서울', '개발자']

# 로그 라인에서 타임스탬프와 메시지 분리
log = "[2024-03-15 14:30:00] ERROR: 파일을 찾을 수 없습니다"
parts = re.split(r'\]\s*', log, maxsplit=1)
timestamp = re.sub(r'[\[\]]', '', parts[0])  # 대괄호 제거
message = parts[1]
print(f"시간: {timestamp}")
print(f"메시지: {message}")
# 출력:
# 시간: 2024-03-15 14:30:00
# 메시지: ERROR: 파일을 찾을 수 없습니다
```

#### 패턴 4: 간단한 템플릿 치환

함수 치환을 활용하면 간단한 템플릿 엔진을 구현할 수 있습니다:

```python
import re

def simple_template(template, data):
    """{{key}} 형식의 플레이스홀더를 data 딕셔너리의 값으로 치환"""
    def replacer(match):
        key = match.group(1)
        return data.get(key, match.group(0))  # 키가 없으면 원본 유지
    
    return re.sub(r'\{\{(\w+)\}\}', replacer, template)

template = "안녕하세요, {{name}}님! {{city}}에서 오셨군요. 나이는 {{age}}세입니다."
data = {"name": "홍길동", "city": "서울", "age": "30"}

print(simple_template(template, data))
# 출력: 안녕하세요, 홍길동님! 서울에서 오셨군요. 나이는 30세입니다.
```

이 예시에서는:
- 패턴 `\{\{(\w+)\}\}`가 `{{key}}` 형태의 플레이스홀더를 찾습니다
- 캡처 그룹 `(\w+)`가 키 이름을 추출합니다
- `replacer` 함수가 딕셔너리에서 값을 찾아 반환합니다
- 키가 없으면 `match.group(0)` (원본 전체)을 반환하여 플레이스홀더를 그대로 유지합니다

### 상세 예시

> 🔍 **예시 7.6.1: camelCase → snake_case 변환**
>
> **상황**: JavaScript 스타일의 `camelCase` 변수명을 Python 스타일의 `snake_case`로 변환하고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> def camel_to_snake(name):
>     # 대문자 앞에 밑줄 삽입 후 소문자로 변환
>     result = re.sub(r'([A-Z])', r'_\1', name)
>     return result.lower().lstrip('_')
> 
> test_names = ["getUserName", "setMaxValue", "HTMLParser", "getHTTPResponse"]
> for name in test_names:
>     print(f"{name:20s} → {camel_to_snake(name)}")
> # 출력:
> # getUserName          → get_user_name
> # setMaxValue          → set_max_value
> # HTMLParser           → h_t_m_l_parser
> # getHTTPResponse      → get_h_t_t_p_response
> ```
>
> **핵심 포인트**: `([A-Z])` 패턴으로 대문자를 캡처하고, 치환 문자열 `r'_\1'`으로 그 앞에 밑줄을 삽입합니다. 연속 대문자(HTMLParser)의 경우 각 문자가 개별적으로 처리되는 점에 유의하세요.
>
> 💬 **참고**: 연속 대문자를 하나의 단어로 처리하는 더 정교한 패턴은 Chapter 9(전후방 탐색)에서 다룰 수 있는 기법입니다.

> 🔍 **예시 7.6.2: 숫자에 천 단위 콤마 삽입 (함수 치환 활용)**
>
> **상황**: 텍스트 속 숫자에 천 단위 구분 콤마를 삽입하고 싶다.
>
> **풀이**:
> ```python
> import re
> 
> def add_commas(match):
>     number = match.group()
>     # 파이썬 포맷팅을 활용
>     return f"{int(number):,}"
> 
> text = "총 매출은 1500000원이고, 비용은 320000원입니다. 수량은 5개입니다."
> result = re.sub(r'\d{4,}', add_commas, text)
> print(result)
> # 출력: 총 매출은 1,500,000원이고, 비용은 320,000원입니다. 수량은 5개입니다.
> ```
>
> **핵심 포인트**: `\d{4,}` 패턴은 4자리 이상의 숫자만 매칭합니다(1~3자리 숫자는 콤마가 불필요하므로). 함수 치환에서 파이썬의 문자열 포맷팅(`f"{n:,}"`)을 활용하여 콤마를 삽입합니다.

### 흔한 실수와 주의사항

> ⚠️ **주의: `re.sub()`과 `re.split()`을 사용할 때 `str` 메서드로 충분한지 먼저 고려하세요**
>
> 모든 치환에 `re.sub()`을, 모든 분할에 `re.split()`을 쓸 필요는 없습니다. **고정 문자열** 기반의 단순 작업은 `str.replace()`와 `str.split()`이 더 빠르고 읽기 쉽습니다:
> ```python
> # 정규표현식이 불필요한 경우 (단순 고정 문자열 치환)
> text.replace("old", "new")      # ✅ 이쪽이 더 낫다
> re.sub(r'old', 'new', text)     # ❌ 불필요하게 복잡
> 
> # 정규표현식이 필요한 경우 (패턴 기반 치환)
> re.sub(r'\s+', ' ', text)       # ✅ 패턴이 필요하므로 적절
> ```

> ⚠️ **주의: 다단계 치환의 순서가 중요합니다**
>
> 여러 `re.sub()`을 연쇄적으로 적용할 때, 순서에 따라 결과가 달라질 수 있습니다. 이전 단계의 치환 결과가 다음 단계의 입력이 되기 때문입니다. 각 단계를 적용할 때 중간 결과를 확인하는 습관을 들이세요.

### Section 요약

- `re.sub()`과 `re.split()`을 조합하면 데이터 포맷 변환, 텍스트 정규화, 구조화된 텍스트 파싱 등을 수행할 수 있다
- 다단계 처리 패턴: 먼저 불필요한 부분 제거 → 구조 파악 → 포맷 변환 순으로 진행하는 것이 일반적이다
- 함수 치환과 역참조를 결합하면 간단한 템플릿 엔진도 구현할 수 있다
- 단순 고정 문자열 작업에는 `str.replace()` / `str.split()`이 더 적합하다

> **보충 설명**: `[a-z]+`는 영문 소문자만 매칭하므로 한글과 숫자는 영향을 받지 않습니다. `\b` 단어 경계를 사용하여 단어 단위로 매칭합니다. `3.12`의 숫자 부분은 `[a-z]`에 매칭되지 않으므로 그대로 남습니다.
> </details>

> **Exercise 7.3**: 다음 로그 문자열을 `re.split()`으로 파싱하여 각 필드를 분리하세요. 구분자는 ` | `(공백-파이프-공백)입니다. 단, 첫 번째 필드(타임스탬프)와 나머지로만 분할하고 싶다면 어떻게 해야 하는지도 답하세요.
>
> ```python
> log = "2024-03-15 14:30:00 | ERROR | db_connection | timeout after 30s"
> ```
>
> <details>
> <summary>모범 답안</summary>
>
> **접근 방법**: `r'\s*\|\s*'` 패턴으로 파이프와 주변 공백을 구분자로 분할합니다.
>
> **풀이 과정**:
> ```python
> import re
> 
> log = "2024-03-15 14:30:00 | ERROR | db_connection | timeout after 30s"
> 
> # 전체 분할
> fields = re.split(r'\s*\|\s*', log)
> print(fields)
> # 출력: ['2024-03-15 14:30:00', 'ERROR', 'db_connection', 'timeout after 30s']
> 
> # 첫 번째 필드만 분리 (maxsplit=1)
> parts = re.split(r'\s*\|\s*', log, maxsplit=1)
> print(parts)
> # 출력: ['2024-03-15 14:30:00', 'ERROR | db_connection | timeout after 30s']
> ```
>
> **정답**: 전체 분할 시 `['2024-03-15 14:30:00', 'ERROR', 'db_connection', 'timeout after 30s']`
>
> **보충 설명**: `\|`는 파이프 문자의 리터럴 매칭입니다(선택 연산자 `|`와 구분하기 위해 이스케이프). `\s*`로 파이프 앞뒤의 공백을 흡수하여 깔끔한 분할 결과를 얻습니다.
> </details>

> **Exercise 7.4**: 다음 텍스트에서 `re.sub()`과 `re.subn()`을 사용하여 모든 이메일 주소를 `[이메일 숨김]`으로 치환하고, 치환된 이메일 개수를 출력하세요.

> ```

> **힌트**: 간단한 이메일 패턴 `\S+@\S+`을 사용하세요. (완벽한 이메일 검증은 Chapter 12에서 다룹니다.)
>
> <details>
> <summary>모범 답안</summary>
>
> **접근 방법**: `\S+@\S+` 패턴으로 이메일 주소를 매칭하고, `re.subn()`으로 치환과 횟수를 동시에 얻습니다.

## D3. Problem (심화 문제) — 깊이 있는 사고와 창의적 문제 해결

> **Problem 7.1**: **마크다운 → HTML 변환기 (기초)**
>
> 다음 간단한 마크다운 문법을 HTML로 변환하는 함수 `md_to_html(text)`를 작성하세요:
> - `**텍스트**` → `<strong>텍스트</strong>` (굵게)
> - `*텍스트*` → `<em>텍스트</em>` (기울임)
> - `` `코드` `` → `<code>코드</code>` (인라인 코드)
>
> 주의: `**굵게**` 변환을 먼저 처리해야 `*기울임*`과 충돌하지 않습니다.
>
> 테스트 입력:
> ```python
> text = "이것은 **굵은 텍스트**이고, *기울인 텍스트*이며, `코드`입니다."
> ```
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 여러 패턴의 `re.sub()`을 순서대로 적용하되, 더 긴 패턴(`**`)을 먼저 처리하여 짧은 패턴(`*`)과의 충돌을 방지합니다.
>
> **풀이 전략**:
> 1. `**...**`를 먼저 `<strong>...</strong>`으로 변환
> 2. 그 다음 `*...*`를 `<em>...</em>`으로 변환
> 3. 마지막으로 `` `...` ``를 `<code>...</code>`로 변환
>
> **상세 풀이**:
> ```python
> import re
> 
> def md_to_html(text):
>     # 1. 굵게: ** ... ** (먼저 처리해야 *와 충돌 방지)
>     text = re.sub(r'\*\*([^*]+)\*\*', r'<strong>\1</strong>', text)
>     # 2. 기울임: * ... *
>     text = re.sub(r'\*([^*]+)\*', r'<em>\1</em>', text)
>     # 3. 인라인 코드: ` ... `
>     text = re.sub(r'`([^`]+)`', r'<code>\1</code>', text)
>     return text
> 
> text = "이것은 **굵은 텍스트**이고, *기울인 텍스트*이며, `코드`입니다."
> print(md_to_html(text))
> # 출력: 이것은 <strong>굵은 텍스트</strong>이고, <em>기울인 텍스트</em>이며, <code>코드</code>입니다.
> ```
>
> **확장 생각**: 이 접근법은 간단한 마크다운에는 잘 동작하지만, 중첩된 서식(예: `***굵고 기울인***`)이나 여러 줄에 걸친 서식에는 한계가 있습니다. 실제 마크다운 파서는 정규표현식만으로는 완전히 구현하기 어려우며, 전용 파서 라이브러리를 사용하는 것이 바람직합니다.
> </details>

> **Problem 7.2**: **숫자 마스킹 시스템**
>
> 개인정보 보호를 위해 텍스트에서 특정 패턴의 숫자를 마스킹하는 함수 `mask_numbers(text)` 를 작성하세요. 규칙은 다음과 같습니다:
> - 연속 숫자가 10자리 이상이면: 앞 3자리와 뒤 4자리만 남기고 나머지를 `*`로 치환
> - 연속 숫자가 4~9자리이면: 뒤 4자리만 남기고 나머지를 `*`로 치환
> - 연속 숫자가 3자리 이하이면: 그대로 유지
>
> 테스트:
> ```python
> text = "전화: 01012345678, 카드: 1234567890123456, 수량: 15, 코드: 9876"
> # 기대 출력: "전화: 010****5678, 카드: 123*********3456, 수량: 15, 코드: *876"
> ```
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 함수 치환을 사용하여 매칭된 숫자의 길이에 따라 다른 마스킹 로직을 적용합니다.
>
> **풀이 전략**:
> 1. `\d+` 패턴으로 모든 연속 숫자를 매칭
> 2. 함수 치환에서 숫자의 길이를 확인하여 조건별 처리
>
> **상세 풀이**:
> ```python
> import re
> 
> def mask_numbers(text):
>     def masker(match):
>         num = match.group()
>         length = len(num)
>         if length >= 10:
>             # 앞 3자리 + 마스킹 + 뒤 4자리
>             masked_len = length - 7
>             return num[:3] + '*' * masked_len + num[-4:]
>         elif length >= 4:
>             # 마스킹 + 뒤 4자리
>             masked_len = length - 4
>             return '*' * masked_len + num[-4:]
>         else:
>             return num  # 3자리 이하는 그대로
>     
>     return re.sub(r'\d+', masker, text)
> 
> text = "전화: 01012345678, 카드: 1234567890123456, 수량: 15, 코드: 9876"
> print(mask_numbers(text))
> # 출력: 전화: 010****5678, 카드: 123*********3456, 수량: 15, 코드: *876
> ```
>
> **확장 생각**: 실제 시스템에서는 하이픈이 포함된 전화번호(`010-1234-5678`), 공백이 포함된 카드번호(`1234 5678 9012 3456`) 등도 처리해야 합니다. 이 경우 먼저 `re.sub()`으로 구분자를 제거한 후 마스킹하고, 다시 원래 형식으로 복원하는 다단계 처리가 필요합니다.
> </details>

> **Problem 7.3**: **`re.sub()`과 `re.split()`의 종합 활용 — 간이 계산기**
>
> 덧셈과 뺄셈만 포함된 수식 문자열(예: `"10 + 25 - 3 + 7 - 2"`)을 받아, 계산 결과를 반환하는 함수 `simple_calc(expr)`을 작성하세요.
>
> 조건:
> - 입력은 양의 정수와 `+`, `-` 연산자만 포함합니다
> - 연산자 앞뒤에 공백이 있을 수도, 없을 수도 있습니다
> - `re.split()`을 사용하여 숫자와 연산자를 분리하세요
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 캡처 그룹이 포함된 `re.split()`을 사용하면 숫자와 연산자를 모두 보존하면서 분리할 수 있습니다.
>
> **풀이 전략**:
> 1. 공백을 제거하여 정규화
> 2. `re.split()`으로 숫자와 연산자를 분리 (캡처 그룹 사용)
> 3. 순차적으로 연산 수행
>
> **상세 풀이**:
> ```python
> import re
> 
> def simple_calc(expr):
>     # 1. 공백 제거
>     expr = re.sub(r'\s+', '', expr)
>     
>     # 2. 연산자를 구분자로 사용하되, 연산자도 보존 (캡처 그룹)
>     tokens = re.split(r'([+-])', expr)
>     tokens = [t for t in tokens if t]  # 빈 문자열 제거
>     
>     # 3. 첫 번째 숫자로 시작
>     result = int(tokens[0])
>     
>     # 4. 연산자-숫자 쌍을 순회하며 계산
>     i = 1
>     while i < len(tokens):
>         operator = tokens[i]
>         number = int(tokens[i + 1])
>         if operator == '+':
>             result += number
>         elif operator == '-':
>             result -= number
>         i += 2
>     
>     return result
> 
> print(simple_calc("10 + 25 - 3 + 7 - 2"))   # 37
> print(simple_calc("100-50+25"))               # 75
> print(simple_calc("1 + 2 + 3 + 4 + 5"))      # 15
> ```
>
> **확장 생각**: 이 간이 계산기는 `+`와 `-`만 처리합니다. 곱셈/나눗셈과 연산자 우선순위를 지원하려면 정규표현식만으로는 부족하고, 구문 분석(parsing) 기법이 필요합니다. 이는 정규표현식의 한계를 보여주는 좋은 예이기도 합니다.
> </details>
