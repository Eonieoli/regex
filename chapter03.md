## 3.1 문자 클래스의 기본: `[...]`

> 💡 **한 줄 요약**: 대괄호 `[...]` 안에 나열한 문자 중 **하나**와 매칭되는 패턴을 만들 수 있습니다.

### 직관적 이해

Chapter 2에서 배운 마침표(`.`)는 "아무 문자 하나"와 매칭되는 만능 와일드카드였습니다. 하지만 실제 텍스트 처리에서는 "아무 문자"가 아니라 **특정 종류의 문자**만 허용해야 하는 경우가 훨씬 많습니다.

> 🔗 **연결**: Chapter 2의 마침표(`.`)는 줄바꿈을 제외한 모든 문자와 매칭되었습니다. 이번에 배울 문자 클래스는 `.`보다 훨씬 정밀한 도구입니다.

비유를 들어보겠습니다. 레스토랑에서 메뉴를 고를 때를 생각해보세요.

- 마침표(`.`)는 **"아무거나 주세요"** 와 같습니다 — 스테이크든, 파스타든, 샐러드든 무엇이든 OK.
- 문자 클래스(`[...]`)는 **"이 중에서 골라주세요"** 와 같습니다 — "파스타, 리조또, 피자 중 하나요" 처럼 선택지를 한정.

왜 이런 것이 필요할까요? 예를 들어 전화번호에서 지역번호를 추출한다고 합시다. 지역번호의 첫 문자는 `0`이고, 두 번째 문자는 `1`, `2`, `3`, `4`, `5`, `6` 중 하나입니다. 마침표를 쓰면 `0.`이 되어 `0A`나 `0!` 같은 원하지 않는 것까지 매칭됩니다. 문자 클래스를 쓰면 `0[123456]`처럼 정확히 원하는 문자만 허용할 수 있습니다.

### 핵심 개념 설명

> 📝 **문자 클래스(Character Class)**: 대괄호 `[` 와 `]` 사이에 문자들을 나열하여, 해당 위치에서 나열된 문자 중 **정확히 하나**와 매칭되는 패턴 요소입니다.

**기본 문법**: `[문자들]`

문자 클래스의 핵심 규칙은 단순합니다:

1. 대괄호 안에 허용할 문자를 나열합니다.
2. 해당 위치에서 나열된 문자 중 **하나만** 매칭됩니다 (OR 관계).
3. 문자 클래스 자체는 **정확히 한 글자**에 대응합니다.

```python
import re

# [abc]는 'a', 'b', 'c' 중 하나와 매칭
re.findall(r'[abc]', 'apple banana cherry')
# 결과: ['a', 'b', 'a', 'a', 'a', 'c']

# [aeiou]는 영문 소문자 모음 하나와 매칭
re.findall(r'[aeiou]', 'Hello World')
# 결과: ['e', 'o', 'o']
```

**문자 클래스 내부의 메타문자 규칙**

Chapter 2에서 배운 메타문자들(`.`, `^`, `$`, `*`, `+`, `?`, `|` 등)은 대괄호 **바깥**에서 특별한 의미를 가졌습니다. 그런데 대괄호 **안에서는** 대부분의 메타문자가 특별한 의미를 잃고 **리터럴 문자**로 취급됩니다. 이것은 문자 클래스의 매우 편리한 특성입니다.

```python
# 대괄호 안의 .은 "아무 문자"가 아니라 리터럴 마침표
re.findall(r'[.]', 'a.b.c')
# 결과: ['.', '.']

# 대괄호 안의 *도 리터럴 별표
re.findall(r'[*+?]', '2+3=5, 2*3=6, really?')
# 결과: ['+', '*', '?']
```

단, 대괄호 안에서 **여전히 특별한 의미를 가지는 문자**가 몇 가지 있습니다:

| 문자 | 대괄호 내부에서의 의미 | 리터럴로 사용하려면 |
|------|----------------------|-------------------|
| `]` | 문자 클래스의 끝 | `\]` 또는 맨 앞에 배치: `[]abc]` |
| `\` | 이스케이프 시작 | `\\` |
| `^` | 부정 (맨 앞에 올 때만) | 맨 앞이 아닌 곳에 배치: `[a^b]` |
| `-` | 범위 (문자 사이에 올 때) | `\-` 또는 맨 앞/뒤에 배치: `[-abc]`, `[abc-]` |

```python
# 대괄호를 리터럴로 포함하기
re.findall(r'[\[\]]', 'array[0] = list[1]')
# 결과: ['[', ']', '[', ']']

# 하이픈을 리터럴로 포함 (맨 앞이나 뒤에 배치)
re.findall(r'[-ab]', 'a-b-c')
# 결과: ['a', '-', 'b', '-']
```

> 🔍 **예시 3.1.1: 색상 코드의 첫 글자 매칭**
>
> **상황**: 웹 색상 이름 목록에서 'red', 'green', 'blue' 중 하나의 첫 글자를 찾고 싶습니다.
>
> **풀이**:
> ```python
> import re
> colors = 'red green blue yellow purple'
> pattern = r'[rgb]'
> result = re.findall(pattern, colors)
> print(result)
> # ['r', 'g', 'b', 'r', 'p']  ← 'purple'의 'p'는 아님, 'r'은 'purple' 안의 r
> ```
> 잠깐, 결과를 자세히 보면 `'purple'` 안의 `r`도 매칭됩니다. 문자 클래스는 문자열 **어디에서든** 해당 문자를 찾으므로, 단어의 첫 글자만 매칭하려면 추가적인 기법이 필요합니다(이후 `\b`에서 다룹니다).
>
> **핵심 포인트**: `[rgb]`는 `r` 또는 `g` 또는 `b` 한 글자를 매칭합니다. 위치에 관계없이 문자열 어디에서든 찾습니다.

> 🔍 **예시 3.1.2: 한글 종성 매칭**
>
> **상황**: 특정 한글 글자들이 텍스트에 포함되어 있는지 확인하고 싶습니다.
>
> ```python
> import re
> text = '가나다라마바사'
> result = re.findall(r'[가다바]', text)
> print(result)
> # ['가', '다', '바']
> ```
>
> **핵심 포인트**: 문자 클래스는 영문자뿐 아니라 한글, 숫자, 특수문자 등 어떤 문자든 나열할 수 있습니다.

> ⚠️ **주의**: 문자 클래스 `[abc]`는 **한 글자**와 매칭됩니다. `abc`라는 문자열 전체와 매칭되는 것이 아닙니다. `"abc"` 전체를 매칭하고 싶다면 그냥 `abc`라고 쓰면 됩니다(리터럴 매칭). 문자 클래스는 "a 또는 b 또는 c **중 하나의 글자**"를 의미합니다.

> ⚠️ **주의**: 문자 클래스에서 같은 문자를 중복 나열해도 에러는 나지 않지만 의미가 없습니다. `[aab]`는 `[ab]`와 동일하게 동작합니다.

**Section 3.1 요약**

- 문자 클래스 `[...]`는 대괄호 안의 문자 중 **하나**와 매칭됩니다.
- 문자 클래스는 **한 글자 위치**에 대응하며, OR 관계로 동작합니다.
- 대괄호 안에서는 대부분의 메타문자가 리터럴로 취급됩니다.
- `]`, `\`, `^`(맨 앞), `-`(문자 사이)만 대괄호 내부에서 특별한 의미를 가집니다.
- 한글, 특수문자 등 어떤 문자든 문자 클래스에 포함할 수 있습니다.

## 3.2 범위 표현과 부정

> 💡 **한 줄 요약**: 하이픈(`-`)으로 연속된 문자의 범위를 간결하게 표현하고, 캐럿(`^`)으로 특정 문자를 **제외**하는 패턴을 만들 수 있습니다.

### 직관적 이해

Section 3.1에서 `[0123456789]`처럼 문자를 하나씩 나열할 수 있다는 것을 배웠습니다. 하지만 알파벳 소문자 26개를 모두 나열하면 `[abcdefghijklmnopqrstuvwxyz]` — 이건 너무 길고 실수하기도 쉽습니다.

이럴 때 **범위 표현**을 사용합니다. `[a-z]`라고 쓰면 a부터 z까지 연속된 모든 문자를 의미합니다. 마치 호텔 방 번호를 말할 때 "301호부터 310호까지"라고 줄여 말하는 것과 같습니다.

반대로, "이 문자들만 **빼고** 전부 매칭하고 싶다"는 경우도 있습니다. 예를 들어 "숫자를 제외한 모든 문자"를 찾고 싶을 때, 숫자가 아닌 문자를 모두 나열하는 건 불가능에 가깝습니다. 이때 **부정 문자 클래스** `[^...]`를 사용합니다.

### 핵심 개념 설명

**범위 표현: `[a-z]`**

> 📝 **범위 표현(Range)**: 문자 클래스 안에서 하이픈(`-`)을 사용하여 유니코드 코드포인트 순서상 연속된 문자의 범위를 지정하는 표현입니다.

하이픈은 두 문자 사이에 놓일 때 "시작 문자부터 끝 문자까지의 모든 문자"를 의미합니다.

```python
import re

# [a-z]는 영문 소문자 a부터 z까지
re.findall(r'[a-z]', 'Hello World 123')
# 결과: ['e', 'l', 'l', 'o', 'o', 'r', 'l', 'd']

# [A-Z]는 영문 대문자 A부터 Z까지
re.findall(r'[A-Z]', 'Hello World 123')
# 결과: ['H', 'W']

# [0-9]는 숫자 0부터 9까지
re.findall(r'[0-9]', 'Hello World 123')
# 결과: ['1', '2', '3']
```

**여러 범위를 하나의 문자 클래스에 결합**할 수 있습니다:

```python
# 영문 대소문자 모두: [a-zA-Z]
re.findall(r'[a-zA-Z]', 'Hello 123!')
# 결과: ['H', 'e', 'l', 'l', 'o']

# 영문자와 숫자 모두: [a-zA-Z0-9]
re.findall(r'[a-zA-Z0-9]', 'Hi-5!')
# 결과: ['H', 'i', '5']

# 범위와 개별 문자를 함께: [a-z_]는 소문자 + 밑줄
re.findall(r'[a-z_]', 'my_var = 10')
# 결과: ['m', 'y', '_', 'v', 'a', 'r']
```

**한글 범위**도 지정할 수 있습니다:

```python
# [가-힣]은 한글 완성형 음절 전체 (가, 각, 간, ... 힣)
re.findall(r'[가-힣]', '안녕하세요 Hello 123')
# 결과: ['안', '녕', '하', '세', '요']

# [ㄱ-ㅎ]은 한글 자음
re.findall(r'[ㄱ-ㅎ]', 'ㅋㅋㅋ 재밌다 ㅎㅎ')
# 결과: ['ㅋ', 'ㅋ', 'ㅋ', 'ㅎ', 'ㅎ']
```

> 💬 **참고**: 범위 표현이 동작하는 원리는 유니코드 코드포인트의 순서에 기반합니다. `[가-힣]`이 가능한 이유는 한글 음절 '가'(U+AC00)부터 '힣'(U+D7A3)까지가 유니코드에서 연속으로 배치되어 있기 때문입니다. 유니코드와 한국어 처리에 대한 자세한 내용은 Chapter 11에서 다룹니다.

> ⚠️ **주의**: 범위의 시작 문자는 끝 문자보다 코드포인트가 작아야 합니다. `[z-a]`처럼 거꾸로 쓰면 에러가 발생합니다.
>
> ```python
> re.findall(r'[z-a]', 'hello')
> # re.error: bad character range z-a
> ```

> ⚠️ **주의**: `[A-z]`처럼 대문자와 소문자를 하나의 범위로 쓰면, ASCII 코드표에서 `Z`(90)와 `a`(97) 사이에 있는 `[`, `\`, `]`, `^`, `_`, `` ` `` 까지 포함됩니다. 대소문자를 모두 포함하려면 `[A-Za-z]`로 범위를 분리하세요.

**부정 문자 클래스: `[^...]`**

> 📝 **부정 문자 클래스(Negated Character Class)**: 대괄호 안의 맨 앞에 캐럿(`^`)을 붙이면, 나열된 문자를 **제외한** 모든 문자와 매칭됩니다.

```python
# [^0-9]는 숫자가 아닌 모든 문자
re.findall(r'[^0-9]', 'Hello 123 World')
# 결과: ['H', 'e', 'l', 'l', 'o', ' ', ' ', 'W', 'o', 'r', 'l', 'd']

# [^a-zA-Z]는 영문자가 아닌 모든 문자
re.findall(r'[^a-zA-Z]', 'Hello, World! 123')
# 결과: [',', ' ', '!', ' ', '1', '2', '3']

# [^가-힣]은 한글 음절이 아닌 모든 문자
re.findall(r'[^가-힣]', '안녕 Hello')
# 결과: [' ', 'H', 'e', 'l', 'l', 'o']
```

> 🔗 **연결**: Chapter 2에서 `^`는 문자열의 **시작**을 나타내는 앵커였습니다. 같은 `^` 기호가 **대괄호 안의 맨 앞**에 오면 "부정"의 의미로 완전히 다르게 동작합니다. 위치에 따라 의미가 달라지는 것이므로 혼동하지 않도록 주의하세요.

캐럿이 맨 앞이 **아닌** 곳에 오면 리터럴 문자로 취급됩니다:

```python
# [a^b]에서 ^는 부정이 아니라 리터럴 ^
re.findall(r'[a^b]', 'a^b = c')
# 결과: ['a', '^', 'b']
```

> 🔍 **예시 3.2.1: 파일 확장자 패턴**
>
> **상황**: 파일명에서 확장자 앞의 마침표를 포함해 `.txt`, `.csv`, `.tsv` 중 하나인지 매칭하고 싶습니다.
>
> **풀이**:
> ```python
> import re
> files = ['data.txt', 'image.png', 'report.csv', 'log.tsv', 'note.doc']
> pattern = r'\.[tc][ss][vt]'  # 이렇게 하면 너무 복잡...
>
> # 더 간단한 접근: 선택(|)을 활용
> pattern = r'\.(txt|csv|tsv)'
> # 하지만 아직 그룹핑의 캡처 동작을 배우지 않았으므로,
> # 문자 클래스만으로 간단히 해결할 수 있는 다른 예시를 봅시다.
>
> # 확장자의 첫 글자가 t 또는 c인 파일 찾기
> for f in files:
>     if re.search(r'\.[tc]', f):
>         print(f)
> # data.txt, report.csv, log.tsv
> ```
>
> **핵심 포인트**: 문자 클래스와 이스케이프를 조합하면(`\.`로 리터럴 마침표, `[tc]`로 특정 문자 선택) 실용적인 패턴을 만들 수 있습니다.

> 🔍 **예시 3.2.2: 위험 문자 제거**
>
> **상황**: 사용자 입력에서 영문자, 숫자, 공백을 제외한 특수문자를 모두 찾아내고 싶습니다.
>
> **풀이**:
> ```python
> import re
> user_input = "Hello! My name is Kim <script>alert('XSS')</script>"
> dangerous_chars = re.findall(r"[^a-zA-Z0-9 ]", user_input)
> print(dangerous_chars)
> # ['!', '<', '>', '(', "'", ')', '<', '/', '>']
> ```
>
> **핵심 포인트**: 부정 문자 클래스 `[^a-zA-Z0-9 ]`는 "영문자, 숫자, 공백이 **아닌** 모든 문자"를 매칭합니다. 이는 입력 검증이나 보안 처리에서 매우 유용한 패턴입니다.

**Section 3.2 요약**

- 하이픈(`-`)으로 범위를 지정합니다: `[a-z]`, `[0-9]`, `[가-힣]`
- 여러 범위를 하나의 문자 클래스에 결합할 수 있습니다: `[a-zA-Z0-9]`
- `[^...]`는 나열된 문자를 **제외한** 문자와 매칭됩니다 (부정).
- `^`는 대괄호 안 **맨 앞**에서만 부정을 의미하며, 다른 위치에서는 리터럴입니다.
- 범위의 시작은 끝보다 코드포인트가 작아야 합니다 (`[a-z]` ✅, `[z-a]` ❌).

## 3.3 사전 정의 문자 클래스

> 💡 **한 줄 요약**: `\d`, `\w`, `\s`는 자주 사용되는 문자 클래스를 **한 글자로 축약**한 것이며, 대문자(`\D`, `\W`, `\S`)는 그 부정형입니다.

### 직관적 이해

지금까지 `[0-9]`, `[a-zA-Z0-9_]`, `[ \t\n]` 같은 문자 클래스를 직접 작성하는 법을 배웠습니다. 그런데 이런 패턴은 너무 자주 쓰이기 때문에, 정규표현식에서는 **축약 표기법(shorthand)**을 제공합니다. 마치 자주 가는 곳을 "서울특별시 강남구 테헤란로..."라고 매번 쓰는 대신 "강남역"이라고 부르는 것과 같습니다.

이 축약 표기법은 **이스케이프 시퀀스** 형태로 백슬래시(`\`)와 알파벳 한 글자를 조합합니다.

> 🔗 **연결**: Chapter 2에서 백슬래시(`\`)가 메타문자의 특별한 의미를 제거하는 이스케이프 문자라고 배웠습니다. 하지만 `\d`, `\w`, `\s`에서의 백슬래시는 반대로 **일반 문자에 특별한 의미를 부여**합니다. 이 이중적 역할을 기억해두세요.

### 핵심 개념 설명

**세 가지 사전 정의 문자 클래스와 그 부정형**

| 축약 표기 | 의미 | 대략적 동등 문자 클래스 | 부정형 | 부정형 의미 |
|-----------|------|------------------------|--------|------------|
| `\d` | 숫자 (digit) | `[0-9]` | `\D` | 숫자가 아닌 문자 |
| `\w` | 단어 문자 (word) | `[a-zA-Z0-9_]` | `\W` | 단어 문자가 아닌 문자 |
| `\s` | 공백 문자 (space) | `[ \t\n\r\f\v]` | `\S` | 공백이 아닌 문자 |

> 📝 **`\d` (digit)**: 숫자 한 글자와 매칭됩니다. 기본적으로 `[0-9]`와 유사하지만, 파이썬 3의 기본 유니코드 모드에서는 아라비아-인도 숫자(٠١٢) 등 다른 문자 체계의 숫자도 포함할 수 있습니다.

> 📝 **`\w` (word character)**: "단어를 구성하는 문자" 한 글자와 매칭됩니다. 영문 대소문자, 숫자, 밑줄(`_`)이 포함됩니다. 파이썬 3에서는 한글 등 유니코드 문자도 포함됩니다.

> 📝 **`\s` (whitespace)**: 공백 문자 한 글자와 매칭됩니다. 스페이스(` `), 탭(`\t`), 줄바꿈(`\n`), 캐리지 리턴(`\r`), 폼 피드(`\f`), 수직 탭(`\v`)이 포함됩니다.

> 💬 **참고**: 위 표에서 "대략적 동등"이라고 쓴 이유는, 파이썬 3의 기본 모드가 유니코드 모드이기 때문입니다. `\d`는 유니코드의 모든 숫자를, `\w`는 유니코드의 모든 "단어 문자"를 포함합니다. ASCII 범위만으로 정확히 제한하려면 `re.ASCII` 플래그가 필요한데, 이는 Chapter 10에서 다룹니다. 현 단계에서는 위 표의 동등 관계를 기본으로 이해해주세요.

```python
import re

text = "전화: 010-1234-5678, Email: kim@test.com"

# \d: 숫자 찾기
print(re.findall(r'\d', text))
# ['0', '1', '0', '1', '2', '3', '4', '5', '6', '7', '8']

# \D: 숫자가 아닌 문자 찾기
print(re.findall(r'\D', text))
# ['전', '화', ':', ' ', '-', '-', ',', ' ', 'E', 'm', 'a', 'i', 'l', ...]

# \w: 단어 문자 찾기 (한글도 포함!)
print(re.findall(r'\w', text))
# ['전', '화', '0', '1', '0', '1', '2', '3', '4', '5', '6', '7', '8',
#  'E', 'm', 'a', 'i', 'l', 'k', 'i', 'm', 't', 'e', 's', 't', 'c', 'o', 'm']

# \s: 공백 문자 찾기
print(re.findall(r'\s', text))
# [' ', ' ']
```

**부정형의 활용**

대문자 버전(`\D`, `\W`, `\S`)은 소문자 버전의 정확한 반대입니다. `\d`가 매칭하는 것을 `\D`는 매칭하지 않고, `\d`가 매칭하지 않는 것을 `\D`가 매칭합니다.

```python
text = "abc 123 가나다"

# \W: 단어 문자가 아닌 것 = 공백, 구두점 등
print(re.findall(r'\W', text))
# [' ', ' ']

# \S: 공백이 아닌 것 = 눈에 보이는 모든 문자
print(re.findall(r'\S', text))
# ['a', 'b', 'c', '1', '2', '3', '가', '나', '다']
```

**사전 정의 문자 클래스를 대괄호 안에서 사용하기**

`\d`, `\w`, `\s` 등은 대괄호 안에서도 사용할 수 있으며, 다른 문자나 범위와 결합할 수 있습니다:

```python
# [\d-]: 숫자 또는 하이픈
re.findall(r'[\d-]', '010-1234-5678')
# ['0', '1', '0', '-', '1', '2', '3', '4', '-', '5', '6', '7', '8']

# [\w.]: 단어 문자 또는 마침표
re.findall(r'[\w.]', 'kim@test.com')
# ['k', 'i', 'm', 't', 'e', 's', 't', '.', 'c', 'o', 'm']  ← @는 제외됨
```

> 🔍 **예시 3.3.1: 텍스트에서 숫자만 추출하기**
>
> **상황**: "2024년 1월 매출: ₩1,500,000"에서 숫자만 추출하고 싶습니다.
>
> **풀이**:
> ```python
> import re
> text = "2024년 1월 매출: ₩1,500,000"
> digits = re.findall(r'\d', text)
> print(digits)
> # ['2', '0', '2', '4', '1', '1', '5', '0', '0', '0', '0', '0']

> ```
> 원하는 결과는 아닐 수 있습니다 — "2024", "1", "1500000"처럼 연속된 숫자 덩어리를 추출하려면 Chapter 4에서 배울 **수량자**가 필요합니다.
>
> **핵심 포인트**: `\d`는 숫자 **한 글자**를 매칭합니다. 여러 글자의 숫자를 하나의 덩어리로 추출하려면 수량자가 필요합니다.

> 🔍 **예시 3.3.2: 공백 문자 감지**
>
> **상황**: 사용자 입력에 탭이나 줄바꿈 같은 보이지 않는 공백이 섞여 있는지 확인하고 싶습니다.
>
> **풀이**:

> 🔬 **Deep Dive: `\w`와 파이썬 3의 유니코드 기본 동작**
>
> *이 부분은 심화 내용입니다. 처음 학습 시 건너뛰어도 됩니다.*
>
> 파이썬 3에서 `\w`는 기본적으로 유니코드의 "단어 문자"를 매칭합니다. 이것은 `[a-zA-Z0-9_]`보다 **훨씬 넓은 범위**입니다:
>
> ```python
> import re
> # 파이썬 3 기본: \w는 한글, 일본어, 중국어 등도 매칭

> # ['한', '글', 'A', 'B', 'C', '가', '나', '다']
>
> # 아라비아 숫자도 \d에 매칭

> # <re.Match object; span=(0, 1), match='٣'>
> ```
>

> ⚠️ **주의**: `\d`, `\w`, `\s`는 반드시 raw string(`r''`) 안에서 사용하세요. 일반 문자열에서 `\d`는 파이썬 인터프리터가 먼저 해석을 시도할 수 있습니다. `\d` 자체는 파이썬에 정의된 이스케이프가 아니라 그냥 통과되지만, `\b`는 파이썬에서 백스페이스(`\x08`)로 해석되어 문제가 됩니다. 일관성을 위해 항상 `r''`을 사용하는 습관을 들이세요.

- 대문자(`\D`, `\W`, `\S`)는 각각의 부정형입니다.
- 사전 정의 문자 클래스는 대괄호 안에서도 사용할 수 있습니다: `[\d-]`, `[\w.]`
- 파이썬 3에서는 기본적으로 유니코드 모드로 동작하여, `\w`가 한글 등도 포함합니다.
- 항상 raw string(`r''`)을 사용하세요.

## 3.4 단어 경계: `\b`와 `\B`

> 💡 **한 줄 요약**: `\b`는 단어와 비단어 문자 사이의 **경계 위치**를 매칭하며, 문자 자체가 아닌 **위치**를 지정하는 앵커입니다.

### 직관적 이해

이전까지 배운 문자 클래스(`[...]`, `\d`, `\w`, `\s`)는 모두 실제 **문자**를 매칭했습니다. 반면, `\b`는 문자가 아닌 **위치(경계)**를 매칭합니다.

> 🔗 **연결**: Chapter 2에서 배운 `^`(문자열 시작)와 `$`(문자열 끝)도 문자가 아닌 위치를 매칭하는 **앵커**였습니다. `\b`도 같은 종류의 앵커이지만, 문자열의 시작/끝이 아니라 **단어의 경계**를 지정합니다.

왜 단어 경계가 필요할까요? 구체적인 상황을 봅시다:

```python
import re
text = "cat은 category의 일부이고, catfish도 있다"

# 'cat'을 그냥 찾으면?
print(re.findall(r'cat', text))
# ['cat', 'cat', 'cat']  ← category의 cat, catfish의 cat도 포함!
```

"cat"이라는 독립된 단어만 찾고 싶은데, `category`와 `catfish` 안의 `cat`도 함께 찾아버립니다. 이것은 마치 전화번호부에서 "이"씨를 찾는데 "이승만", "이순신"뿐 아니라 "최**이**경"까지 찾아버리는 것과 같습니다. `\b`를 사용하면 "이"씨라는 *독립된 단위*만 정확히 찾을 수 있습니다.

### 핵심 개념 설명

> 📝 **단어 경계(`\b`, Word Boundary)**: 단어 문자(`\w`)와 비단어 문자(`\W`) 사이의 **위치**, 또는 문자열의 시작/끝과 단어 문자 사이의 **위치**를 매칭하는 제로 너비 앵커입니다. 어떤 문자도 소비하지 않고 위치만 확인합니다.

단어 경계가 존재하는 위치를 시각적으로 표현하면 다음과 같습니다 (`|`로 경계 표시):

```
문자열: "Hello World 123"
경계:   |Hello| |World| |123|

H 앞 = 문자열 시작 + 단어 문자 → 경계 ✅
o 뒤 = 단어 문자 + 공백(비단어) → 경계 ✅
W 앞 = 공백(비단어) + 단어 문자 → 경계 ✅
공백과 공백 사이 = 비단어 + 비단어 → 경계 ❌
```

**`\b`의 동작 규칙**:

단어 경계는 다음 세 가지 경우에 존재합니다:
1. 문자열의 **시작** 위치이면서, 첫 문자가 단어 문자(`\w`)인 경우
2. 문자열의 **끝** 위치이면서, 마지막 문자가 단어 문자인 경우
3. 문자열 **중간**에서 한쪽은 단어 문자이고 다른 쪽은 비단어 문자(`\W`)인 경우

```python
import re

# \bcat\b: 독립된 단어 'cat'만 매칭
text = "cat은 category의 일부이고, catfish도 있다. 나의 cat!"
print(re.findall(r'\bcat\b', text))
# ['cat', 'cat']  ← 독립된 'cat'만! (처음의 cat, 마지막의 cat)

# \bcat: 'cat'으로 시작하는 단어
print(re.findall(r'\bcat', text))
# ['cat', 'cat', 'cat']  ← cat, category의 cat, catfish의 cat, 마지막 cat 중 시작 경계가 있는 것

# cat\b: 'cat'으로 끝나는 단어
print(re.findall(r'cat\b', text))
# ['cat', 'cat']  ← 독립된 cat들 (category나 catfish의 cat은 제외)
```

**`\B`: 단어 경계가 아닌 위치**

> 📝 **`\B` (Non-word Boundary)**: `\b`의 정확한 반대입니다. 단어 경계가 **아닌** 위치를 매칭합니다. 즉, 양쪽이 모두 단어 문자이거나 모두 비단어 문자인 위치입니다.

```python
# \Bcat\B: 단어 내부에 있는 'cat' (양쪽 모두 경계가 아닌 위치)
text = "cat category catfish education"
print(re.findall(r'\Bcat\B', text))
# ['cat']  ← education의 'cat'만 (양쪽 모두 단어 문자로 둘러싸임)

# \Bcat: 단어 시작이 아닌 위치의 'cat'
print(re.findall(r'\Bcat', text))
# ['cat']  ← education의 'cat'
```

> ⚠️ **주의**: `\b`를 사용할 때 반드시 **raw string** `r'\b...'`을 사용하세요. 일반 문자열 `'\b'`에서 `\b`는 파이썬이 **백스페이스 문자**(ASCII 8)로 해석합니다. 이것은 가장 흔한 실수 중 하나입니다.
>
> ```python
> # ❌ 잘못된 사용: 일반 문자열
> re.findall('\bcat\b', 'cat category')  # \b가 백스페이스로 해석됨
> # 결과: []  ← 아무것도 매칭되지 않음!
>
> # ✅ 올바른 사용: raw string
> re.findall(r'\bcat\b', 'cat category')
> # 결과: ['cat']
> ```

> 🔍 **예시 3.4.1: 독립된 단어 검색**
>
> **상황**: 영어 텍스트에서 정관사 "the"만 찾고 싶은데, "there", "other", "them" 등은 제외해야 합니다.
>
> **풀이**:
> ```python
> import re
> text = "the cat sat there with the other cats, them too"
> 
> # \b 없이 검색
> print(re.findall(r'the', text))
> # ['the', 'the', 'the', 'the']  ← there, other, them의 the도 포함
> 
> # \b를 사용한 정확한 단어 검색
> print(re.findall(r'\bthe\b', text))
> # ['the', 'the']  ← 독립된 'the'만
> ```
>
> **핵심 포인트**: `\b단어\b` 패턴은 해당 단어가 독립된 단어로 존재하는 경우만 매칭합니다.

> 🔍 **예시 3.4.2: 한국어에서의 단어 경계**
>
> **상황**: 한국어 텍스트에서 `\b`가 어떻게 동작하는지 확인합니다.
>
> **풀이**:
> ```python
> import re
> text = "사과와 배, 사과주스, 풋사과"
>
> # 한글도 \w에 포함되므로 \b가 한글 단어 경계에서도 동작
> # 하지만 한국어는 띄어쓰기 기반이므로, 경계는 공백/구두점과 한글 사이에 형성
> print(re.findall(r'\b사과\b', text))
> # ['사과']  ← 첫 번째 '사과'만 (독립 단어)
> # '사과주스'의 '사과'는 뒤에 '주'(단어 문자)가 이어지므로 뒷쪽 경계 없음
> # '풋사과'의 '사과'는 앞에 '풋'(단어 문자)이 있으므로 앞쪽 경계 없음
> ```
>
> **핵심 포인트**: 한국어에서 `\b`는 한글과 공백/구두점 사이에 경계를 형성합니다. 하지만 한국어는 복합어가 붙어 쓰이는 경우가 많아, `\b`만으로 완벽한 단어 단위 매칭이 어려울 수 있습니다.

> ⚠️ **주의**: `\b`는 문자를 **소비하지 않습니다**. 다시 말해, `\b`가 매칭하는 것은 문자가 아니라 *위치*입니다. `re.findall(r'\b', 'Hello World')`를 실행하면 빈 문자열 `['', '', '', '']`이 반환됩니다 — 4개의 경계 위치를 찾았지만 각 위치에서 매칭된 문자는 없기 때문입니다.

**Section 3.4 요약**

- `\b`는 단어 문자(`\w`)와 비단어 문자(`\W`) 사이의 **경계 위치**를 매칭합니다.
- `\bword\b` 패턴으로 독립된 단어만 정밀하게 검색할 수 있습니다.
- `\B`는 `\b`의 반대로, 단어 경계가 **아닌** 위치를 매칭합니다.
- `\b`는 문자를 소비하지 않는 **앵커(제로 너비 단언)**입니다.
- 반드시 raw string(`r'\b'`)을 사용해야 합니다 (일반 문자열에서 `\b`는 백스페이스).

## 3.5 문자 클래스 조합 패턴

> 💡 **한 줄 요약**: 문자 클래스와 이전에 배운 메타문자들을 조합하면, 실용적인 패턴을 구성할 수 있습니다.

### 직관적 이해

지금까지 문자 클래스의 개별 요소들을 배웠습니다. 이제 이것들을 **조합**하여 실제 텍스트 처리에 사용할 수 있는 패턴을 만들어 봅시다. 요리를 배울 때 재료 손질법을 개별적으로 익힌 후, 실제 요리에서 여러 기법을 함께 사용하는 것과 같습니다.

이 Section에서는 Chapter 2와 3에서 배운 모든 개념을 종합적으로 활용합니다. 아직 수량자(Chapter 4)를 배우지 않았으므로, 정확한 글자 수를 지정하는 방법은 제한적이지만, 그래도 유용한 패턴을 많이 만들 수 있습니다.

### 핵심 개념 설명

**조합의 기본 원리**

정규표현식은 각 요소를 **왼쪽에서 오른쪽으로 순서대로** 매칭합니다. 따라서 패턴의 각 위치에 리터럴 문자, 메타문자, 문자 클래스를 자유롭게 배치할 수 있습니다.

```
패턴: \d\d\d-\d\d\d\d
해석: 숫자, 숫자, 숫자, 리터럴 하이픈, 숫자, 숫자, 숫자, 숫자
매칭: "010-1234"의 앞 7글자 부분인 "010-1234" 패턴 구조
```

> 🔗 **연결**: Chapter 2에서 배운 리터럴 매칭, 이스케이프(`\.`, `\-`), 앵커(`^`, `$`), 선택(`|`) 등을 문자 클래스와 함께 사용할 수 있습니다.

```python
import re

# 예: 간단한 날짜 형식 매칭 (MM/DD)
# \d\d는 숫자 두 자리, /는 리터럴 슬래시
text = "오늘은 02/28, 내일은 03/01입니다"
print(re.findall(r'\d\d/\d\d', text))
# ['02/28', '03/01']

# 예: 16진수 색상 코드의 한 자리 (#F 또는 #A 등)
# \#은 리터럴 #, [0-9A-Fa-f]는 16진수 한 자리
text = "색상: #F, #a, #G, #3"
print(re.findall(r'#[0-9A-Fa-f]', text))
# ['#F', '#a', '#3']  ← #G는 제외 (G는 16진수가 아님)
```

**앵커와 문자 클래스의 결합**

```python
# ^[A-Z]: 대문자로 시작하는 문자열
texts = ['Hello World', 'hello world', '123 abc', 'World']
for t in texts:
    if re.search(r'^[A-Z]', t):
        print(f"대문자 시작: {t}")
# 대문자 시작: Hello World
# 대문자 시작: World

# [.!?]$: 마침표, 느낌표, 물음표로 끝나는 문자열
sentences = ['Hello.', 'Hi!', 'How?', 'Bye', 'OK...']
for s in sentences:
    if re.search(r'[.!?]$', s):
        print(f"문장부호 끝: {s}")
# 문장부호 끝: Hello.
# 문장부호 끝: Hi!
# 문장부호 끝: How?
# 문장부호 끝: OK...  ← 마지막 문자가 .이므로 매칭
```

**선택(`|`)과 문자 클래스의 구분**

```python
# [abc]는 한 글자 선택, a|b|c도 결과가 같지만 원리가 다릅니다
# 하지만 여러 글자 패턴의 선택은 |만 가능합니다
text = "cat or dog or bird"

# [cd]는 'c' 또는 'd' 한 글자
print(re.findall(r'[cd]', text))
# ['c', 'd', 'd']

# cat|dog는 'cat' 또는 'dog' 전체 문자열
print(re.findall(r'cat|dog', text))
# ['cat', 'dog']
```

> 🔍 **예시 3.5.1: 간단한 시간 형식 검증**
>
> **상황**: "HH:MM" 형식의 시간 패턴을 매칭하고 싶습니다 (예: "09:30", "14:55").
>
> **풀이**:
> ```python
> import re
> times = ['09:30', '14:55', '25:99', '9:5', 'ab:cd', '00:00']
>
> # \d\d:\d\d — 숫자 두 자리, 콜론, 숫자 두 자리
> for t in times:
>     match = re.search(r'^\d\d:\d\d$', t)
>     if match:
>         print(f"매칭: {t}")
>     else:
>         print(f"불일치: {t}")
> # 매칭: 09:30
> # 매칭: 14:55
> # 매칭: 25:99  ← 형식은 맞지만 유효하지 않은 시간!
> # 불일치: 9:5  ← 숫자가 한 자리
> # 불일치: ab:cd  ← 숫자가 아님
> # 매칭: 00:00
> ```
>
> 더 정확한 시간 검증(시간은 0~23, 분은 0~59)을 하려면:
> ```python
> # 시간: [01]\d 또는 2[0-3], 분: [0-5]\d
> pattern = r'^([01]\d|2[0-3]):[0-5]\d$'
> for t in times:
>     if re.search(pattern, t):
>         print(f"유효한 시간: {t}")
>     else:
>         print(f"유효하지 않음: {t}")
> # 유효한 시간: 09:30
> # 유효한 시간: 14:55
> # 유효하지 않음: 25:99  ← 이제 올바르게 거부됨!
> # 유효하지 않음: 9:5
> # 유효하지 않음: ab:cd
> # 유효한 시간: 00:00
> ```
>
> 💬 **참고**: 위 예시에서 `()`는 선택(`|`)의 범위를 제한하기 위한 그룹핑으로 사용했습니다. 그룹핑에 대해서는 Chapter 6에서 자세히 다룹니다. 여기서는 `|`가 패턴 전체가 아닌 `(...)` 내부에서만 적용된다는 점만 이해하면 됩니다.
>
> **핵심 포인트**: 문자 클래스를 적절히 조합하면 유효 범위를 가진 패턴을 작성할 수 있습니다. `[01]\d`는 00~19, `2[0-3]`은 20~23을 커버합니다.

> 🔍 **예시 3.5.2: 단어 경계와 문자 클래스를 결합한 검색**
>
> **상황**: 영문 텍스트에서 대문자로 시작하는 단어만 찾고 싶습니다.
>
> **풀이**:
> ```python
> import re
> text = "The Quick Brown Fox jumps over the Lazy Dog"
>
> # \b[A-Z]: 단어 경계 뒤에 대문자가 오는 위치
> print(re.findall(r'\b[A-Z]', text))
> # ['T', 'Q', 'B', 'F', 'L', 'D']
> ```
>
> 이 예시에서는 대문자 **첫 글자**만 추출됩니다. 대문자로 시작하는 **단어 전체**를 추출하려면 수량자(Chapter 4)가 필요합니다.
>
> **핵심 포인트**: `\b`(위치 지정) + `[A-Z]`(문자 종류 지정)의 조합으로 "단어의 시작 위치에 있는 대문자"를 정밀하게 찾을 수 있습니다.

> ⚠️ **주의**: 문자 클래스를 조합할 때, 각 요소가 **한 글자**에 대응한다는 것을 항상 기억하세요. `\d\d`는 "두 자리 숫자"이지, "10 이상의 숫자"가 아닙니다. `\d\d`는 `00`, `01`, ..., `99`를 매칭하며, `100`을 매칭하려면 `\d\d\d`가 필요합니다. 가변 길이 반복은 Chapter 4의 수량자에서 다룹니다.

**Section 3.5 요약**

- 리터럴 문자, 메타문자, 문자 클래스, 앵커를 자유롭게 조합하여 패턴을 구성합니다.
- 각 요소는 **왼쪽에서 오른쪽으로** 순서대로 매칭됩니다.
- 문자 클래스(`[abc]`)는 한 글자 선택, 선택(`|`)은 여러 글자 패턴의 선택에 사용합니다.
- `\b`와 문자 클래스를 결합하면 "단어의 특정 위치에 있는 특정 종류의 문자"를 정밀하게 찾을 수 있습니다.
- 각 요소가 **한 글자에 대응**한다는 원칙을 기억하세요. 가변 길이 반복은 Chapter 4에서 다룹니다.

## Chapter 핵심 요약 (Chapter Summary)

이번 Chapter에서는 정규표현식에서 **특정 종류의 문자**를 지정하는 방법과 **단어 경계**를 다루었습니다.

1. **문자 클래스 `[...]`** (Section 3.1): 대괄호 안에 문자를 나열하여, 해당 위치에서 허용할 문자 집합을 정의합니다. 대괄호 안에서는 대부분의 메타문자가 리터럴로 취급됩니다.

2. **범위와 부정** (Section 3.2): 하이픈(`-`)으로 연속 문자 범위를 지정하고(`[a-z]`, `[0-9]`), 캐럿(`^`)으로 특정 문자를 제외합니다(`[^0-9]`).

3. **사전 정의 문자 클래스** (Section 3.3): `\d`(숫자), `\w`(단어 문자), `\s`(공백)은 자주 쓰는 문자 클래스의 축약형이며, 대문자 버전(`\D`, `\W`, `\S`)은 부정형입니다.

4. **단어 경계** (Section 3.4): `\b`는 단어 문자와 비단어 문자 사이의 경계 위치를 매칭하는 앵커로, `\bword\b`로 독립된 단어를 정밀 검색합니다.

5. **조합 패턴** (Section 3.5): 문자 클래스를 앵커, 리터럴, 선택 등과 결합하면 날짜, 시간, 코드 등 실용적인 패턴을 만들 수 있습니다.

**한계 설명**: 수량자 없이는 **가변 길이의 대문자 단어**를 하나의 패턴으로 표현할 수 없습니다. 2글자, 3글자, 4글자... 각각에 대해 별도의 패턴이 필요합니다. Chapter 4에서 수량자를 배우면 `\b[A-Z][A-Z]+\b` 한 줄로 "두 글자 이상의 대문자 단어"를 완벽하게 표현할 수 있습니다.

**확장 생각**: 이 문제는 수량자의 필요성을 체감하게 해줍니다. "특정 문자가 반복되는 횟수"를 유연하게 지정하는 것은 정규표현식의 핵심 능력이며, 다음 Chapter에서 배울 내용입니다.

> **Problem 3.3**: 다음 규칙에 맞는 **사용자 ID 검증 패턴**을 작성하세요. 아직 수량자를 배우지 않았으므로, 정확히 **6글자 길이**의 사용자 ID만 검증합니다.
>
> **규칙**:
> - 정확히 6글자
> - 첫 글자는 영문 소문자
> - 나머지 5글자는 영문 소문자 또는 숫자
> - 공백이나 특수문자 불허
>
> **테스트 케이스**:
> - `"kim123"` → 유효 ✅
> - `"1abcde"` → 유효하지 않음 (숫자로 시작) ❌
> - `"abcDE1"` → 유효하지 않음 (대문자 포함) ❌
> - `"abc12"` → 유효하지 않음 (5글자) ❌
> - `"abc1234"` → 유효하지 않음 (7글자) ❌

<details>
<summary>풀이 가이드</summary>

**핵심 아이디어**: `^`와 `$` 앵커로 전체 문자열을 커버하고, 각 위치에 적절한 문자 클래스를 배치합니다.

**풀이 전략**: 첫 글자 `[a-z]` + 나머지 5글자 `[a-z0-9]` × 5 + 끝 앵커

**상세 풀이**:
```python
import re

def validate_id(user_id):
    pattern = r'^[a-z][a-z0-9][a-z0-9][a-z0-9][a-z0-9][a-z0-9]$'
    return bool(re.search(pattern, user_id))

# 테스트
test_cases = [
    ("kim123", True),
    ("1abcde", False),
    ("abcDE1", False),
    ("abc12", False),
    ("abc1234", False),
    ("hello1", True),
]

for user_id, expected in test_cases:
    result = validate_id(user_id)
    status = "✅" if result == expected else "❌"
    print(f"{status} '{user_id}': {result} (기대: {expected})")
# ✅ 'kim123': True (기대: True)
# ✅ '1abcde': False (기대: False)
# ✅ 'abcDE1': False (기대: False)
# ✅ 'abc12': False (기대: False)
# ✅ 'abc1234': False (기대: False)
# ✅ 'hello1': True (기대: True)
```

**확장 생각**: `[a-z0-9]`를 5번 반복 작성하는 것은 분명 불편합니다. Chapter 4에서 범위 수량자 `{n}`을 배우면 `[a-z][a-z0-9]{5}`로 간결하게 표현할 수 있고, `{4,12}`처럼 가변 길이도 지원할 수 있습니다. 현재 Chapter에서의 한계를 인식하는 것이 다음 Chapter 학습의 동기가 됩니다.

## 다음 단계 안내 (What's Next)

이번 Chapter에서 문자 클래스와 단어 경계를 배웠지만, 한 가지 아쉬운 점을 느끼셨을 것입니다 — **반복 횟수를 유연하게 지정할 수 없다**는 것입니다. `\d\d\d`처럼 일일이 나열하는 것은 비효율적이고, 가변 길이에 대응할 수 없습니다.
