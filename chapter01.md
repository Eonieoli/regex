## 1.1 정규표현식이란 무엇인가

> 💡 **한 줄 요약**: 정규표현식은 문자열에서 특정 패턴을 찾고, 추출하고, 변환하기 위한 "문자열 검색의 미니 언어"입니다.

### 직관적 이해

여러분이 거대한 도서관에서 책을 찾는다고 상상해봅시다. 만약 "해리 포터와 마법사의 돌"이라는 정확한 제목을 알고 있다면, 사서에게 그 제목을 말하면 됩니다. 하지만 만약 "제목에 '마법'이 들어가고, 2000년 이후에 출판된 소설"을 찾고 싶다면 어떨까요? 이때는 단순히 제목 하나를 말하는 것이 아니라, **찾고 싶은 책의 "규칙"이나 "패턴"**을 설명해야 합니다.

정규표현식(Regular Expression)은 바로 이런 상황에서 사용됩니다. 텍스트 속에서 특정 **패턴**에 맞는 문자열을 찾기 위한 도구입니다. "abc@gmail.com"이라는 정확한 이메일을 찾는 것이 아니라, "이메일 주소처럼 생긴 모든 문자열"을 찾을 수 있는 것이죠.

정규표현식이 필요한 상황은 일상적인 프로그래밍에서 놀랄 만큼 자주 등장합니다.

- 수천 줄의 로그 파일에서 에러 메시지만 골라내고 싶을 때
- 사용자가 입력한 전화번호가 올바른 형식인지 확인하고 싶을 때
- 웹페이지에서 모든 URL 링크를 추출하고 싶을 때
- 문서에서 날짜 표기 형식을 한꺼번에 통일하고 싶을 때

이 모든 작업의 공통점은 **"특정 규칙(패턴)에 맞는 문자열을 찾거나 변환한다"**는 것입니다. 정규표현식은 바로 이 "규칙"을 간결하게 표현하는 언어입니다.

### 핵심 개념 설명

> 📝 **정규표현식(Regular Expression)**: 문자열에서 특정 패턴을 기술하기 위한 형식 언어(formal language)입니다. 줄여서 **regex** 또는 **regexp**라고도 부릅니다. 정규표현식을 사용하면 복잡한 문자열 검색·추출·치환 작업을 간결한 패턴 한 줄로 수행할 수 있습니다.

정규표현식이라는 이름이 다소 딱딱하게 느껴질 수 있습니다. "정규(Regular)"라는 단어는 컴퓨터 과학의 **형식 언어 이론(formal language theory)**에서 온 것으로, 수학자 스티븐 클리니(Stephen Kleene)가 1950년대에 "정규 집합(regular set)"이라는 개념을 정의한 데서 유래합니다. 이후 유닉스(Unix) 운영체제의 텍스트 편집기 `ed`에서 `/regular expression/` 형태로 검색 기능이 구현되면서, 프로그래머들의 일상 도구가 되었습니다. 오늘날에는 파이썬, 자바스크립트, 자바, C# 등 거의 모든 프로그래밍 언어에서 정규표현식을 지원합니다.

정규표현식을 이해하기 위해 가장 중요한 세 가지 개념이 있습니다.

> 📝 **패턴(Pattern)**: 찾고자 하는 문자열의 규칙을 기술한 것입니다. 정규표현식의 핵심은 이 패턴을 작성하는 것입니다. 예를 들어, "숫자 3개 뒤에 하이픈이 오고, 다시 숫자 4개가 오는 문자열"이라는 규칙이 하나의 패턴입니다.

> 📝 **매칭(Matching)**: 주어진 문자열이 패턴에 일치하는지 확인하는 과정입니다. 매칭의 결과는 "일치한다(match)" 또는 "일치하지 않는다(no match)"입니다. 일치하는 부분이 있다면, 그 부분의 위치와 내용도 알 수 있습니다.

> 📝 **정규표현식 엔진(Regex Engine)**: 패턴을 해석하고, 문자열에 대해 매칭을 수행하는 프로그램입니다. 파이썬에서는 `re` 모듈이 정규표현식 엔진 역할을 합니다. 우리가 패턴을 작성하면, 엔진이 그 패턴을 읽고 문자열을 하나하나 검사하여 일치하는 부분을 찾아줍니다.

이 세 개념의 관계를 정리하면 다음과 같습니다.

```
[우리가 작성] 패턴(Pattern)  ──→  정규표현식 엔진  ──→  매칭 결과
                                      ↑
[검사 대상]   대상 문자열    ──────┘
```

### 정규표현식의 활용 분야

정규표현식은 텍스트를 다루는 거의 모든 분야에서 활용됩니다. 주요 활용 분야를 살펴봅시다.

**1. 데이터 검증(Validation)**: 사용자 입력이 올바른 형식인지 확인합니다.
- 이메일 주소가 유효한 형식인지 확인
- 전화번호가 올바른 형식(예: 010-1234-5678)인지 확인
- 비밀번호가 보안 규칙(대문자 포함, 특수문자 포함 등)을 만족하는지 확인

**2. 데이터 추출(Extraction)**: 비정형 텍스트에서 원하는 정보만 골라냅니다.
- 웹페이지 HTML에서 모든 링크 URL 추출
- 로그 파일에서 IP 주소나 타임스탬프 추출
- 문서에서 금액이나 날짜 등 특정 형식의 데이터 추출

**3. 데이터 변환(Transformation)**: 텍스트의 형식을 일괄적으로 변경합니다.
- 날짜 형식을 "2024-03-15"에서 "2024년 3월 15일"로 변환
- 전화번호에서 하이픈을 제거하거나 추가
- 코드에서 변수명의 명명 규칙을 일괄 변경

**4. 텍스트 분석(Analysis)**: 텍스트의 구조를 파악하고 분석합니다.
- 로그 파일에서 에러 발생 빈도 분석
- 문서에서 특정 단어의 출현 패턴 분석
- 소스 코드에서 주석의 비율 분석

### 상세 예시

> 🔍 **예시 1.1.1: 전화번호 찾기 — 일상적 비유**
>
> **상황**: 수천 줄짜리 고객 상담 기록 파일에서 모든 전화번호를 찾아야 합니다. 전화번호는 "010-1234-5678", "02-123-4567", "031-1234-5678" 등 다양한 형식으로 적혀 있습니다.
>
> **문자열 메서드로 접근한다면**: 모든 가능한 전화번호 형식을 나열하거나, 복잡한 조건문을 작성해야 합니다. "숫자가 나오고, 하이픈이 오고, 다시 숫자가 오는" 규칙을 코드로 표현하려면 여러 줄이 필요합니다.
>
> **정규표현식으로 접근한다면**: `\d{2,3}-\d{3,4}-\d{4}`라는 패턴 하나로 위의 모든 형식을 찾을 수 있습니다. (이 패턴의 문법은 이후 Chapter에서 하나씩 배웁니다. 지금은 "이런 것이 가능하다"는 점만 기억하세요!)
>
> **핵심 포인트**: 정규표현식은 "정확한 문자열"이 아니라 "문자열의 패턴(규칙)"을 기술할 수 있기 때문에, 다양한 형태의 전화번호를 한 번에 찾아낼 수 있습니다.

> 🔍 **예시 1.1.2: 로그 파일에서 에러 추출 — 전문적 활용**
>
> **상황**: 웹 서버의 접속 로그에서 상태 코드가 500번대(서버 에러)인 요청만 찾고 싶습니다. 로그의 각 줄은 `192.168.1.1 - - [10/Oct/2024:13:55:36] "GET /index.html HTTP/1.1" 200 2326` 같은 형식입니다.
>
> **정규표현식이 없다면**: 각 줄을 공백으로 분리하고, 상태 코드 위치를 파악하고, 해당 값이 500~599 사이인지 조건문으로 확인해야 합니다. 로그 형식이 조금만 달라지면 코드를 다시 수정해야 합니다.
>
> **정규표현식이 있다면**: `" 5\d\d "`라는 간단한 패턴으로 상태 코드 500번대를 포함하는 줄을 바로 걸러낼 수 있습니다.
>
> **핵심 포인트**: 정규표현식은 텍스트의 구조적 패턴을 간결하게 표현할 수 있어, 로그 분석 같은 반복적인 텍스트 처리 작업을 크게 단순화합니다.

### Section 요약

- **정규표현식**은 문자열에서 특정 패턴을 찾고, 추출하고, 변환하기 위한 미니 언어이다
- 핵심 3개념: **패턴**(찾을 규칙), **매칭**(일치 확인 과정), **엔진**(매칭을 수행하는 프로그램)
- 데이터 검증, 추출, 변환, 분석 등 텍스트 처리의 거의 모든 영역에서 활용된다
- 1950년대 수학 이론에서 출발하여 현재 모든 주요 프로그래밍 언어에서 지원되는 보편적 도구이다

## 1.2 왜 정규표현식을 배워야 하는가

> 💡 **한 줄 요약**: 파이썬 문자열 메서드만으로는 해결하기 어렵거나 코드가 지나치게 길어지는 텍스트 처리 문제를, 정규표현식은 간결하고 강력하게 해결합니다.

### 직관적 이해

맥가이버 칼(스위스 군용 칼)을 생각해봅시다. 일상적인 작업에는 일반 칼이면 충분합니다. 하지만 와인 오프너가 필요하거나, 드라이버가 필요하거나, 가위가 필요한 상황이 동시에 닥치면 맥가이버 칼이 훨씬 유용하겠죠. 파이썬의 문자열 메서드(`find`, `replace`, `split` 등)는 일반 칼처럼 간단한 작업에 적합하고, 정규표현식은 맥가이버 칼처럼 복잡한 텍스트 처리 상황에서 빛을 발합니다.

중요한 것은, 정규표현식을 배운다고 해서 문자열 메서드를 버리는 것이 아니라는 점입니다. **상황에 따라 적절한 도구를 선택하는 능력**이 진정한 실력입니다.

### 핵심 개념 설명

파이썬의 문자열 메서드가 처리하기 어려운 상황을 구체적으로 살펴봅시다.

#### 문자열 메서드의 한계

파이썬 문자열 메서드는 **고정된 문자열**을 다루는 데 적합합니다. 그러나 다음과 같은 상황에서는 한계에 부딪힙니다.

**한계 1: 유연한 패턴 매칭이 불가능하다**

```python
# "apple" 또는 "Apple" 또는 "APPLE"을 찾고 싶다면?
text = "I like Apple and apple products"
# lower()를 쓰면 원본 텍스트가 변형된다
text.lower().find("apple")  # 찾을 수는 있지만 원래 위치/형태 정보 손실
```

단순히 대소문자만의 문제가 아닙니다. "전화번호 형식의 문자열"처럼 구조는 같지만 구체적 값이 다른 수많은 문자열을 하나의 규칙으로 찾는 것은 문자열 메서드만으로는 매우 번거롭습니다.

**한계 2: 복합 조건의 검색이 복잡하다**

```python
# 텍스트에서 4자리 연도(1900~2099)를 모두 찾고 싶다면?
text = "설립: 1998, 상장: 2003, 매출: 1조 2000억"

# 문자열 메서드로는...
results = []
for i in range(len(text)):
    if i + 4 <= len(text) and text[i:i+4].isdigit():
        year = int(text[i:i+4])
        if 1900 <= year <= 2099:
            results.append(text[i:i+4])
# 결과에 "2000"(금액의 일부)도 섞일 수 있다 — 문맥 구분이 어렵다
```

문자열 메서드로도 불가능하지는 않지만, 코드가 길어지고 엣지 케이스 처리가 어렵습니다.

**한계 3: 추출과 변환을 동시에 하기 어렵다**

"2024-03-15" 형식의 날짜를 "2024년 3월 15일"로 변환하려면, 먼저 하이픈으로 분리하고, 각 부분을 추출하고, 다시 조합해야 합니다. 문자열 메서드로도 가능하지만, 원본 텍스트 안에 이런 날짜가 여러 개 섞여 있다면 코드는 급격히 복잡해집니다.

#### 정규표현식의 강점

정규표현식은 위의 한계들을 어떻게 극복할까요?

| 문자열 메서드의 한계 | 정규표현식의 해결 |
|---|---|
| 고정 문자열만 검색 가능 | **패턴으로 유연한 검색** 가능 |
| 복합 조건에 여러 줄의 코드 필요 | **하나의 패턴**으로 복합 조건 표현 |
| 추출·변환을 별도로 처리 | **매칭과 동시에 추출·변환** 가능 |
| 반복 구조 표현이 어려움 | **수량자로 반복 구조를 간결하게** 표현 |

그러나 정규표현식이 만능은 아닙니다.

#### 문자열 메서드가 더 나은 경우

| 상황 | 권장 도구 | 이유 |
|---|---|---|
| 고정된 문자열 포함 여부 확인 | `in` 연산자, `find()` | 더 빠르고 읽기 쉬움 |
| 고정된 문자열 치환 | `replace()` | 더 직관적 |
| 공백 기준 분리 | `split()` | 더 간단함 |
| 앞뒤 공백 제거 | `strip()` | 더 효율적 |

> ⚠️ **주의**: 정규표현식은 강력하지만, 단순한 작업에 정규표현식을 사용하면 오히려 코드가 읽기 어려워질 수 있습니다. "정규표현식으로 문제를 풀려다 보면, 이제 문제가 두 개가 된다"는 유명한 농담이 있을 정도입니다. **적재적소**가 중요합니다.

### 상세 예시

> 🔍 **예시 1.2.1: 이메일 추출 — 문자열 메서드 vs 정규표현식**
>
> **상황**: 다음 텍스트에서 모든 이메일 주소를 찾아야 합니다.
> ```
> 문의: support@example.com 또는 sales@company.co.kr로 연락주세요.
> 개인 이메일: kim.minho_99@gmail.com
> ```
>
> **문자열 메서드로 접근** (개념적 설명):
> - `@` 기호가 있는 위치를 먼저 찾고
> - `@` 앞뒤로 공백이나 특수문자가 나올 때까지 확장하고
> - 유효한 이메일 문자(영문, 숫자, `.`, `_`, `-`)만 포함하는지 확인
> - 이 로직을 구현하면 약 15~20줄의 코드가 필요
>
> **정규표현식으로 접근** (개념적 설명):
> - `[\w.-]+@[\w.-]+\.\w+` 와 같은 패턴 한 줄로 세 개의 이메일을 모두 찾을 수 있음
> - (이 패턴의 문법은 이후 Chapter들에서 차근차근 배웁니다)
>
> **핵심 포인트**: 정규표현식은 "문자열의 구조적 규칙"을 간결하게 표현할 수 있기 때문에, 복잡한 패턴 매칭 작업에서 코드량을 극적으로 줄여줍니다.

> 🔍 **예시 1.2.2: 도구 선택의 판단 기준**
>
> **상황**: 다양한 텍스트 처리 작업에 대해 적절한 도구를 선택해봅시다.
>
> | 작업 | 적합한 도구 | 이유 |
> |---|---|---|
> | 문자열에 "error"가 포함되어 있는지 확인 | `"error" in text` | 단순 포함 여부 — 문자열 메서드가 간단 |
> | 대소문자 무관하게 "error" 찾기 | `"error" in text.lower()` 또는 정규표현식 | 간단한 경우 문자열 메서드, 원문 보존 필요시 정규표현식 |
> | "error", "Error", "ERROR", "warning", "Warning" 모두 찾기 | 정규표현식 | 여러 패턴을 하나의 규칙으로 — 정규표현식이 효율적 |
> | 로그에서 IP 주소 형식의 문자열 모두 추출 | 정규표현식 | 구조적 패턴 매칭 — 정규표현식 필수 |
> | 쉼표(`,`) 기준으로 문자열 분리 | `text.split(",")` | 단일 고정 구분자 — 문자열 메서드가 간단 |
> | 쉼표, 세미콜론, 탭 등 여러 구분자로 분리 | 정규표현식 | 복수 구분자 — 정규표현식이 효율적 |
>
> **핵심 포인트**: "이 작업이 고정된 문자열을 다루는가, 아니면 패턴(규칙)을 다루는가?"가 도구 선택의 가장 중요한 기준입니다.

### Section 요약

- 파이썬 문자열 메서드는 **고정된 문자열** 처리에 적합하지만, **유연한 패턴 매칭**, **복합 조건 검색**, **추출·변환 동시 처리**에 한계가 있다
- 정규표현식은 이런 한계를 **패턴 기반의 간결한 표현**으로 극복한다
- 그러나 단순한 작업에는 문자열 메서드가 더 적합하다 — **적재적소**가 핵심
- 도구 선택 기준: "고정 문자열 → 문자열 메서드" / "패턴(규칙) 기반 → 정규표현식"

## 1.3 정규표현식의 기본 구조

> 💡 **한 줄 요약**: 정규표현식의 패턴은 "있는 그대로 매칭되는 **리터럴 문자**"와 "특별한 의미를 가진 **메타문자**"의 조합으로 이루어집니다.

### 직관적 이해

정규표현식의 패턴을 이해하기 위해, **검색 엔진의 검색어**와 비교해봅시다.

- 구글에서 `파이썬 정규표현식`을 검색하면, "파이썬"과 "정규표현식"이라는 단어가 포함된 페이지를 찾습니다. 이것은 있는 그대로의 단어를 검색한 것이죠.
- 그런데 구글에서 `"파이썬 정규표현식"`처럼 따옴표로 감싸면, 정확히 그 문구가 연속으로 나오는 페이지만 찾습니다.
- 또한 `파이썬 -자바`처럼 `-` 기호를 붙이면, "자바"가 포함되지 않은 결과를 찾습니다.

여기서 핵심은, `파이썬`이라는 글자 자체는 있는 그대로 검색되지만, `""`나 `-` 같은 기호는 검색의 **동작 방식을 제어하는 특별한 의미**를 가진다는 것입니다.

정규표현식도 똑같은 원리입니다.

- **리터럴 문자**: 있는 그대로 매칭되는 일반 문자 (예: `a`, `1`, `가`)
- **메타문자**: 특별한 의미를 가진 문자 (예: `.`, `*`, `^`, `$`)

이 두 종류의 문자를 조합하여 패턴을 만드는 것이 정규표현식의 전부입니다. 패턴을 "읽는 법"과 "쓰는 법"을 익히는 것이 이 과정의 핵심 목표입니다.

### 핵심 개념 설명

#### 리터럴 문자 (Literal Characters)

> 📝 **리터럴 문자(Literal Character)**: 정규표현식에서 특별한 의미 없이 **문자 그 자체로 매칭되는 문자**입니다. 영문자, 숫자, 한글, 공백 등 대부분의 문자가 리터럴 문자에 해당합니다.

패턴 `hello`는 문자열에서 정확히 `h`, `e`, `l`, `l`, `o`가 연속으로 나오는 부분을 찾습니다. 각 문자가 리터럴(있는 그대로)로 매칭되는 것이죠. 이것은 여러분이 이미 알고 있는 파이썬의 `"hello" in text`와 비슷한 개념입니다.

리터럴 매칭의 핵심 특성은 다음과 같습니다.

- **대소문자를 구분합니다**: 패턴 `hello`는 "Hello"나 "HELLO"와 매칭되지 않습니다
- **연속된 문자만 매칭합니다**: 패턴 `cat`은 "cat"과 매칭되지만, "c a t"과는 매칭되지 않습니다
- **부분 매칭이 가능합니다**: 패턴 `cat`은 "my cat is cute"에서 "cat" 부분과 매칭됩니다

#### 메타문자 (Metacharacters) — 개념적 소개

> 📝 **메타문자(Metacharacter)**: 정규표현식에서 **문자 그 자체가 아닌 특별한 의미나 기능**을 가진 문자입니다. 메타문자를 사용하면 "아무 문자 하나", "줄의 시작", "0회 이상 반복" 같은 규칙을 표현할 수 있습니다.

파이썬 정규표현식에서 사용되는 핵심 메타문자는 다음과 같습니다.

```
.  ^  $  *  +  ?  |  \  (  )  [  ]  {  }
```

이 Chapter에서는 메타문자의 **존재와 역할**만 소개합니다. 각 메타문자의 상세한 의미와 사용법은 Chapter 2부터 순서대로 배웁니다. 지금은 "이런 문자들이 정규표현식에서 특별하게 취급된다"는 점만 기억해주세요.

몇 가지 메타문자를 맛보기로 살펴봅시다.

| 메타문자 | 의미 (맛보기) | 예시 (개념적) |
|---|---|---|
| `.` | 아무 문자 하나 | `a.c` → "abc", "a1c", "a@c" 등과 매칭 |
| `*` | 바로 앞 요소가 0번 이상 반복 | `ab*c` → "ac", "abc", "abbc", "abbbc" 등과 매칭 |
| `^` | 문자열의 시작 | `^Hello` → 문자열이 "Hello"로 시작하는 경우만 매칭 |
| `$` | 문자열의 끝 | `end$` → 문자열이 "end"로 끝나는 경우만 매칭 |

> 💬 **참고**: 위의 메타문자들은 Chapter 2에서 하나씩 자세히 다룹니다. 수량자(`*`, `+`, `?`)는 Chapter 4에서 심도 있게 학습합니다. 지금은 개념적 이해만으로 충분합니다.

#### 패턴의 구성

정규표현식의 패턴은 리터럴 문자와 메타문자의 조합입니다. 간단한 예를 들어보겠습니다.

패턴 `cat`의 구성:
```
c  →  리터럴 문자 'c'
a  →  리터럴 문자 'a'
t  →  리터럴 문자 't'
```
이 패턴은 순수하게 리터럴로만 이루어져 있으므로, 정확히 "cat"이라는 문자열과 매칭됩니다.

패턴 `c.t`의 구성:
```
c  →  리터럴 문자 'c'
.  →  메타문자: 아무 문자 하나
t  →  리터럴 문자 't'
```
이 패턴은 "c로 시작하고, 아무 문자 하나가 오고, t로 끝나는 3글자"와 매칭됩니다. "cat", "cut", "c1t", "c@t" 등이 모두 매칭됩니다.

이처럼 리터럴 문자와 메타문자를 조합하면, **고정된 부분**과 **유연한 부분**을 동시에 표현할 수 있습니다. 이것이 정규표현식의 핵심 원리입니다.

### 상세 예시

> 🔍 **예시 1.3.1: 리터럴 패턴의 매칭**
>
> **상황**: 패턴 `Python`이 다양한 문자열과 매칭되는지 확인해봅시다.
>
> | 대상 문자열 | 매칭 결과 | 이유 |
> |---|---|---|
> | `"I love Python"` | ✅ 매칭 ("Python" 부분) | 리터럴 "Python"이 포함되어 있음 |
> | `"python is great"` | ❌ 비매칭 | 대소문자 구분: "python" ≠ "Python" |
> | `"PYTHON"` | ❌ 비매칭 | 대소문자 구분: "PYTHON" ≠ "Python" |
> | `"Python3 and Python2"` | ✅ 매칭 (2곳) | "Python"이 두 번 나타남 |
> | `"Pytho"` | ❌ 비매칭 | "n"이 빠져서 완전한 매칭이 아님 |
>
> **핵심 포인트**: 리터럴 패턴은 정확한 문자 일치를 요구합니다. 대소문자도 구분하며, 부분 문자열로서 포함되어 있으면 매칭됩니다.

> 🔍 **예시 1.3.2: 리터럴과 메타문자의 구분**
>
> **상황**: 다음 패턴들에서 어떤 문자가 리터럴이고 어떤 문자가 메타문자인지 구분해봅시다.
>
> | 패턴 | 리터럴 문자 | 메타문자 | 패턴의 의미 (개념적) |
> |---|---|---|---|
> | `hello` | h, e, l, l, o | (없음) | 정확히 "hello" |
> | `h.llo` | h, l, l, o | `.` | "h + 아무 문자 + llo" |
> | `^hello` | h, e, l, l, o | `^` | 문자열 시작이 "hello" |
> | `hello$` | h, e, l, l, o | `$` | 문자열 끝이 "hello" |
>
> **핵심 포인트**: 같은 알파벳 `h`라도 정규표현식에서는 항상 리터럴입니다. 반면 `.`, `^`, `$` 같은 기호는 메타문자로서 특별한 의미를 가집니다. 어떤 문자가 메타문자인지 아는 것이 정규표현식 학습의 첫걸음입니다.

> ⚠️ **주의**: 메타문자 목록(`. ^ $ * + ? | \ ( ) [ ] { }`)을 통째로 외우려 하지 마세요. 각 메타문자는 Chapter 2~4에서 하나씩 배울 때 자연스럽게 익혀집니다. 지금은 "정규표현식에는 리터럴 문자와 메타문자 두 종류가 있다"는 구분만 기억하면 충분합니다.

### Section 요약

- 정규표현식 패턴은 **리터럴 문자**(있는 그대로 매칭)와 **메타문자**(특별한 의미)의 조합이다
- 리터럴 매칭은 대소문자를 구분하며, 부분 문자열 매칭이 가능하다
- 핵심 메타문자: `. ^ $ * + ? | \ ( ) [ ] { }` — 각각의 상세 의미는 이후 Chapter에서 학습
- 리터럴과 메타문자를 조합하면, 고정된 부분과 유연한 부분을 동시에 표현할 수 있다

## 1.4 정규표현식 테스트 환경 구축

> 💡 **한 줄 요약**: 정규표현식은 반드시 직접 실행해보면서 학습해야 하며, 파이썬 인터프리터와 온라인 테스터(regex101)가 핵심 학습 도구입니다.

### 직관적 이해

요리를 배울 때 레시피만 읽는 것과 직접 주방에 서서 재료를 만지고 불을 조절하는 것은 완전히 다른 경험입니다. 정규표현식도 마찬가지입니다. 패턴을 "읽는 것"과 "직접 작성하고 결과를 확인하는 것"은 학습 효과에서 큰 차이가 납니다.

이 Section에서는 앞으로의 학습에서 계속 사용할 두 가지 핵심 도구를 설정합니다.

1. **파이썬 인터프리터**: 실제 코드를 실행하며 정규표현식을 테스트
2. **regex101.com**: 패턴을 시각적으로 분석하고 디버깅할 수 있는 온라인 도구

### 핵심 개념 설명

#### 도구 1: 파이썬 인터프리터에서 정규표현식 맛보기

파이썬에서 정규표현식을 사용하려면 `re` 모듈을 불러와야(import) 합니다. 아래는 가장 간단한 정규표현식 사용 예시입니다. 지금은 코드의 세부 문법을 이해할 필요 없이, **"이런 식으로 사용하는구나"** 정도만 파악하면 됩니다.

> 💬 **참고**: `re` 모듈의 함수(`search`, `findall` 등)는 Chapter 5에서 자세히 다룹니다. 지금은 테스트 환경을 익히는 것이 목적이므로, 아래 코드를 따라 쳐보고 결과를 확인하는 것에 집중하세요.

```python
import re  # 정규표현식 모듈 불러오기

# 패턴 "cat"이 문자열에 있는지 찾기
text = "I have a cat and a catfish"
result = re.search("cat", text)

if result:
    print(f"찾았습니다: '{result.group()}'")
    print(f"위치: {result.start()}번째 ~ {result.end()}번째")
else:
    print("찾지 못했습니다")
```

실행 결과:
```
찾았습니다: 'cat'
위치: 9번째 ~ 12번째
```

`re.search("cat", text)`는 "`text` 문자열에서 패턴 `"cat"`과 매칭되는 첫 번째 부분을 찾아라"라는 의미입니다. 결과를 보면, 9번째 위치(0부터 셈)에서 "cat"을 찾았음을 알 수 있습니다.

문자열 안의 모든 매칭을 찾으려면 `re.findall()`을 사용합니다.

```python
import re

text = "I have a cat and a catfish"
results = re.findall("cat", text)
print(results)  # ['cat', 'cat']  — "cat"이 2번 발견됨
```

## Chapter 핵심 요약 (Chapter Summary)

이 Chapter에서는 정규표현식의 세계로 들어가는 첫 발을 내디뎠습니다.

- **Section 1.1**: 정규표현식은 **문자열에서 패턴을 찾고, 추출하고, 변환하는 미니 언어**입니다. 핵심 3개념은 패턴(규칙), 매칭(일치 확인), 엔진(매칭 수행자)이며, 데이터 검증·추출·변환·분석 등 모든 텍스트 처리에 활용됩니다.

- **Section 1.2**: 파이썬 문자열 메서드는 고정 문자열 처리에 적합하지만, **유연한 패턴 매칭**에는 한계가 있습니다. 정규표현식은 이 한계를 극복하지만, 단순한 작업에는 문자열 메서드가 더 낫습니다. **적재적소**가 핵심입니다.

- **Section 1.3**: 정규표현식 패턴은 **리터럴 문자**(있는 그대로 매칭)와 **메타문자**(특별한 의미)의 조합으로 이루어집니다. 이 두 종류의 문자를 구분하는 것이 정규표현식 이해의 첫걸음입니다.

- **Section 1.4**: 정규표현식 학습의 핵심은 **직접 실험하기**입니다. 파이썬 인터프리터와 regex101.com을 활용한 실습 중심 학습이 가장 효과적입니다.

> **Exercise 1.2**: 다음 각 시나리오에서 (1) 정규표현식이 필요한 이유, (2) 순수 문자열 메서드로 구현할 경우의 어려움을 설명하시오.
>
> **시나리오 A**: 수천 줄의 고객 리뷰 텍스트에서 "별점: N점" (N은 1~5 사이 숫자) 형태의 모든 평점 정보를 추출해야 한다.
>
> **시나리오 B**: 설정 파일에서 `key=value` 형식의 모든 줄을 찾아, key와 value를 각각 분리해야 한다. 단, key와 value에는 공백이 포함될 수 있고, 주석 줄(`#`으로 시작)은 무시해야 한다.
>
> <details>
> <summary>모범 답안</summary>
>
> **시나리오 A**:
> - **(1) 정규표현식이 필요한 이유**: "별점: N점"에서 N은 1~5 중 하나이므로 고정 문자열이 아닙니다. "별점: 1점", "별점: 2점", ..., "별점: 5점"을 모두 찾아야 하며, N의 값을 추출해야 합니다. 정규표현식을 사용하면 하나의 패턴으로 이 모든 경우를 처리할 수 있습니다.
> - **(2) 문자열 메서드의 어려움**: `find()`로 "별점: "를 찾고, 그 뒤의 문자가 숫자인지 확인하고, 그 뒤에 "점"이 오는지 확인해야 합니다. 각 단계마다 인덱스를 조작해야 하므로 코드가 복잡하고, "별점: 10점"(범위 밖) 같은 엣지 케이스 처리가 번거롭습니다.
>
> **시나리오 B**:
> - **(1) 정규표현식이 필요한 이유**: key와 value 사이의 `=`을 기준으로 분리해야 하지만, value 자체에 `=`이 포함될 수도 있습니다. 또한 `#` 주석 줄을 먼저 필터링해야 합니다. 정규표현식을 사용하면 "줄 시작이 `#`이 아니고, `=`을 기준으로 앞뒤를 캡처"하는 규칙을 하나의 패턴으로 표현할 수 있습니다.
> - **(2) 문자열 메서드의 어려움**: `split("=")`은 value에 `=`이 포함된 경우 오동작합니다. `split("=", 1)`로 해결할 수 있지만, 주석 필터링, 공백 처리 등을 조합하면 코드가 상당히 길어집니다.
> </details>

- **공식 문서**: [Python re 모듈 공식 문서](https://docs.python.org/3/library/re.html) — 레퍼런스로 참고하세요. 지금 당장 전부 읽을 필요는 없습니다.
- **입문 연습**: [RegexOne](https://regexone.com) — 대화형으로 정규표현식 기초를 연습할 수 있는 사이트입니다.
- **서적**: Jeffrey Friedl, 《Mastering Regular Expressions》 (O'Reilly) — 정규표현식의 바이블이라 불리는 책. 이 과정을 모두 마친 후 더 깊은 공부를 원할 때 추천합니다.
