## 8.1 비캡처 그룹: `(?:...)`

> 💡 **한 줄 요약**: 비캡처 그룹은 "그룹핑은 하되, 캡처는 하지 않는" 소괄호입니다 — 패턴 구조를 잡되, `findall()` 결과를 오염시키지 않습니다.

### 직관적 이해

사진을 찍을 때를 생각해 봅시다. 여러 사람이 함께 서 있는데, 사진을 찍으면 모든 사람이 프레임에 담깁니다. 그런데 가끔은 사람들을 한 팀으로 **묶어서 배치**하고 싶지만, 그 팀 사진을 별도로 **보관할 필요는 없는** 경우가 있습니다. "그냥 자리 배치를 위해 묶었을 뿐이야."

정규표현식에서도 마찬가지입니다. Chapter 6에서 우리는 소괄호 `()`가 두 가지 역할을 동시에 한다는 것을 배웠습니다:

1. **그룹핑**: 패턴을 하나의 단위로 묶기 (수량자 적용, 선택 범위 지정 등)
2. **캡처**: 매칭된 내용을 별도로 저장하여 나중에 꺼내 쓸 수 있게 하기

> 🔗 **연결**: Chapter 6에서 배운 캡처 그룹 `()`의 두 가지 역할 — 그룹핑과 캡처 — 을 떠올려 봅시다.

그런데 그룹핑만 필요하고 캡처는 필요 없는 상황이 꽤 자주 있습니다. 이때 일반 캡처 그룹을 사용하면 불필요한 캡처가 발생하여 `findall()` 결과가 예상과 달라지는 문제가 생깁니다.

**비캡처 그룹** `(?:...)`은 이 문제를 깔끔하게 해결합니다: 그룹핑 기능은 유지하면서 캡처 기능만 꺼버리는 것입니다.

### 핵심 개념 설명

> 📝 **비캡처 그룹 (Non-capturing Group)**: `(?:...)`로 표기하며, 소괄호의 그룹핑 기능(패턴 묶기)은 수행하되, 매칭 결과를 캡처하지 않는 그룹입니다. 그룹 번호가 부여되지 않으며, `group(n)`이나 `groups()`에 포함되지 않습니다.

**문법**: 일반 소괄호 `(` 바로 뒤에 `?:`를 붙입니다.

```
캡처 그룹:     (pattern)
비캡처 그룹:   (?:pattern)
```

**비캡처 그룹이 필요한 두 가지 핵심 상황**:

**상황 1: 선택(`|`)의 범위를 제한하고 싶을 때**

Chapter 2에서 배운 선택 연산자 `|`는 그 양쪽 전체를 대상으로 합니다. 패턴의 일부에만 선택을 적용하려면 그룹으로 묶어야 하는데, 캡처가 필요 없는 경우가 대부분입니다.

```python
import re

# 목표: "http" 또는 "https"로 시작하는 패턴
text = "http://example.com and https://secure.com"

# 캡처 그룹을 사용하면...
result = re.findall(r'(https?)://\w+\.\w+', text)
print(result)  # ['http', 'https'] — 전체 URL이 아닌 캡처된 부분만!

# 비캡처 그룹을 사용하면...
result = re.findall(r'(?:https?)://\w+\.\w+', text)
print(result)  # ['http://example.com', 'https://secure.com'] — 전체 매칭!
```

**상황 2: 수량자를 그룹 단위로 적용하고 싶을 때**

```python
# 목표: "ab"가 2~4번 반복되는 패턴 찾기
text = "ab abab ababab abababab"

# 비캡처 그룹으로 반복 단위 지정
result = re.findall(r'(?:ab){2,4}', text)
print(result)  # ['abab', 'ababab', 'abababab']
```

> 🔗 **연결**: Chapter 4에서 배운 수량자 `{n,m}`을 그룹 단위에 적용하는 패턴입니다.

**캡처 그룹과 비캡처 그룹의 핵심 차이 — `findall()` 동작**

이것이 비캡처 그룹을 이해하는 가장 중요한 포인트입니다:

> 🔗 **연결**: Chapter 6.4에서 배운 핵심 규칙을 복습합니다 — `findall()`은 캡처 그룹이 있으면 **전체 매칭이 아닌 캡처된 부분만** 반환합니다.

```python
text = "2024-01-15, 2024-02-20, 2024-12-25"

# 캡처 그룹 사용: findall()이 캡처된 부분만 반환
result1 = re.findall(r'(\d{4})-(\d{2})-(\d{2})', text)
print(result1)  # [('2024', '01', '15'), ('2024', '02', '20'), ('2024', '12', '25')]

# 비캡처 그룹 사용: findall()이 전체 매칭을 반환
result2 = re.findall(r'(?:\d{4})-(?:\d{2})-(?:\d{2})', text)
print(result2)  # ['2024-01-15', '2024-02-20', '2024-12-25']

# 사실 이 경우 그룹이 아예 필요 없음 — 그룹 없는 버전
result3 = re.findall(r'\d{4}-\d{2}-\d{2}', text)
print(result3)  # ['2024-01-15', '2024-02-20', '2024-12-25']
```

위 예시의 세 번째 경우처럼, 그룹핑 자체가 불필요하다면 비캡처 그룹도 쓸 필요가 없습니다. 비캡처 그룹은 **그룹핑은 꼭 필요하지만 캡처는 원치 않을 때** 사용합니다.

**그룹 번호에 미치는 영향**

비캡처 그룹은 그룹 번호를 소비하지 않습니다. 이것은 캡처 그룹과 비캡처 그룹을 섞어 쓸 때 중요합니다:

```python
# 캡처와 비캡처 혼합
pattern = r'(?:https?)://([\w.]+)/(\w+)'
#          ^^^^^^^^^^       ^^^^^^^^^  ^^^^^
#          비캡처(번호 없음) 그룹 1     그룹 2

m = re.search(pattern, 'https://www.example.com/products')
if m:
    print(m.group(0))  # 'https://www.example.com/products' (전체 매칭)
    print(m.group(1))  # 'www.example.com'  (첫 번째 캡처 그룹)
    print(m.group(2))  # 'products'         (두 번째 캡처 그룹)
    print(m.groups())  # ('www.example.com', 'products')
```

`(?:https?)`는 그룹 번호를 차지하지 않으므로, `([\w.]+)`이 그룹 1, `(\w+)`이 그룹 2가 됩니다.

### 상세 예시

> 🔍 **예시 8.1.1: 선택 범위 제한 — 파일 확장자 매칭**
>
> **상황**: 텍스트에서 Python 또는 JavaScript 파일명을 모두 찾고 싶습니다. 파일 확장자가 `.py` 또는 `.js`인 파일명을 추출해야 합니다.
>
> **풀이**:
> ```python
> import re
> text = "main.py utils.js config.yaml test.py app.js"
>
> # 비캡처 그룹으로 확장자 선택
> result = re.findall(r'\w+\.(?:py|js)', text)
> print(result)  # ['main.py', 'utils.js', 'test.py', 'app.js']
>
> # 만약 캡처 그룹을 썼다면?
> result_bad = re.findall(r'\w+\.(py|js)', text)
> print(result_bad)  # ['py', 'js', 'py', 'js'] — 확장자만 캡처됨!
> ```
>
> **핵심 포인트**: `|`의 선택 범위를 `py`와 `js`로 제한하기 위해 그룹핑이 필요하지만, 전체 파일명을 결과로 원하므로 비캡처 그룹을 사용합니다.

> 🔍 **예시 8.1.2: 반복 단위 지정 — IP 주소 패턴**
>
> **상황**: IPv4 주소처럼 보이는 패턴(숫자.숫자.숫자.숫자)을 찾고 싶습니다.
>
> **풀이**:
> ```python
> text = "서버 IP: 192.168.1.100, 게이트웨이: 10.0.0.1"
>
> # "숫자+."가 3번 반복된 후 숫자+
> result = re.findall(r'(?:\d{1,3}\.){3}\d{1,3}', text)
> print(result)  # ['192.168.1.100', '10.0.0.1']
> ```
>
> **핵심 포인트**: `\d{1,3}\.`(숫자 1~3개와 마침표)를 하나의 단위로 묶어 `{3}`번 반복합니다. 캡처는 불필요하므로 `(?:...)`를 사용합니다. 캡처 그룹을 사용하면 `findall()`이 마지막 반복의 캡처 결과만 반환하여 원하지 않는 결과가 나옵니다.

### 흔한 실수와 주의사항

> ⚠️ **주의: `findall()`에서 캡처 그룹의 함정**
>
> 가장 흔한 실수는 그룹핑 목적으로 `()`를 사용했다가 `findall()` 결과가 예상과 달라지는 것입니다.
>
> ```python
> # 의도: "Mr." 또는 "Mrs."를 포함하는 전체 이름 찾기
> text = "Mr. Kim, Mrs. Lee, Ms. Park"
>
> # ❌ 잘못된 접근: 캡처 그룹 사용
> result = re.findall(r'(Mr|Mrs)\. \w+', text)
> print(result)  # ['Mr', 'Mrs'] — 호칭만 나옴!
>
> # ✅ 올바른 접근: 비캡처 그룹 사용
> result = re.findall(r'(?:Mr|Mrs)\. \w+', text)
> print(result)  # ['Mr. Kim', 'Mrs. Lee'] — 전체 이름!
> ```

> ⚠️ **주의: 비캡처 그룹의 `?:` 오타**
>
> `(?:...)`에서 `?:`를 빼먹으면 일반 캡처 그룹 `(...)`이 됩니다. `?`와 `:` 사이에 공백이 있어도 안 됩니다. 또한 `?`만 쓰고 `:`를 빼먹으면 수량자 `?`(0 또는 1회)가 되어 완전히 다른 의미가 됩니다.
>
> ```python
> (?:abc)   # ✅ 비캡처 그룹
> (abc)     # ❌ 캡처 그룹 (의도와 다른 결과)
> (? :abc)  # ❌ 문법 오류
> (?abc)    # ❌ 문법 오류 (파이썬에서 유효하지 않은 확장 표기)
> ```

### Section 요약

- **비캡처 그룹 `(?:...)`**은 패턴을 묶는 그룹핑 기능만 수행하고, 매칭 결과를 캡처하지 않습니다.
- 비캡처 그룹은 **그룹 번호를 소비하지 않으므로**, 다른 캡처 그룹의 번호에 영향을 주지 않습니다.
- `findall()`을 사용할 때 전체 매칭 결과를 원한다면, 그룹핑이 필요한 곳에 비캡처 그룹을 사용하세요.
- 그룹핑 자체가 불필요한 상황에서는 비캡처 그룹도 쓸 필요가 없습니다 — 꼭 필요할 때만 사용합니다.

## 8.2 명명 그룹: `(?P<name>...)`

> 💡 **한 줄 요약**: 명명 그룹은 캡처 그룹에 번호 대신 **이름표**를 붙이는 것입니다 — 복잡한 패턴의 가독성이 극적으로 향상됩니다.

### 직관적 이해

회사에서 사물함을 사용한다고 생각해 봅시다. 사물함에 1번, 2번, 3번... 이렇게 번호만 붙어 있으면, "내 물건이 몇 번 사물함이더라?" 하고 매번 기억해야 합니다. 하지만 사물함에 "김철수의 함"이라고 이름표가 붙어 있으면 훨씬 직관적이겠죠?

Chapter 6에서 우리는 캡처 그룹을 번호로 참조하는 방법을 배웠습니다: `group(1)`, `group(2)`, `group(3)`... 그런데 패턴이 복잡해져서 캡처 그룹이 5개, 10개가 되면, "group(3)이 뭐였더라?" 하고 헷갈리기 시작합니다. 패턴 중간에 그룹을 하나 추가하면 뒤의 모든 번호가 밀려나는 문제도 있습니다.

**명명 그룹**은 이 문제를 해결합니다. 그룹에 번호 대신 의미 있는 **이름**을 부여하여, `group('year')`, `group('month')`, `group('day')`처럼 이름으로 결과를 참조할 수 있게 합니다.

### 핵심 개념 설명

> 📝 **명명 그룹 (Named Group)**: `(?P<name>...)`으로 표기하며, 캡처 그룹에 고유한 이름을 부여하는 파이썬 정규표현식 확장 문법입니다. 번호(`group(n)`)와 이름(`group('name')`) 모두로 결과를 참조할 수 있습니다.

**문법**:

```
(?P<이름>패턴)
```

- `(?P<`로 시작하고 `>...)`으로 끝납니다.
- `이름`은 유효한 파이썬 식별자여야 합니다 (영문자 또는 `_`로 시작, 영문자/숫자/`_`로 구성).
- 같은 패턴 안에서 이름은 고유해야 합니다 (중복 불가).

> 💬 **참고**: `(?P<...>)` 문법에서 `P`는 Python에서 유래한 것으로, 다른 정규표현식 엔진에서는 `(?<name>...)` 또는 `(?'name'...)` 문법을 사용하기도 합니다. 파이썬의 `re` 모듈에서는 반드시 `P`를 포함해야 합니다.

**기본 사용법**:

```python
import re

# 날짜 패턴에 이름 부여
pattern = r'(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})'
m = re.search(pattern, '오늘은 2024-01-15입니다')

if m:
    # 이름으로 참조
    print(m.group('year'))    # '2024'
    print(m.group('month'))   # '01'
    print(m.group('day'))     # '15'
    
    # 번호로도 여전히 참조 가능
    print(m.group(1))         # '2024'
    print(m.group(2))         # '01'
    print(m.group(3))         # '15'
```

**`groupdict()` — 명명 그룹 결과를 딕셔너리로**

명명 그룹의 가장 강력한 기능 중 하나는 `groupdict()` 메서드입니다. 모든 명명 그룹의 결과를 `{이름: 매칭값}` 형태의 딕셔너리로 반환합니다.

> 📝 **`groupdict()`**: 매치 객체의 모든 명명 그룹을 `{그룹이름: 매칭문자열}` 딕셔너리로 반환하는 메서드입니다. 명명 그룹이 아닌 일반 캡처 그룹은 포함되지 않습니다.

```python
pattern = r'(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})'
m = re.search(pattern, '2024-01-15')

if m:
    date_dict = m.groupdict()
    print(date_dict)  # {'year': '2024', 'month': '01', 'day': '15'}
    
    # 딕셔너리이므로 바로 활용 가능
    print(f"{date_dict['year']}년 {date_dict['month']}월 {date_dict['day']}일")
    # 출력: 2024년 01월 15일
```

**명명 그룹과 `re.sub()` — 이름 기반 역참조**

> 🔗 **연결**: Chapter 7에서 배운 `re.sub()`의 역참조 치환(`\1`, `\g<1>`)을 떠올려 봅시다.

Chapter 7에서 치환 문자열에서 `\1`이나 `\g<1>`로 캡처 그룹을 참조하는 방법을 배웠습니다. 명명 그룹은 `\g<name>`으로도 참조할 수 있습니다:

```python
# 날짜 형식 변환: YYYY-MM-DD → DD/MM/YYYY
text = '2024-01-15, 2024-12-25'
pattern = r'(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})'

result = re.sub(pattern, r'\g<day>/\g<month>/\g<year>', text)
print(result)  # '15/01/2024, 25/12/2024'
```

**`finditer()`와 명명 그룹의 조합**

여러 매칭 결과를 구조화된 데이터로 변환할 때, `finditer()`와 명명 그룹의 조합이 매우 유용합니다:

```python
text = "김철수(30세), 이영희(25세), 박지민(35세)"
pattern = r'(?P<name>[가-힣]+)\((?P<age>\d+)세\)'

people = []
for m in re.finditer(pattern, text):
    people.append(m.groupdict())

print(people)
# [{'name': '김철수', 'age': '30'}, {'name': '이영희', 'age': '25'}, {'name': '박지민', 'age': '35'}]
```

> 💬 **참고**: 위 예시에서 `[가-힣]`은 한글 음절 범위를 나타내는 문자 클래스입니다. 이에 대한 자세한 내용은 Chapter 11에서 다룹니다. 지금은 "한글 한 글자를 매칭한다"는 정도로 이해하면 됩니다.

### 상세 예시

> 🔍 **예시 8.2.1: 로그 데이터 파싱**
>
> **상황**: 웹 서버 로그에서 IP 주소, 날짜, HTTP 메서드, 경로를 추출하고 싶습니다.
>
> **풀이**:
> ```python
> log = '192.168.1.1 - [2024-01-15] "GET /index.html"'
>
> pattern = (
>     r'(?P<ip>[\d.]+)'        # IP 주소
>     r' - '
>     r'\[(?P<date>[\d-]+)\]'  # 날짜
>     r' "'
>     r'(?P<method>\w+)'       # HTTP 메서드
>     r' '
>     r'(?P<path>\S+)'         # 경로
>     r'"'
> )
>
> m = re.search(pattern, log)
> if m:
>     info = m.groupdict()
>     print(info)
>     # {'ip': '192.168.1.1', 'date': '2024-01-15',
>     #  'method': 'GET', 'path': '/index.html'}
> ```
>
> **핵심 포인트**: 명명 그룹을 사용하면 추출 결과가 `groupdict()`를 통해 바로 의미 있는 딕셔너리가 됩니다. `group(1)`, `group(2)` 같은 번호를 기억할 필요가 없습니다.
>
> 💬 **참고**: 파이썬에서 긴 문자열은 괄호 안에서 여러 줄의 문자열 리터럴을 나열하면 자동으로 연결(concatenation)됩니다. 이 기법은 복잡한 패턴의 가독성을 높이는 데 매우 유용합니다.

> 🔍 **예시 8.2.2: 명명 그룹과 비캡처 그룹 혼합 사용**
>
> **상황**: 전화번호에서 지역번호와 나머지 번호를 추출하되, 구분자(하이픈 또는 마침표)는 캡처하지 않으려 합니다.
>
> **풀이**:
> ```python
> text = "연락처: 02-1234-5678, 010.9876.5432"
>
> pattern = r'(?P<area>\d{2,3})(?:[-.])'  \
>           r'(?P<middle>\d{3,4})(?:[-.])'  \
>           r'(?P<last>\d{4})'
>
> for m in re.finditer(pattern, text):
>     print(m.groupdict())
> # {'area': '02', 'middle': '1234', 'last': '5678'}
> # {'area': '010', 'middle': '9876', 'last': '5432'}
> ```
>
> **핵심 포인트**: 캡처가 필요한 부분(지역번호, 중간, 끝)은 명명 그룹으로, 구분자처럼 캡처가 불필요한 그룹핑은 비캡처 그룹 `(?:...)`으로 처리합니다.

### 흔한 실수와 주의사항

> ⚠️ **주의: 명명 그룹의 문법 실수**
>
> 명명 그룹의 문법은 꽤 길기 때문에 오타가 나기 쉽습니다:
>
> ```python
> (?P<name>\d+)    # ✅ 올바름
> (?P<name >\d+)   # ❌ 이름과 > 사이에 공백
> (?p<name>\d+)    # ❌ P가 소문자
> (?P<123name>\d+) # ❌ 이름이 숫자로 시작
> (?P<name>\d+) ... (?P<name>\w+)  # ❌ 같은 이름을 두 번 사용
> ```

> ⚠️ **주의: `groupdict()`는 명명 그룹만 포함**
>
> `groupdict()`는 명명 그룹의 결과만 반환합니다. 일반 캡처 그룹(번호만 있는 그룹)은 `groupdict()`에 포함되지 않습니다. 모든 그룹의 결과가 필요하다면 `groups()`를 사용하세요.
>
> ```python
> pattern = r'(?P<name>[a-z]+)-(\d+)'
> m = re.search(pattern, 'hello-42')
> if m:
>     print(m.groupdict())  # {'name': 'hello'} — 번호 그룹은 빠짐
>     print(m.groups())     # ('hello', '42') — 모든 그룹 포함
> ```

### Section 요약

- **명명 그룹 `(?P<name>...)`**은 캡처 그룹에 의미 있는 이름을 부여합니다.
- `group('name')`으로 이름으로 결과를 참조하고, `groupdict()`로 딕셔너리 형태로 추출할 수 있습니다.
- `re.sub()` 치환 문자열에서 `\g<name>`으로 명명 그룹을 역참조할 수 있습니다.
- `finditer()`와 `groupdict()`를 결합하면 텍스트에서 구조화된 데이터를 쉽게 추출할 수 있습니다.
- 명명 그룹은 번호 기반 참조도 여전히 지원하므로, 기존 코드와의 호환성이 유지됩니다.

## 8.3 패턴 내 역참조: `\1`과 `(?P=name)`

> 💡 **한 줄 요약**: 패턴 내 역참조는 "아까 캡처한 **바로 그 문자열**이 다시 나타나는 곳"을 찾는 기법입니다.

### 직관적 이해

친구와 "말 따라하기" 게임을 한다고 생각해 봅시다. 친구가 "사과"라고 말하면, 여러분도 정확히 "사과"라고 따라해야 합니다. "과일"이라고 하면 안 되고, "배"라고 해도 안 됩니다 — 정확히 같은 단어를 반복해야 합니다.

정규표현식에서의 역참조가 바로 이것입니다. 패턴의 앞부분에서 캡처된 **그 정확한 문자열**이 뒷부분에서 **다시 등장**하는 것을 요구합니다.

> 🔗 **연결**: Chapter 7에서 배운 역참조는 *치환 문자열* 안에서 `\1`이나 `\g<1>`을 사용하여 캡처된 내용을 치환 결과에 삽입하는 것이었습니다. 이번에 배우는 역참조는 *패턴 자체* 안에서 이전 캡처 내용과 동일한 문자열을 매칭하는 것입니다. 같은 `\1` 기호지만, 사용되는 맥락이 다릅니다.

이것은 단순히 "같은 종류의 문자"가 아니라 "정확히 같은 문자열"을 의미한다는 점이 핵심입니다:

- `(.)\1` → 첫 글자가 `a`로 캡처되면 `\1`도 반드시 `a`여야 합니다 → `aa`, `bb`, `cc` 등 매칭
- `(.)\1` ≠ `.{2}` → `.{2}`는 아무 두 글자(예: `ab`, `xy`)를 매칭하지만, `(.)\1`은 **같은** 글자의 반복만 매칭

### 핵심 개념 설명

> 📝 **패턴 내 역참조 (Backreference in Pattern)**: 패턴 안에서 이전에 캡처된 그룹의 **실제 매칭 결과**를 다시 매칭하도록 요구하는 표현입니다. `\n`(n은 그룹 번호) 또는 `(?P=name)`(명명 그룹의 이름)으로 표기합니다.

**번호 기반 역참조: `\1`, `\2`, ...**

```python
import re

# 같은 문자가 연속으로 반복되는 곳 찾기
pattern = r'(.)\1'
text = "aabbcc hello mississippi"

result = re.findall(r'(.)\1', text)
print(result)  # ['a', 'b', 'c', 'l', 's', 's', 'p']
# 주의: findall은 캡처 그룹의 내용만 반환하므로 반복된 한 글자만 나옴

# finditer로 전체 매칭 확인
for m in re.finditer(r'(.)\1', text):
    print(m.group(0), end=' ')  # aa bb cc ll ss ss pp
```

**이름 기반 역참조: `(?P=name)`**

명명 그룹을 사용할 때는 `(?P=name)` 문법으로 역참조합니다:

```python
# 같은 단어가 연속으로 반복되는 곳 찾기
pattern = r'(?P<word>\b\w+\b)\s+(?P=word)'
text = "the the quick brown fox fox jumped"

for m in re.finditer(pattern, text):
    print(f"중복 단어: '{m.group('word')}' at position {m.start()}")
# 중복 단어: 'the' at position 0
# 중복 단어: 'fox' at position 20
```

**역참조의 동작 원리를 좀 더 자세히 살펴봅시다:**

```python
# 패턴: (\w+)\s+\1
# 입력: "hello hello world"

# 매칭 과정:
# 1. (\w+)가 "hello"를 캡처 → 그룹 1 = "hello"
# 2. \s+가 공백 " "을 매칭
# 3. \1이 그룹 1의 값인 "hello"를 찾으려 함
# 4. 다음 위치에 "hello"가 있으므로 매칭 성공!
# → 전체 매칭: "hello hello"
```

```python
# 같은 패턴으로 매칭 실패 사례:
# 패턴: (\w+)\s+\1
# 입력: "hello world"

# 매칭 과정:
# 1. (\w+)가 "hello"를 캡처 → 그룹 1 = "hello"
# 2. \s+가 공백 " "을 매칭
# 3. \1이 "hello"를 찾으려 하지만, 다음은 "world"
# 4. 매칭 실패 → 엔진이 다른 시작점으로 이동
```

**역참조의 실용적 활용: 따옴표 쌍 매칭**

역참조의 가장 대표적인 실용 사례는 여는 구분자와 닫는 구분자가 같아야 하는 패턴입니다:

```python
# 작은따옴표 또는 큰따옴표로 감싸진 문자열 찾기
# 여는 따옴표와 닫는 따옴표가 같아야 함

pattern = r'''(["']).*?\1'''
#            ^^^^^^      ^^
#         따옴표 캡처  같은 따옴표

text = '''She said "hello" and 'goodbye' but "not this''''

for m in re.finditer(pattern, text):
    print(m.group(0))
# "hello"
# 'goodbye'
# "not this' 는 따옴표 종류가 달라서 매칭 안 됨
```

> 🔗 **연결**: Chapter 4에서 배운 비탐욕적 수량자 `.*?`를 사용하여 따옴표 사이의 최소한의 내용을 매칭합니다. `.*`(탐욕적)을 사용하면 첫 여는 따옴표부터 마지막 닫는 따옴표까지 과도하게 매칭됩니다.

### 상세 예시

> 🔍 **예시 8.3.1: 중복 단어 탐지**
>
> **상황**: 영어 문서에서 실수로 같은 단어를 연속으로 두 번 쓴 곳을 찾아 수정하고 싶습니다.
>
> **풀이**:
> ```python
> text = "This is is a test test document for for checking"
>
> # 중복 단어 찾기
> pattern = r'\b(\w+)\s+\1\b'
>
> # 찾기
> for m in re.finditer(pattern, text):
>     print(f"중복 발견: '{m.group(0)}'")
> # 중복 발견: 'is is'
> # 중복 발견: 'test test'
> # 중복 발견: 'for for'
>
> # 중복 단어를 하나로 치환
> result = re.sub(r'\b(\w+)\s+\1\b', r'\1', text)
> print(result)  # "This is a test document for checking"
> ```
>
> **핵심 포인트**: `\b`로 단어 경계를 지정하여 "the"가 "there"의 일부로 매칭되는 것을 방지합니다. `\1`은 `(\w+)`가 캡처한 정확한 단어를 다시 매칭합니다.

> 🔍 **예시 8.3.2: HTML/XML 태그 쌍 매칭**
>
> **상황**: 간단한 HTML에서 여는 태그와 닫는 태그가 일치하는 요소를 찾고 싶습니다.
>
> **풀이**:
> ```python
> text = "<b>bold</b> <i>italic</i> <b>mixed</i>"
>
> # 여는 태그와 닫는 태그가 같은 요소 찾기
> pattern = r'<(?P<tag>\w+)>.*?</(?P=tag)>'
>
> for m in re.finditer(pattern, text):
>     print(f"태그: {m.group('tag')}, 내용: {m.group(0)}")
> # 태그: b, 내용: <b>bold</b>
> # 태그: i, 내용: <i>italic</i>
> # <b>mixed</i>는 태그가 다르므로 매칭 안 됨
> ```
>
> **핵심 포인트**: `(?P<tag>\w+)`로 여는 태그 이름을 캡처하고, `(?P=tag)`로 닫는 태그가 같은 이름인지 확인합니다.
>
> 💬 **참고**: 실제 HTML/XML 파싱에는 정규표현식보다 전용 파서(예: `BeautifulSoup`, `lxml`)가 적합합니다. 이 예시는 역참조의 동작을 보여주기 위한 것이며, 실무에서 복잡한 HTML을 정규표현식으로 파싱하는 것은 권장되지 않습니다.

### 깊이 있는 이해 (Deep Dive)

> 🔬 **Deep Dive: 역참조와 백트래킹의 관계**
>
> *이 부분은 심화 내용입니다. 처음 학습 시 건너뛰어도 됩니다.*
>
> 역참조가 있는 패턴에서 매칭 엔진의 동작을 살펴보면 흥미로운 점이 있습니다. `(\w+)\s+\1`로 "abc abc xyz"를 매칭할 때:
>
> 1. `(\w+)`는 탐욕적이므로 처음에 "abc"를 캡처합니다 (실제로는 먼저 가능한 한 많이 매칭하려 하지만, 공백 등의 제약으로 적절히 조정됩니다).
> 2. `\s+`가 공백을 매칭합니다.
> 3. `\1`이 캡처된 "abc"를 찾습니다. 다음 위치에 "abc"가 있으므로 성공!
>
> 그런데 만약 `(\w+)\s+\1`로 "abcx abc xyz"를 매칭하면 어떻게 될까요?
>
> - `(\w+)`가 "abcx"를 캡처 → `\s+`가 공백 매칭 → `\1`이 "abcx"를 찾지만 "abc"가 있어서 실패
> - 엔진이 백트래킹하여 `(\w+)`가 "abc"만 캡처하도록 시도 → 하지만 "x "가 `\s+`와 매칭 안 됨
> - 최종적으로 이 위치에서는 매칭 실패
>
> 이처럼 역참조는 백트래킹의 양을 증가시킬 수 있으므로, 성능에 민감한 상황에서는 주의가 필요합니다. 이에 대해서는 Chapter 15와 16에서 자세히 다룹니다.

### 흔한 실수와 주의사항

> ⚠️ **주의: 역참조 `\1`과 문자 클래스의 혼동**
>
> `\1`은 "그룹 1이 캡처한 정확한 문자열"을 의미합니다. 이것은 `\d`(숫자 한 개)나 `\w`(단어 문자 한 개) 같은 문자 클래스와 완전히 다릅니다. 역참조는 **동적**입니다 — 같은 패턴이라도 입력에 따라 `\1`이 가리키는 실제 문자열이 달라집니다.
>
> ```python
> # (\d)\1 — "같은 숫자의 반복"
> re.findall(r'(\d)\1', '1123345567')
> # ['1', '3', '5'] → 11, 33, 55를 찾음 (22, 44 등은 없음)
>
> # \d\d — "아무 숫자 두 개" (역참조 아님)
> re.findall(r'\d\d', '1123345567')
> # ['11', '23', '34', '55', '67'] → 모든 연속 숫자 쌍
> ```

> ⚠️ **주의: raw string에서의 역참조**
>
> 패턴 내 역참조는 반드시 raw string을 사용해야 합니다. 일반 문자열에서 `\1`은 파이썬 문자열 이스케이프로 해석될 수 있습니다.
>
> ```python
> # ✅ raw string 사용
> pattern = r'(\w+)\s+\1'
>
> # ❌ 일반 문자열 — \1이 문자열 이스케이프로 해석될 수 있음
> pattern = '(\\w+)\\s+\\1'  # 동작은 하지만 가독성이 떨어짐
> ```

### Section 요약

- **패턴 내 역참조**는 이전에 캡처한 **정확한 문자열**이 다시 나타나는 것을 매칭합니다.
- `\1`, `\2` 등은 번호 기반 역참조, `(?P=name)`은 이름 기반 역참조입니다.
- 대표적인 활용: 중복 단어 탐지, 따옴표 쌍 매칭, 태그 쌍 매칭.
- 역참조는 "같은 종류"가 아닌 "정확히 같은 값"을 요구한다는 점이 핵심입니다.
- 치환 문자열에서의 `\1`(Chapter 7)과 패턴 내의 `\1`(이번 Section)은 사용 맥락이 다르지만, 둘 다 캡처 그룹의 내용을 참조한다는 공통점이 있습니다.

## 8.4 조건 패턴: `(?(id)yes|no)`

> 💡 **한 줄 요약**: 조건 패턴은 "이전 그룹이 매칭에 성공했는가?"에 따라 패턴을 분기하는 "if-else"입니다.

### 직관적 이해

비가 올 때 우산을 가져가고, 비가 오지 않으면 가져가지 않는 것처럼 — 조건에 따라 행동이 달라지는 상황은 일상에서 흔합니다. 정규표현식에서도 비슷한 상황이 있습니다: "앞에서 여는 괄호가 있었다면, 닫는 괄호도 있어야 한다. 여는 괄호가 없었다면, 닫는 괄호도 없어야 한다."

조건 패턴은 정규표현식에서 이런 **조건 분기**를 가능하게 합니다. 특정 캡처 그룹이 매칭에 참여했는지 여부에 따라 이후 패턴을 달리 적용합니다.

### 핵심 개념 설명

> 📝 **조건 패턴 (Conditional Pattern)**: `(?(id)yes-pattern|no-pattern)` 형태로, 지정한 캡처 그룹(id)이 매칭에 성공했으면 `yes-pattern`을, 그렇지 않으면 `no-pattern`을 적용하는 표현입니다. `|no-pattern` 부분은 생략 가능합니다.

**문법**:

```
(?(그룹번호)매칭시패턴|비매칭시패턴)
(?(그룹이름)매칭시패턴|비매칭시패턴)
(?(그룹번호)매칭시패턴)              — no-pattern 생략
```

- `id`에는 그룹 번호(1, 2, 3...) 또는 명명 그룹 이름을 넣습니다.
- 해당 그룹이 매칭에 참여했으면 `yes-pattern`이 적용되고, 참여하지 않았으면 `no-pattern`이 적용됩니다.
- `no-pattern`을 생략하면, 그룹이 매칭되지 않았을 때는 아무 조건 없이 통과합니다.

**기본 예시: 괄호 쌍 매칭**

가장 대표적인 사용 사례입니다. 숫자 앞에 여는 괄호가 있으면 닫는 괄호도 있어야 하고, 없으면 없어야 합니다:

```python
import re

# 숫자 앞에 (가 있으면 뒤에 )가 와야 함
# 숫자 앞에 (가 없으면 뒤에 )가 오면 안 됨
pattern = r'(\()?\d+(?(1)\))'
#          ^^^^^              — 그룹 1: 선택적 여는 괄호
#                ^^^          — 숫자 1개 이상
#                   ^^^^^^^^^  — 그룹 1이 매칭됐으면 닫는 괄호 필요

tests = ['123', '(456)', '(789', '012)']
for t in tests:
    m = re.fullmatch(pattern, t)
    print(f"'{t}': {'매칭' if m else '불일치'}")
# '123': 매칭       — 괄호 없이 숫자만: OK
# '(456)': 매칭     — 괄호 쌍 완전: OK
# '(789': 불일치    — 여는 괄호만: FAIL
# '012)': 불일치    — 닫는 괄호만: FAIL
```

동작을 단계별로 살펴보겠습니다:

- `(\()?`: 여는 괄호 `(`를 선택적으로(`?`) 캡처합니다. `?`가 붙어 있으므로, 괄호가 없어도 매칭은 성공합니다 (그룹이 빈 매칭).
  - 괄호가 있으면: 그룹 1이 `(`를 캡처 → 매칭에 "참여"함
  - 괄호가 없으면: 그룹 1이 매칭 시도를 건너뜀 → 매칭에 "참여하지 않음"
- `\d+`: 숫자를 매칭합니다.
- `(?(1)\))`: 그룹 1이 매칭에 참여했으면(`(` 가 있었으면) `\)`(닫는 괄호)를 요구합니다. 참여하지 않았으면 아무것도 요구하지 않습니다.

**명명 그룹과 함께 사용**

```python
# 더 가독성 높은 버전
pattern = r'(?P<open>\()?\d+(?(open)\))'

m = re.fullmatch(pattern, '(42)')
print(m is not None)  # True

m = re.fullmatch(pattern, '42')
print(m is not None)  # True

m = re.fullmatch(pattern, '(42')
print(m is not None)  # False
```

### 상세 예시

> 🔍 **예시 8.4.1: 선택적 접두사-접미사 쌍 매칭**
>
> **상황**: 문자열이 `<`로 시작하면 `>`로 끝나야 하고, `<` 없이 시작하면 `>`도 없어야 하는 패턴을 만들려 합니다.
>
> **풀이**:
> ```python
> pattern = r'(?P<bracket><)?[\w\s]+(?(bracket)>)'
>
> tests = ['hello', '<hello>', '<hello', 'hello>']
> for t in tests:
>     m = re.fullmatch(pattern, t)
>     print(f"'{t}': {'✅ 매칭' if m else '❌ 불일치'}")
> # 'hello': ✅ 매칭
> # '<hello>': ✅ 매칭
> # '<hello': ❌ 불일치
> # 'hello>': ❌ 불일치
> ```
>
> **핵심 포인트**: `(?P<bracket><)?`로 여는 꺾쇠를 선택적으로 캡처한 후, `(?(bracket)>)`로 "여는 꺾쇠가 있었다면 닫는 꺾쇠도 요구"합니다.

> 🔍 **예시 8.4.2: yes-pattern과 no-pattern 모두 사용**
>
> **상황**: 전화번호가 하이픈으로 구분되어 있으면 하이픈 형식(010-1234-5678)이어야 하고, 하이픈이 없으면 연속 숫자(01012345678)여야 합니다.
>
> **풀이**:
> ```python
> # 첫 하이픈이 있으면 두 번째 하이픈도 있어야, 없으면 숫자만 이어져야
> pattern = r'\d{3}(?P<sep>-)?\d{4}(?(sep)-|\d*)\d{4}'
>
> tests = ['010-1234-5678', '01012345678', '010-12345678', '0101234-5678']
> for t in tests:
>     m = re.fullmatch(pattern, t)
>     print(f"'{t}': {'✅' if m else '❌'}")
> # '010-1234-5678': ✅
> # '01012345678': ✅
> # '010-12345678': ❌  (첫 하이픈 있으면 두 번째도 필요)
> # '0101234-5678': ❌  (첫 하이픈 없으면 두 번째도 없어야)
> ```
>
> **핵심 포인트**: `(?(sep)-|\d*)`에서 `sep` 그룹이 매칭됐으면(`-`가 있었으면) `-`를, 매칭 안 됐으면 `\d*`를 적용합니다.

### 흔한 실수와 주의사항

> ⚠️ **주의: 조건 패턴은 고급 기능이며 주의가 필요합니다**
>
> 조건 패턴은 정규표현식 중에서도 고급 기능입니다. 다음 점에 유의하세요:
>
> 1. **과용하지 마세요**: 조건 패턴이 복잡해지면 패턴의 가독성이 급격히 떨어집니다. 두 개의 별도 패턴을 `|`로 연결하거나, 파이썬 코드로 조건을 처리하는 것이 더 명확할 수 있습니다.
> 2. **그룹 참여 여부의 판단**: `?` 수량자로 선택적인 그룹만 조건에 활용할 수 있습니다. 항상 매칭되는 그룹은 조건으로서의 의미가 없습니다 (항상 yes-pattern이 적용되므로).
> 3. **디버깅 어려움**: 조건 패턴이 포함된 정규표현식은 디버깅이 어렵습니다. regex101 같은 도구에서 단계별로 테스트하세요.

### Section 요약

- **조건 패턴 `(?(id)yes|no)`**은 특정 캡처 그룹의 매칭 참여 여부에 따라 패턴을 분기합니다.
- 괄호 쌍 매칭, 구분자 일관성 검증 등 "쌍을 이루는 요소"의 검증에 유용합니다.
- `no-pattern`은 생략할 수 있으며, 그룹 번호 또는 명명 그룹 이름으로 참조합니다.
- 복잡한 조건 패턴은 가독성을 크게 해치므로, 정말 필요한 경우에만 사용하는 것이 좋습니다.

## 8.5 고급 그룹핑 종합 패턴

> 💡 **한 줄 요약**: 이 Section에서는 비캡처 그룹, 명명 그룹, 역참조, 조건 패턴을 조합하여 실전 수준의 복합 패턴을 설계합니다.

### 직관적 이해

이제까지 배운 도구들을 각각의 도구함에 넣어두었다면, 이 Section에서는 여러 도구를 동시에 꺼내어 하나의 작업을 완성합니다. 마치 레고 블록을 개별적으로 익힌 후에 완성된 구조물을 만드는 것과 같습니다.

### 종합 예시

> 🔍 **예시 8.5.1: 날짜 파싱과 형식 변환**
>
> **상황**: 텍스트에서 다양한 형식의 날짜를 추출하고, 구조화된 딕셔너리로 변환하고 싶습니다. 지원하는 형식: `2024-01-15`, `2024/01/15`, `15.01.2024`
>
> **풀이**:
> ```python
> import re
>
> # 다양한 날짜 형식을 하나의 패턴으로 파싱
> # 형식 1: YYYY-MM-DD 또는 YYYY/MM/DD
> # 형식 2: DD.MM.YYYY
> pattern = r'''
>     (?:                                        # 비캡처 그룹: 전체 선택
>         (?P<year1>\d{4})(?:[-/])               # 형식1 연도 + 구분자(비캡처)
>         (?P<month1>\d{2})(?:[-/])              # 형식1 월 + 구분자(비캡처)
>         (?P<day1>\d{2})                        # 형식1 일
>     |                                          # 또는
>         (?P<day2>\d{2})\.                      # 형식2 일 + .
>         (?P<month2>\d{2})\.                    # 형식2 월 + .
>         (?P<year2>\d{4})                       # 형식2 연도
>     )
> '''
>
> texts = ['2024-01-15', '2024/03/20', '25.12.2024']
>
> for text in texts:
>     m = re.search(pattern, text, re.VERBOSE)
>     if m:
>         d = m.groupdict()
>         # None이 아닌 값 추출
>         year = d['year1'] or d['year2']
>         month = d['month1'] or d['month2']
>         day = d['day1'] or d['day2']
>         print(f"{text} → {year}년 {month}월 {day}일")
> # 2024-01-15 → 2024년 01월 15일
> # 2024/03/20 → 2024년 03월 20일
> # 25.12.2024 → 2024년 12월 25일
> ```
>
> **핵심 포인트**: 비캡처 그룹 `(?:...)`으로 선택(`|`)과 구분자를 묶고, 명명 그룹으로 연도/월/일에 이름을 붙여 구조화합니다. 구분자(`-`, `/`, `.`)는 캡처가 불필요하므로 비캡처 그룹을 사용합니다.
>
> 💬 **참고**: 위 예시에서 `re.VERBOSE` 플래그를 사용했습니다. 이 플래그는 패턴 안에 공백과 주석을 넣어 가독성을 높여줍니다. Chapter 10에서 자세히 다룰 예정이며, 지금은 "패턴에 공백과 `#` 주석을 허용하는 옵션"이라고만 이해하면 됩니다.

> 🔍 **예시 8.5.2: 역참조를 활용한 CSV 인용 필드 파싱**
>
> **상황**: CSV 데이터에서 필드를 추출합니다. 필드는 큰따옴표로 감싸져 있을 수도 있고, 감싸져 있지 않을 수도 있습니다. 큰따옴표가 있으면 내부에 쉼표가 허용됩니다.
>
> **풀이**:
> ```python
> csv_line = 'John,"Seoul, Korea",30,"Developer, Senior"'
>
> # 인용 필드 또는 비인용 필드 매칭
> pattern = r'(?:"(?P<quoted>[^"]*)")|(?P<unquoted>[^,]*)'
>
> fields = []
> for m in re.finditer(pattern, csv_line):
>     value = m.group('quoted') if m.group('quoted') is not None else m.group('unquoted')
>     if value is not None:
>         fields.append(value)
>
> print(fields)
> # ['John', 'Seoul, Korea', '30', 'Developer, Senior']
> ```
>
> **핵심 포인트**: 명명 그룹으로 인용/비인용 필드를 구분하여 추출합니다. 비캡처 그룹과 명명 그룹을 적절히 혼합하여 깔끔한 패턴을 구성합니다.

> 🔍 **예시 8.5.3: 키-값 쌍 추출**
>
> **상황**: 설정 파일에서 키-값 쌍을 추출합니다. 값은 따옴표로 감싸져 있을 수도, 없을 수도 있습니다.
>
> **풀이**:
> ```python
> config_text = '''
> name = "John Doe"
> age = 30
> city = 'Seoul'
> email = john@example.com
> '''
>
> # 키 = 값 패턴 (값은 따옴표 있거나 없거나)
> pattern = r'(?P<key>\w+)\s*=\s*(?:(?P<q>["\'])(?P<qval>.*?)(?P=q)|(?P<val>\S+))'
>
> config = {}
> for m in re.finditer(pattern, config_text):
>     key = m.group('key')
>     value = m.group('qval') if m.group('q') else m.group('val')
>     config[key] = value
>
> print(config)
> # {'name': 'John Doe', 'age': '30', 'city': 'Seoul', 'email': 'john@example.com'}
> ```
>
> **핵심 포인트**: 이 패턴은 이번 Chapter에서 배운 거의 모든 기법을 결합합니다:
> - `(?P<key>\w+)`: 명명 그룹으로 키 추출
> - `(?:...|...)`: 비캡처 그룹으로 인용/비인용 선택
> - `(?P<q>["\'])`: 여는 따옴표 캡처
> - `(?P=q)`: 역참조로 닫는 따옴표가 같은 종류인지 확인
> - `(?P<qval>.*?)`: 비탐욕적 수량자로 따옴표 내 값 추출

### 흔한 실수와 주의사항

> ⚠️ **주의: 패턴이 너무 복잡해질 때는 분할하세요**
>
> 고급 그룹핑 기법을 모두 동원하면 매우 강력한 패턴을 만들 수 있지만, 유지보수하기 어려운 "읽기 불가능한" 패턴이 되기도 합니다. 다음 원칙을 기억하세요:
>
> 1. **하나의 패턴이 3줄 이상이면** 분할을 고려하세요.
> 2. **`re.VERBOSE` 플래그**를 활용하여 주석을 달면 가독성이 크게 향상됩니다 (Chapter 10에서 학습).

# Part C: 통합 및 정리 (Integration Block)

## Chapter 핵심 요약 (Chapter Summary)

1. **비캡처 그룹 `(?:...)`**: 그룹핑은 하되 캡처는 하지 않습니다. `findall()` 결과를 깔끔하게 유지하고, 그룹 번호를 절약합니다.

2. **명명 그룹 `(?P<name>...)`**: 캡처 그룹에 이름을 부여하여 가독성을 높입니다. `group('name')`, `groupdict()`로 결과를 참조하고, `\g<name>`으로 치환에서 역참조합니다.

3. **패턴 내 역참조 `\1`, `(?P=name)`**: 이전에 캡처된 **정확한 문자열**이 다시 나타나는 것을 매칭합니다. 중복 단어 탐지, 따옴표 쌍 매칭 등에 활용됩니다.

4. **조건 패턴 `(?(id)yes|no)`**: 캡처 그룹의 매칭 참여 여부에 따라 패턴을 분기합니다. 괄호 쌍 매칭 등에 유용하지만, 복잡해지면 가독성이 떨어지므로 주의가 필요합니다.

5. **종합 활용**: 네 가지 기법을 조합하면 날짜 파싱, CSV 파싱, 키-값 추출 등 실전 수준의 복잡한 데이터 추출이 가능합니다.

## 자기 점검 질문 (Self-Check Questions)

**Q1.** `(?:abc)`와 `(abc)`의 가장 핵심적인 차이는 무엇인가?

<details>
<summary>정답</summary>

`(?:abc)`는 **비캡처 그룹**으로, 그룹핑만 하고 매칭 결과를 캡처하지 않습니다. `group(n)`이나 `groups()`에 포함되지 않으며 그룹 번호도 부여되지 않습니다. `(abc)`는 **캡처 그룹**으로, 그룹핑과 캡처 두 가지를 모두 수행합니다.
</details>

**Q2.** O/X: `findall()`에 캡처 그룹이 포함된 패턴을 사용하면, 전체 매칭 결과가 반환된다.

<details>
<summary>정답</summary>

**X.** `findall()`에 캡처 그룹이 있으면, 전체 매칭이 아닌 **캡처된 그룹의 내용만** 반환됩니다. 전체 매칭을 원하면 비캡처 그룹 `(?:...)`을 사용하거나 `finditer()`를 사용해야 합니다.
</details>

**Q3.** 명명 그룹 `(?P<year>\d{4})`의 결과를 가져오는 두 가지 방법은?

<details>
<summary>정답</summary>

1. 이름으로: `m.group('year')`
2. 번호로: `m.group(1)` (해당 그룹이 1번째 캡처 그룹인 경우)

추가로 `m.groupdict()`를 사용하면 `{'year': '2024'}` 형태의 딕셔너리로도 얻을 수 있습니다.
</details>

**Q4.** O/X: `groupdict()`는 명명 그룹과 일반 캡처 그룹의 결과를 모두 포함한다.

<details>
<summary>정답</summary>

**X.** `groupdict()`는 **명명 그룹의 결과만** 포함합니다. 일반 캡처 그룹(이름 없이 번호만 있는 그룹)은 포함되지 않습니다.
</details>

**Q5.** 패턴 `(.)\1`이 매칭하는 것은 "아무 두 글자"인가, "같은 글자의 반복"인가?

<details>
<summary>정답</summary>

**같은 글자의 반복**입니다. `(.)`가 첫 글자를 캡처하고, `\1`은 캡처된 **정확히 같은 글자**를 다시 요구합니다. 따라서 `aa`, `bb`, `11` 등은 매칭되지만 `ab`, `1a` 등은 매칭되지 않습니다.
</details>

**Q6.** `re.sub()` 치환 문자열에서 명명 그룹을 참조하는 구문은?

<details>
<summary>정답</summary>

`\g<name>` 형식을 사용합니다. 예: `re.sub(r'(?P<word>\w+)', r'\g<word>!', text)`
</details>

**Q7.** 조건 패턴 `(?(1)\)|)`에서, 그룹 1이 매칭에 참여하지 않았을 때 어떤 패턴이 적용되는가?

<details>
<summary>정답</summary>

빈 패턴이 적용됩니다 (`|` 뒤에 아무것도 없음). 즉, 아무 조건 없이 통과합니다. 이는 `no-pattern`이 빈 문자열인 것과 같습니다.
</details>

> <summary>모범 답안</summary>
>
> **접근 방법**: 각 캡처할 부분을 명명 그룹으로, 구조적 구분자(`[`, `]`, `:` 등)는 리터럴 또는 비캡처 그룹으로 처리합니다.
>
> **풀이 과정**:
> ```python
> import re
>
> logs = """
> [2024-01-15 09:23:45] ERROR: Database connection failed
> [2024-01-15 09:24:10] INFO: Retrying connection
> [2024-01-15 09:24:15] WARNING: Connection slow
> """
>
> pattern = (
>     r'\[(?P<date>\d{4}-\d{2}-\d{2})\s+'
>     r'(?P<time>\d{2}:\d{2}:\d{2})\]\s+'
>     r'(?P<level>\w+):\s+'
>     r'(?P<message>.+)'
> )
>
> entries = [m.groupdict() for m in re.finditer(pattern, logs)]
>
> for entry in entries:
>     print(entry)
> # {'date': '2024-01-15', 'time': '09:23:45', 'level': 'ERROR', 'message': 'Database connection failed'}
> # {'date': '2024-01-15', 'time': '09:24:10', 'level': 'INFO', 'message': 'Retrying connection'}
> # {'date': '2024-01-15', 'time': '09:24:15', 'level': 'WARNING', 'message': 'Connection slow'}
> ```
>
> **정답**: 위의 `entries` 리스트
>
> **보충 설명**: 이 패턴에서는 비캡처 그룹을 사용할 필요가 없습니다 — `[]` 같은 구조 문자는 리터럴로 직접 매칭하면 됩니다. 비캡처 그룹은 선택(`|`)이나 수량자 적용이 필요할 때 사용합니다.
> </details>

> **Exercise 8.4**: 텍스트에서 HTML 스타일의 짝이 맞는 태그 쌍을 찾아 태그 이름과 내용을 추출하세요. 역참조를 사용하여 여는 태그와 닫는 태그가 일치하는지 확인하세요.
>
> ```python
> text = "<h1>Title</h1> <p>Paragraph</p> <b>Bold<b> <div>Content</span>"
> ```
>
> <details>
> <summary>모범 답안</summary>
>
> **접근 방법**: `(?P<tag>\w+)`로 여는 태그를 캡처하고, `(?P=tag)`로 닫는 태그가 같은지 역참조합니다.
>
> **풀이 과정**:
> ```python
> import re
>
> text = "<h1>Title</h1> <p>Paragraph</p> <b>Bold<b> <div>Content</span>"
>
> pattern = r'<(?P<tag>\w+)>(?P<content>.*?)</(?P=tag)>'
>
> for m in re.finditer(pattern, text):
>     print(f"태그: {m.group('tag')}, 내용: {m.group('content')}")
> # 태그: h1, 내용: Title
> # 태그: p, 내용: Paragraph
> ```
>
> **정답**: `<h1>Title</h1>`과 `<p>Paragraph</p>`만 매칭됩니다.
>
> **보충 설명**: `<b>Bold<b>`는 닫는 태그가 `</b>`가 아닌 `<b>`이므로 매칭 안 됩니다. `<div>Content</span>`은 여는 태그와 닫는 태그가 다르므로 매칭 안 됩니다. 역참조 덕분에 태그 이름의 일치 여부를 자동으로 검증할 수 있습니다.
> </details>

## D3. Problem (심화 문제) — 깊이 있는 사고와 창의적 문제 해결

> **Problem 8.1**: 다음 텍스트에서 인용된 문자열을 추출하는 패턴을 작성하세요. 조건: (1) 작은따옴표(`'`) 또는 큰따옴표(`"`)로 감싸진 문자열을 모두 찾을 것, (2) 여는 따옴표와 닫는 따옴표가 같은 종류여야 할 것 (역참조 사용), (3) 따옴표 안의 내용만 추출할 것 (따옴표 자체는 제외), (4) 결과를 "따옴표 종류"와 "내용"이 담긴 딕셔너리 리스트로 만들 것.
>
> ```python
> text = '''He said "hello world" and she replied 'goodbye'. Then "it's over" was heard.'''
> ```
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 명명 그룹으로 따옴표를 캡처하고, 역참조로 닫는 따옴표를 매칭하며, 별도의 명명 그룹으로 내용을 추출합니다.
>
> **풀이 전략**:
> 1. `(?P<q>["'])`: 여는 따옴표를 명명 그룹으로 캡처
> 2. `(?P<content>.*?)`: 내용을 비탐욕적으로 캡처
> 3. `(?P=q)`: 닫는 따옴표가 같은 종류인지 역참조
>
> **상세 풀이**:
> ```python
> import re
>
> text = '''He said "hello world" and she replied 'goodbye'. Then "it's over" was heard.'''
>
> pattern = r'''(?P<q>["'])(?P<content>.*?)(?P=q)'''
>
> results = []
> for m in re.finditer(pattern, text):
>     results.append({
>         'quote_type': 'double' if m.group('q') == '"' else 'single',
>         'content': m.group('content')
>     })
>
> print(results)
> # [{'quote_type': 'double', 'content': 'hello world'},
> #  {'quote_type': 'single', 'content': 'goodbye'},
> #  {'quote_type': 'double', 'content': "it's over"}]
> ```
>
> **확장 생각**: "it's over"가 큰따옴표로 감싸져 있으므로 작은따옴표가 내부에 포함되어도 올바르게 매칭됩니다. 이것은 역참조의 강력함을 보여줍니다 — 여는 따옴표가 `"`이면 닫는 따옴표도 `"`를 찾으므로, 내부의 `'`는 무시됩니다. 만약 이스케이프된 따옴표(`\"`)도 처리해야 한다면 패턴이 더 복잡해집니다.
> </details>

> **Problem 8.2**: 다음 마크다운 스타일의 텍스트에서 링크를 추출하세요. 링크 형식은 `[링크텍스트](URL)` 또는 `[링크텍스트](URL "제목")` 입니다. 명명 그룹을 사용하여 각 부분을 추출하고, 제목이 있는 경우와 없는 경우를 모두 처리하세요.
>
> ```python
> text = '''
> 파이썬 공식 사이트는 [Python](https://python.org) 입니다.
> 정규표현식 테스트는 [regex101](https://regex101.com "Regular Expression Tester")에서 하세요.
> 참고 문서: [re 모듈](https://docs.python.org/3/library/re.html)
> '''
> ```
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 링크 텍스트, URL, 선택적 제목을 각각 명명 그룹으로 캡처합니다. 제목 부분은 있을 수도 없을 수도 있으므로 `?` 수량자를 사용합니다.
>
> **풀이 전략**:
> 1. `\[(?P<text>[^\]]+)\]`: 대괄호 안의 링크 텍스트 캡처
> 2. `\((?P<url>\S+?)`: URL 캡처 (비탐욕적)
> 3. `(?:\s+"(?P<title>[^"]*)")?`: 선택적 제목 (비캡처 그룹으로 묶은 후 `?` 적용)
> 4. `\)`: 닫는 소괄호
>
> **상세 풀이**:
> ```python
> import re
>
> text = '''
> 파이썬 공식 사이트는 [Python](https://python.org) 입니다.
> 정규표현식 테스트는 [regex101](https://regex101.com "Regular Expression Tester")에서 하세요.
> 참고 문서: [re 모듈](https://docs.python.org/3/library/re.html)
> '''
>
> pattern = r'\[(?P<text>[^\]]+)\]\((?P<url>\S+?)(?:\s+"(?P<title>[^"]*)")?\)'
>
> links = [m.groupdict() for m in re.finditer(pattern, text)]
>
> for link in links:
>     print(link)
> # {'text': 'Python', 'url': 'https://python.org', 'title': None}
> # {'text': 'regex101', 'url': 'https://regex101.com', 'title': 'Regular Expression Tester'}
> # {'text': 're 모듈', 'url': 'https://docs.python.org/3/library/re.html', 'title': None}
> ```
>
> **확장 생각**: 이 패턴은 비캡처 그룹 `(?:...)?`으로 선택적 부분을 감싸고, 그 안에 명명 그룹을 넣어 "있으면 캡처, 없으면 None" 동작을 구현합니다. 이 기법은 선택적 필드가 있는 구조화된 텍스트 파싱에서 매우 유용합니다.
> </details>

> **Problem 8.3**: 한 줄에 하나의 변수 할당이 있는 파이썬 코드 텍스트에서 변수명, 할당 연산자, 값을 추출하세요. 값이 문자열인 경우 따옴표 종류(역참조로 쌍 검증)와 내용을, 숫자인 경우 숫자 값을, 그 외의 경우 전체 값을 추출하세요.
>
> ```python
> code = '''
> name = "Alice"
> age = 30
> city = 'Seoul'
> pi = 3.14
> data = [1, 2, 3]
> '''
> ```
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 선택(`|`)으로 값의 종류를 분기하고, 문자열 값은 역참조로 따옴표 쌍을 검증하며, 명명 그룹으로 각 부분을 구조화합니다.
>
> **풀이 전략**: 
> - 변수명: `(?P<var>\w+)`
> - 값 분기: 문자열(`(?P<q>["'])...(?P=q)`) | 숫자(`(?P<num>[\d.]+)`) | 기타(`(?P<other>.+)`)
>
> **상세 풀이**:
> ```python
> import re
>
> code = '''
> name = "Alice"
> age = 30
> city = 'Seoul'
> pi = 3.14
> data = [1, 2, 3]
> '''
>
> pattern = (
>     r'(?P<var>\w+)\s*=\s*'
>     r'(?:'
>         r'(?P<q>["\'])(?P<str_val>.*?)(?P=q)'  # 문자열 값
>         r'|(?P<num>-?[\d.]+)'                   # 숫자 값
>         r'|(?P<other>.+)'                        # 기타 값
>     r')'
> )
>
> for m in re.finditer(pattern, code):
>     d = m.groupdict()
>     var = d['var']
>     if d['str_val'] is not None:
>         print(f"{var}: 문자열 = '{d['str_val']}' (따옴표: {d['q']})")
>     elif d['num'] is not None:
>         print(f"{var}: 숫자 = {d['num']}")
>     else:
>         print(f"{var}: 기타 = {d['other']}")
>
> # name: 문자열 = 'Alice' (따옴표: ")
> # age: 숫자 = 30
> # city: 문자열 = 'Seoul' (따옴표: ')
> # pi: 숫자 = 3.14
> # data: 기타 = [1, 2, 3]
> ```
>
> **확장 생각**: 이 패턴은 간단한 경우에만 동작합니다. 실제 파이썬 코드 파싱에는 여러줄 문자열, 이스케이프 문자, 복잡한 표현식 등을 처리해야 하므로 정규표현식보다 전용 파서(`ast` 모듈 등)가 적합합니다. 정규표현식의 한계를 인식하는 것도 중요한 능력입니다.
> </details>
