## 2.1 리터럴 매칭의 원리

> 💡 **한 줄 요약**: 정규표현식에서 일반 문자는 자기 자신과 정확히 일치하는 텍스트를 찾는다.

### 직관적 이해

도서관에서 "파이썬"이라는 단어가 들어간 책을 찾고 있다고 상상해봅시다. 사서에게 "파이썬"이라고 말하면, 사서는 모든 책 제목을 훑으며 정확히 "파이썬"이라는 글자가 포함된 책을 찾아줍니다. "파이선"도 안 되고, "Python"도 안 됩니다. 오직 "파이썬"이라는 세 글자가 순서대로 나타나는 경우만 찾아줍니다.

정규표현식에서 **리터럴 매칭**이 바로 이것입니다. 패턴에 일반 문자(알파벳, 숫자, 한글 등)를 적으면, 텍스트에서 그 문자가 정확히 그 순서대로 나타나는 위치를 찾습니다.

> 🔗 **연결**: Chapter 1에서 정규표현식의 패턴은 "리터럴 문자"와 "메타문자"로 구성된다고 배웠습니다. 이 Section에서는 리터럴 문자의 매칭 동작을 구체적으로 살펴봅니다.

### 핵심 개념 설명

> 📝 **리터럴 매칭(Literal Matching)**: 패턴의 각 문자가 텍스트의 문자와 1:1로 정확히 일치하는 매칭 방식. 특별한 의미 없이 문자 자체를 나타내는 매칭이다.

리터럴 매칭은 정규표현식의 가장 기본적인 동작입니다. 패턴 `cat`은 텍스트에서 `c`, 다음에 `a`, 다음에 `t`가 연속으로 나타나는 위치를 찾습니다.

```python
import re

text = "The cat sat on the mat"
result = re.search(r'cat', text)
print(result.group())  # 'cat'
```

리터럴 매칭에서 기억해야 할 핵심 규칙들이 있습니다.

**규칙 1: 대소문자를 구분한다**

기본적으로 정규표현식은 대소문자를 엄격하게 구분합니다.

```python
text = "Cat and cat are different"

print(re.search(r'cat', text).group())  # 'cat' (소문자 cat을 찾음)
print(re.search(r'Cat', text).group())  # 'Cat' (대문자 Cat을 찾음)
print(re.search(r'CAT', text))          # None (CAT은 없음)
```

**규칙 2: 부분 문자열을 매칭한다**

패턴은 텍스트 전체가 아니라 텍스트의 *일부분*에서도 매칭됩니다. `cat`이라는 패턴은 `caterpillar`에서도 앞부분의 `cat`에 매칭됩니다.

```python
text = "The caterpillar crawled"
result = re.search(r'cat', text)
print(result.group())   # 'cat'
print(result.start())   # 4 (인덱스 4에서 시작)
```

**규칙 3: 가장 먼저 나타나는 위치를 찾는다**

텍스트에 매칭되는 부분이 여러 곳에 있으면, `re.search()`는 가장 처음 나타나는 위치를 반환합니다.

```python
text = "cat hat cat mat"
result = re.search(r'cat', text)
print(result.start())  # 0 (첫 번째 'cat'의 위치)
```

**규칙 4: 공백과 특수문자도 리터럴이다**

공백(` `), 탭, 느낌표 등도 메타문자가 아닌 한 리터럴 문자입니다. 패턴 `hello world`는 `hello`와 공백 한 칸, 그리고 `world`가 정확히 이 순서로 나타나는 곳을 찾습니다.

```python
text = "Say hello world!"
result = re.search(r'hello world', text)
print(result.group())  # 'hello world'
```

### 상세 예시

> 🔍 **예시 2.1.1: 간단한 단어 찾기**
>
> **상황**: 텍스트에서 "error"라는 단어가 포함되어 있는지 확인하고 싶습니다.
> ```python
> log_line = "2024-01-15 09:30:22 [error] Connection timeout"
> result = re.search(r'error', log_line)
> if result:
>     print(f"에러 발견! 위치: {result.start()}")  # 에러 발견! 위치: 25
> ```
> **핵심 포인트**: 리터럴 패턴은 텍스트 어디에서든 매칭됩니다. 대괄호 안의 `error`도 정확히 찾아냅니다.

> 🔍 **예시 2.1.2: 대소문자 구분의 함정**
>
> **상황**: 로그에서 "Error", "error", "ERROR" 등 다양한 형태의 에러 표시를 찾아야 합니다.
> ```python
> logs = [
>     "[Error] File not found",
>     "[error] Connection refused",
>     "[ERROR] System failure"
> ]
>
> for log in logs:
>     result = re.search(r'error', log)
>     print(f"'{log}' → 매칭: {result is not None}")
> # '[Error] File not found' → 매칭: False    ← Error의 E가 대문자!
> # '[error] Connection refused' → 매칭: True
> # '[ERROR] System failure' → 매칭: False     ← 전부 대문자!
> ```
> **핵심 포인트**: 리터럴 매칭은 대소문자를 구분합니다. 대소문자 무시 매칭은 나중에 플래그를 통해 해결할 수 있습니다.

> 💬 **참고**: 대소문자를 무시하고 매칭하는 방법(`re.IGNORECASE` 플래그)은 Chapter 10에서 자세히 다룹니다. 지금은 리터럴 매칭이 기본적으로 대소문자를 구분한다는 점을 기억하세요.

> ⚠️ **주의**: 리터럴 매칭에서 가장 흔한 실수는 "전체 단어"를 매칭하려 했는데 부분 문자열이 매칭되는 경우입니다.
> ```python
> text = "education is important"
> result = re.search(r'cat', text)
> print(result.group())  # 'cat' ← "education"의 일부분!
> ```
> 단어 단위의 정밀한 매칭은 Chapter 3에서 배울 **단어 경계(`\b`)**로 해결할 수 있습니다.

### Section 요약

- 리터럴 매칭은 패턴의 문자가 텍스트의 문자와 1:1로 정확히 일치하는 가장 기본적인 매칭 방식이다
- 기본적으로 **대소문자를 구분**하며, **부분 문자열**에서도 매칭된다
- 공백, 숫자, 한글 등도 리터럴 문자로 그 자체를 매칭한다
- `re.search()`는 가장 처음 나타나는 매칭 위치를 반환한다

## 2.2 임의의 문자: 마침표(`.`)

> 💡 **한 줄 요약**: 마침표(`.`)는 줄바꿈을 제외한 **어떤 문자 하나**와도 매칭되는 와일드카드이다.

### 직관적 이해

카드 게임에서 **조커 카드**를 아시나요? 조커는 어떤 카드 대신이든 사용할 수 있습니다. 정규표현식에서 마침표(`.`)가 바로 이 조커 역할을 합니다. `.`은 알파벳이든, 숫자든, 특수문자든, 공백이든 — 딱 하나의 문자와 매칭됩니다.

왜 이런 기능이 필요할까요? 실제 텍스트 처리에서는 "정확히 어떤 문자인지는 모르지만, 그 자리에 *무언가*가 있다"는 것만 알 때가 있습니다. 예를 들어, "c로 시작하고 t로 끝나는 세 글자 단어"를 찾고 싶다면, 가운데 글자가 무엇이든 상관없이 매칭하고 싶을 것입니다. 바로 이럴 때 `.`을 씁니다.

### 핵심 개념 설명

> 📝 **와일드카드(Wildcard)**: 특정 문자가 아닌 "아무 문자나"를 대표하는 특수 기호. 정규표현식에서 `.`이 이 역할을 한다.

마침표(`.`)는 정규표현식의 첫 번째 **메타문자**입니다. 리터럴 문자가 자기 자신만 매칭하는 것과 달리, `.`은 줄바꿈 문자(`\n`)를 제외한 어떤 문자 하나와도 매칭됩니다.

```python
import re

# 'c'로 시작, 't'로 끝나는 세 글자
pattern = r'c.t'

print(re.search(r'c.t', 'cat').group())   # 'cat' (a와 매칭)
print(re.search(r'c.t', 'cut').group())   # 'cut' (u와 매칭)
print(re.search(r'c.t', 'c3t').group())   # 'c3t' (3과 매칭)
print(re.search(r'c.t', 'c t').group())   # 'c t' (공백과 매칭)
print(re.search(r'c.t', 'c!t').group())   # 'c!t' (!와 매칭)
print(re.search(r'c.t', 'ct'))            # None (사이에 문자가 없음!)
```

핵심 규칙을 정리하면:

| 규칙 | 설명 |
|------|------|
| `.`은 정확히 **한 글자**와 매칭 | 두 글자나 영(0)글자와는 매칭되지 않음 |
| 줄바꿈(`\n`)은 매칭하지 않음 | 기본 동작에서 `.`은 `\n`을 건너뜀 |
| 위치에 상관없이 매칭 | 문자열 어디에서든 작동 |

`.`을 여러 개 나란히 쓰면 연속된 여러 문자와 매칭됩니다:

```python
text = "2024-01-15"
# 네 자리 숫자 뒤에 하이픈
print(re.search(r'....-..-..', text).group())  # '2024-01-15'
```

**줄바꿈 문자와 `.`**

`.`이 유일하게 매칭하지 *않는* 문자가 줄바꿈(`\n`)입니다:

```python
text = "Hello\nWorld"
print(re.search(r'Hello.World', text))  # None — \n은 .과 매칭 안 됨
```

> 💬 **참고**: `.`이 줄바꿈까지 포함하여 매칭하도록 하려면 `re.DOTALL` 플래그를 사용합니다. 이는 Chapter 10에서 다룹니다.

### 상세 예시

> 🔍 **예시 2.2.1: 날짜 형식의 느슨한 매칭**
>
> **상황**: 로그 파일에서 "2024-01-15", "2024/01/15", "2024.01.15" 등 다양한 구분자를 사용하는 날짜를 찾고 싶습니다.
> ```python
> logs = [
>     "Date: 2024-01-15 OK",
>     "Date: 2024/01/15 OK",
>     "Date: 2024.01.15 OK",
> ]
>
> for log in logs:
>     result = re.search(r'2024.01.15', log)
>     if result:
>         print(f"찾음: {result.group()}")
> # 찾음: 2024-01-15
> # 찾음: 2024/01/15
> # 찾음: 2024.01.15
> ```
> **핵심 포인트**: `.`이 `-`, `/`, `.` 등 어떤 구분자든 매칭하므로, 구분자의 종류에 관계없이 날짜를 찾을 수 있습니다. 다만, `2024X01Y15`처럼 예상하지 못한 문자에도 매칭되므로 주의가 필요합니다.

> 🔍 **예시 2.2.2: 파일 확장자 찾기의 함정**
>
> **상황**: 텍스트에서 `.txt` 확장자를 찾으려 합니다.
> ```python
> text = "files: data.txt, notes_txt, image.png"
>
> # 의도: .txt를 찾고 싶음
> result = re.findall(r'.txt', text)
> print(result)  # ['atxt', '.txt', 's_tx'] ← 예상과 다름!
> ```
> **핵심 포인트**: 패턴 `.txt`에서 `.`은 "아무 문자 하나"를 의미합니다. 따라서 `atxt`(`a` + `txt`), `.txt`(`.` + `txt`), `s_tx`와 같은 것도 매칭 후보가 됩니다. 실제 마침표를 매칭하려면 이스케이프가 필요한데, 이는 Section 2.5에서 배웁니다.

> ⚠️ **주의**: `.`의 가장 흔한 함정은 "마침표 문자 자체"를 매칭하려 할 때입니다.
> - 패턴 `3.14`는 `3.14`뿐만 아니라 `3X14`, `3 14`, `3-14` 등과도 매칭됩니다
> - 진짜 마침표를 매칭하려면 `\.`(이스케이프)을 사용해야 합니다 — Section 2.5에서 다룹니다
> - **기억하세요**: `.`은 "아무 문자 하나"이지 "마침표"가 아닙니다!

### Section 요약

- `.`(마침표)은 줄바꿈(`\n`)을 제외한 **어떤 문자 하나**와도 매칭되는 와일드카드이다
- `.`은 정확히 **한 글자**만 매칭하며, 0글자나 2글자 이상과는 매칭되지 않는다
- 실제 마침표 문자를 매칭하려면 이스케이프(`\.`)가 필요하다
- `.`을 여러 개 나란히 쓰면 연속된 여러 문자와 매칭된다

## 2.3 위치 앵커: `^`, `$`, `\A`, `\Z`

> 💡 **한 줄 요약**: 앵커는 문자를 매칭하는 것이 아니라, 문자열의 **위치**(시작 또는 끝)를 지정하는 메타문자이다.

### 직관적 이해

지금까지 배운 리터럴 매칭과 `.`은 모두 **문자를 매칭**하는 도구였습니다. 그런데 때로는 "이 패턴이 문자열의 맨 앞에 있을 때만 매칭하고 싶다"거나, "맨 끝에 있을 때만 매칭하고 싶다"는 요구사항이 있습니다.

이것을 일상생활에 비유해볼까요? "김"으로 **시작하는** 이름과, 그냥 "김"이 **포함된** 이름은 다릅니다. "김민수"와 "박김철"은 둘 다 "김"을 포함하지만, "김"으로 시작하는 이름은 "김민수"뿐입니다. 마찬가지로, "수"로 **끝나는** 이름은 "김민수"이지, "수박맨"은 아닙니다.

정규표현식에서 **앵커(anchor)**가 바로 이 "시작"과 "끝"을 지정하는 도구입니다. 앵커는 문자를 소비하지 않고, 위치만 지정합니다.

> 📝 **앵커(Anchor)**: 텍스트의 특정 **위치**(시작, 끝 등)를 지정하는 메타문자. 문자 자체를 매칭하지 않으며, 매칭이 일어나야 할 위치 조건을 설정한다.

### 핵심 개념 설명

파이썬 정규표현식에는 네 가지 주요 앵커가 있습니다:

| 앵커 | 의미 | 설명 |
|------|------|------|
| `^` | 시작 | 문자열의 시작 위치 (기본 모드) |
| `$` | 끝 | 문자열의 끝 위치 (기본 모드) |
| `\A` | 절대 시작 | 문자열의 절대적 시작 위치 |
| `\Z` | 절대 끝 | 문자열의 절대적 끝 위치 |

**`^` — 시작 앵커**

`^`는 패턴이 문자열의 **맨 처음**에 와야 한다는 조건을 겁니다.

```python
import re

text = "Hello World"

# 'Hello'가 문자열 시작에 있는가?
print(re.search(r'^Hello', text))  # <re.Match object; ...> ✅ 매칭!

# 'World'가 문자열 시작에 있는가?
print(re.search(r'^World', text))  # None ❌ World는 시작에 없음
```

`^`를 패턴의 맨 앞에 놓으면, "이 패턴은 문자열의 시작에서만 찾아라"라는 의미가 됩니다.

**`$` — 끝 앵커**

`$`는 패턴이 문자열의 **맨 끝**에 와야 한다는 조건을 겁니다.

```python
text = "Hello World"

# 'World'가 문자열 끝에 있는가?
print(re.search(r'World$', text))  # <re.Match object; ...> ✅ 매칭!

# 'Hello'가 문자열 끝에 있는가?
print(re.search(r'Hello$', text))  # None ❌ Hello는 끝에 없음
```

**`^`와 `$`를 함께 사용 — 전체 문자열 매칭**

`^`와 `$`를 동시에 사용하면, 문자열 **전체가** 패턴과 일치하는지 검사할 수 있습니다.

```python
# 문자열 전체가 "Hello World"인지 확인
print(re.search(r'^Hello World$', "Hello World"))        # ✅ 매칭
print(re.search(r'^Hello World$', "Hello World!"))       # None ❌ (뒤에 !가 더 있음)
print(re.search(r'^Hello World$', "Say Hello World"))    # None ❌ (앞에 Say가 더 있음)
```

**`\A`와 `\Z` — 절대 앵커**

`\A`와 `\Z`는 `^`, `$`와 비슷하게 각각 문자열의 시작과 끝을 나타냅니다. 기본적인 사용에서는 동일하게 동작합니다:

```python
text = "Hello World"

print(re.search(r'\AHello', text))   # ✅ 매칭 (^Hello와 동일한 결과)
print(re.search(r'World\Z', text))   # ✅ 매칭 (World$와 동일한 결과)
```

그렇다면 왜 두 종류의 앵커가 존재할까요? 핵심적인 차이는 **여러 줄 모드(MULTILINE)**에서 드러납니다.

> 💬 **참고**: `^`/`$`와 `\A`/`\Z`의 결정적 차이는 `re.MULTILINE` 플래그를 사용할 때 나타납니다. `re.MULTILINE` 모드에서 `^`/`$`는 각 줄의 시작/끝에서도 매칭되지만, `\A`/`\Z`는 항상 전체 문자열의 시작/끝에서만 매칭됩니다. 이 플래그는 Chapter 10에서 자세히 배우게 됩니다. 지금은 다음만 기억하세요:
> - `^`/`$` → 기본 모드에서는 `\A`/`\Z`와 동일, **MULTILINE 모드에서는 동작이 달라짐**
> - `\A`/`\Z` → **어떤 모드에서든 항상** 문자열의 절대적 시작/끝만 의미

차이를 미리 맛보기로 살펴보면:

```python
text = "first line\nsecond line\nthird line"

# 기본 모드에서는 ^와 \A가 동일하게 동작
print(re.search(r'^first', text))   # ✅ 매칭
print(re.search(r'\Afirst', text))  # ✅ 매칭

print(re.search(r'^second', text))  # None (기본 모드에서 ^는 문자열 시작만)
print(re.search(r'\Asecond', text)) # None

# MULTILINE 모드에서 차이 발생 (Ch10에서 상세 학습)
print(re.search(r'^second', text, re.MULTILINE))   # ✅ 매칭! (각 줄의 시작에서 매칭)
print(re.search(r'\Asecond', text, re.MULTILINE))  # None (여전히 문자열 절대 시작만)
```

**앵커는 문자를 "소비"하지 않는다**

앵커의 가장 중요한 특성은 문자를 소비하지 않는다는 것입니다. `^`나 `$`는 문자열에서 어떤 문자와 매칭되는 것이 아니라, **위치**와 매칭됩니다.

```python
text = "Hello"
result = re.search(r'^Hello$', text)
print(result.group())  # 'Hello' — 매칭된 것은 "Hello"이지, "^Hello$"가 아님
print(len(result.group()))  # 5 — ^와 $는 매칭 결과에 포함되지 않음
```

### 상세 예시

> 🔍 **예시 2.3.1: 주석 줄 찾기**
>
> **상황**: 파이썬 코드에서 `#`으로 시작하는 주석 줄을 찾고 싶습니다.
> ```python
> lines = [
>     "# 이것은 주석입니다",
>     "x = 5  # 인라인 주석",
>     "print('Hello')",
> ]
>
> for line in lines:
>     if re.search(r'^#', line):
>         print(f"주석 줄: {line}")
> # 주석 줄: # 이것은 주석입니다
> ```
> **핵심 포인트**: `^#`은 줄의 맨 처음에 `#`이 있는 경우만 매칭합니다. `"x = 5  # 인라인 주석"`에서 `#`은 중간에 있으므로 매칭되지 않습니다.

> 🔍 **예시 2.3.2: 파일 확장자 확인**
>
> **상황**: 파일 이름이 `.py`로 끝나는지 확인하고 싶습니다.
> ```python
> filenames = ["main.py", "data.csv", "script.py", "readme.py.bak"]
>
> for name in filenames:
>     if re.search(r'\.py$', name):
>         print(f"파이썬 파일: {name}")
> # 파이썬 파일: main.py
> # 파이썬 파일: script.py
> ```
> **핵심 포인트**: `\.py$`에서 `\.`는 실제 마침표를(Section 2.5에서 상세 설명), `py`는 리터럴 문자를, `$`는 문자열 끝을 의미합니다. `readme.py.bak`은 `.py` 뒤에 `.bak`이 더 있으므로 `$` 조건을 만족하지 못합니다.

> 🔍 **예시 2.3.3: 전체 문자열 검증**
>
> **상황**: 사용자 입력이 정확히 "yes" 또는 "no"인지 확인하고 싶습니다.
> ```python
> inputs = ["yes", "yes!", "no", "not really", "maybe"]
>
> for user_input in inputs:
>     if re.search(r'^yes$', user_input) or re.search(r'^no$', user_input):
>         print(f"유효한 입력: {user_input}")
>     else:
>         print(f"잘못된 입력: {user_input}")
> # 유효한 입력: yes
> # 잘못된 입력: yes!
> # 유효한 입력: no
> # 잘못된 입력: not really
> # 잘못된 입력: maybe
> ```
> **핵심 포인트**: `^yes$`는 문자열이 정확히 "yes"인 경우만 매칭합니다. "yes!"는 끝에 `!`가 있어 `$` 조건을 만족하지 못하고, "not really"는 시작이 "no"가 아니라 "not"이므로 `^no$`에 매칭되지 않습니다.

> ⚠️ **주의**: `^`와 `$`의 위치에 주의하세요.
> - `^`는 **패턴의 맨 앞**에 놓아야 "시작 앵커"로 작동합니다
> - `$`는 **패턴의 맨 뒤**에 놓아야 "끝 앵커"로 작동합니다
> - 패턴 중간에 있는 `^`나 `$`는 다른 의미를 가질 수 있습니다 (`^`는 문자 클래스 `[^...]` 안에서 부정을 의미 — Chapter 3에서 학습)

### Section 요약

- `^`은 문자열의 **시작**, `$`은 문자열의 **끝** 위치를 지정하는 앵커이다
- `\A`와 `\Z`는 **절대적인** 시작/끝 앵커로, MULTILINE 모드에서도 전체 문자열의 시작/끝만 의미한다
- `^`와 `$`를 함께 사용하면 문자열 **전체가** 패턴과 일치하는지 검증할 수 있다
- 앵커는 문자를 소비하지 않으며, **위치만** 지정한다

## 2.4 선택 연산자: `|`

> 💡 **한 줄 요약**: `|`(파이프)는 "또는(OR)"을 의미하며, 여러 패턴 중 하나와 매칭되면 성공이다.

### 직관적 이해

식당 메뉴에서 "커피 또는 차 중 하나를 선택하세요"라고 한다면, 커피를 골라도 되고 차를 골라도 됩니다. 둘 중 하나만 만족하면 됩니다.

정규표현식의 `|`(파이프 기호)가 바로 이 "또는"입니다. `cat|dog`라는 패턴은 "cat 또는 dog"을 의미하며, 텍스트에서 `cat`이나 `dog` 중 하나라도 찾으면 매칭에 성공합니다.

### 핵심 개념 설명

> 📝 **선택 연산자(Alternation)**: `|` 기호로 표현되며, 왼쪽 패턴과 오른쪽 패턴 중 하나가 매칭되면 전체가 매칭으로 처리되는 OR 논리 연산자이다.

기본 사용법은 간단합니다:

```python
import re

text = "I have a cat and a dog"

# cat 또는 dog 찾기
result = re.search(r'cat|dog', text)
print(result.group())  # 'cat' (먼저 나오는 것이 매칭됨)

# 모든 매칭 찾기
results = re.findall(r'cat|dog', text)
print(results)  # ['cat', 'dog']
```

**세 개 이상의 선택지**

`|`을 여러 번 사용하여 세 개 이상의 선택지를 나열할 수 있습니다:

```python
text = "I like red, blue, and green colors"
colors = re.findall(r'red|blue|green|yellow', text)
print(colors)  # ['red', 'blue', 'green']
```

**선택 연산자의 범위 — 중요!**

`|`는 **패턴 전체**를 양쪽으로 나눕니다. 이것이 가장 혼동하기 쉬운 부분입니다.

```python
# 의도: "cat" 또는 "car"를 찾고 싶음
# 올바른 패턴
re.findall(r'cat|car', "I have a cat and a car")  # ['cat', 'car'] ✅

# 주의: |의 범위를 잘못 이해한 경우
# 패턴 'ca(t|r)'는 같은 결과를 내지만, 그룹화가 필요함 (아래 참고)
```

`|`의 범위를 제한하려면 **괄호 `()`로 그룹화**해야 합니다:

```python
# |의 범위 문제 예시
text = "I saw a catfish and a dogfish"

# 의도: "catfish" 또는 "dogfish"
re.findall(r'catfish|dogfish', text)  # ['catfish', 'dogfish'] ✅

# 더 간결하게: 공통 부분을 묶기
re.findall(r'(cat|dog)fish', text)    # ['cat', 'dog'] — ()가 캡처하므로 결과가 다름!
```

> 💬 **참고**: 위 예시에서 `(cat|dog)fish`의 `()`는 **캡처 그룹**으로 작동합니다. `findall()`은 캡처 그룹이 있으면 그룹 안의 내용만 반환하므로, `'catfish'` 대신 `'cat'`이 반환됩니다. 캡처 그룹의 상세 동작은 Chapter 6에서 배우게 됩니다. 지금은 `()`가 `|`의 범위를 제한하는 데 사용된다는 점만 이해하세요.

**선택 연산자와 매칭 순서**

`re.search()`에서 `|`로 연결된 선택지 중 텍스트에서 **더 먼저 나타나는 것**이 매칭됩니다:

```python
text = "dog and cat"
result = re.search(r'cat|dog', text)
print(result.group())  # 'dog' — 패턴에서 cat이 먼저지만, 텍스트에서 dog가 먼저!
```

만약 같은 위치에서 두 선택지 모두 매칭 가능하다면, **패턴에서 왼쪽에 있는 것**이 우선합니다:

```python
text = "catdog"
result = re.search(r'cat|catdog', text)
print(result.group())  # 'cat' — 같은 위치에서 cat이 먼저 시도되어 성공
```

### 상세 예시

> 🔍 **예시 2.4.1: 요일 찾기**
>
> **상황**: 텍스트에서 한글 요일 표현을 찾아야 합니다.
> ```python
> text = "회의는 월요일과 수요일에 있습니다. 금요일은 쉽니다."
> days = re.findall(r'월요일|화요일|수요일|목요일|금요일|토요일|일요일', text)
> print(days)  # ['월요일', '수요일', '금요일']
> ```
> **핵심 포인트**: `|`로 모든 요일을 나열하면 텍스트에서 해당 요일들을 모두 찾을 수 있습니다.

> 🔍 **예시 2.4.2: 프로토콜 식별**
>
> **상황**: URL의 시작 부분에서 프로토콜(http 또는 https 또는 ftp)을 식별하고 싶습니다.
> ```python
> urls = [
>     "https://example.com",
>     "http://test.org",
>     "ftp://files.server.com",
>     "smtp://mail.server.com"
> ]
>
> for url in urls:
>     result = re.search(r'^(https|http|ftp)', url)
>     if result:
>         print(f"{url} → 프로토콜: {result.group()}")
>     else:
>         print(f"{url} → 지원하지 않는 프로토콜")
> # https://example.com → 프로토콜: https
> # http://test.org → 프로토콜: http
> # ftp://files.server.com → 프로토콜: ftp
> # smtp://mail.server.com → 지원하지 않는 프로토콜
> ```
> **핵심 포인트**: `^`와 `|`를 결합하여 "문자열 시작이 https, http, ftp 중 하나인지" 확인합니다. `()`는 여기서 `|`의 범위를 제한하는 역할을 합니다. 만약 `^https|http|ftp`라고 쓰면, "^https" 또는 "http(어디서든)" 또는 "ftp(어디서든)"이라는 뜻이 되어 의도와 달라집니다!

> ⚠️ **주의**: `|`의 범위를 잘못 이해하면 예상치 못한 결과가 나옵니다.
> ```python
> # 잘못된 패턴: ^가 첫 번째 선택지에만 적용됨
> re.search(r'^cat|dog', "dog is here")
> # 매칭됨! — "^cat" 또는 "dog"로 해석되어, dog가 어디서든 매칭
>
> # 올바른 패턴: ^가 전체에 적용되도록 그룹화
> re.search(r'^(cat|dog)', "dog is here")
> # 매칭됨! — 문자열이 "dog"로 시작하므로
>
> re.search(r'^(cat|dog)', "the dog")
> # None — 문자열이 "cat"이나 "dog"로 시작하지 않음
> ```

### Section 요약

- `|`는 "또는(OR)"을 의미하며, 양쪽 패턴 중 하나가 매칭되면 성공이다
- `|`의 범위는 **패턴 전체**에 걸치므로, 범위를 제한하려면 **괄호 `()`**로 그룹화해야 한다
- `re.search()`에서는 텍스트에서 더 먼저 나타나는 매칭이 반환된다
- 같은 위치에서 여러 선택지가 매칭 가능하면, 패턴에서 **왼쪽 선택지가 우선**한다

## 2.5 이스케이프와 raw string

> 💡 **한 줄 요약**: 백슬래시(`\`)로 메타문자의 특수 의미를 해제하고, raw string(`r''`)으로 백슬래시 충돌을 방지한다.

### 직관적 이해

지금까지 `.`, `^`, `$`, `|` 같은 메타문자를 배웠습니다. 이 문자들은 정규표현식에서 특별한 의미를 가지고 있죠. 그런데 만약 텍스트에서 진짜 마침표(`.`)나 진짜 달러 기호(`$`)를 찾고 싶다면 어떻게 할까요?

이것은 마치 대화에서 따옴표를 사용하는 것과 비슷합니다. 누군가 '사과'라고 말하면 과일을 떠올리지만, '"사과"라는 단어'라고 말하면 단어 자체를 가리킵니다. 정규표현식에서 백슬래시(`\`)가 바로 이 "따옴표" 역할을 합니다. `.`은 "아무 문자"를 의미하지만, `\.`은 "진짜 마침표"를 의미합니다.

### 핵심 개념 설명

> 📝 **이스케이프(Escape)**: 메타문자 앞에 백슬래시(`\`)를 붙여 그 특수 의미를 해제하고, 문자 그대로의 의미로 사용하는 것.

**메타문자의 이스케이프**

정규표현식에서 특별한 의미를 가진 메타문자를 리터럴로 매칭하려면 앞에 `\`를 붙입니다:

```python
import re

# 실제 마침표를 찾고 싶을 때
text = "가격은 3.14달러입니다"
print(re.search(r'3\.14', text).group())  # '3.14' ✅ 정확히 마침표를 매칭

# \를 안 붙이면?
print(re.search(r'3.14', text).group())   # '3.14' — 이것도 매칭은 되지만...
print(re.search(r'3.14', "가격은 3X14원").group())  # '3X14' — 이런 것도 매칭됨!
```

이스케이프가 필요한 주요 메타문자 목록:

| 메타문자 | 이스케이프 | 리터럴 의미 |
|---------|-----------|------------|
| `.` | `\.` | 마침표 |
| `^` | `\^` | 캐럿 기호 |
| `$` | `\$` | 달러 기호 |
| `*` | `\*` | 별표 |
| `+` | `\+` | 더하기 기호 |
| `?` | `\?` | 물음표 |
| `\|` | `\\|` | 파이프 기호 |
| `\\` | `\\\\` | 백슬래시 자체 |
| `(` | `\(` | 여는 괄호 |
| `)` | `\)` | 닫는 괄호 |
| `[` | `\[` | 여는 대괄호 |
| `{` | `\{` | 여는 중괄호 |

**특수 이스케이프 시퀀스**

백슬래시(`\`)는 메타문자를 이스케이프하는 것 외에도, 특정 문자와 결합하여 **특수한 문자를 표현**하는 역할도 합니다:

| 시퀀스 | 의미 |
|--------|------|
| `\n` | 줄바꿈(newline) |
| `\t` | 탭(tab) |
| `\r` | 캐리지 리턴(carriage return) |

```python
# 탭으로 구분된 데이터에서 탭을 찾기
text = "이름\t나이\t직업"
parts = re.findall(r'\t', text)
print(len(parts))  # 2 (탭이 2개)
```

> 💬 **참고**: `\d`(숫자), `\w`(단어 문자), `\s`(공백), `\b`(단어 경계) 같은 더 강력한 이스케이프 시퀀스도 있습니다. 이들은 Chapter 3에서 자세히 다룹니다.

**raw string(`r''`) — 이스케이프의 이스케이프 문제 해결**

여기서 한 가지 복잡한 문제가 있습니다. 파이썬 문자열 자체도 `\`를 이스케이프 문자로 사용한다는 점입니다. 즉, **파이썬 문자열의 이스케이프**와 **정규표현식의 이스케이프**가 충돌합니다.

> 📝 **raw string**: 파이썬에서 문자열 앞에 `r`을 붙여 백슬래시를 이스케이프 문자로 해석하지 않게 하는 문자열 표기법. `r'\n'`은 줄바꿈이 아니라 `\`와 `n` 두 글자이다.

이 문제를 구체적으로 살펴봅시다:

```python
# 문제 상황: 텍스트에서 "\n"이라는 두 글자를 찾고 싶음 (줄바꿈이 아님)
text = r"줄바꿈은 \n으로 표현합니다"  # 실제로 \와 n이 들어있는 텍스트

# ❌ 일반 문자열 사용 시 문제
pattern_bad = '\\n'           # 파이썬이 \\ → \로 변환, 결과적으로 \n(줄바꿈)이 됨!
print(repr(pattern_bad))      # '\n' (줄바꿈 한 문자)

# ✅ raw string 사용
pattern_good = r'\\n'         # r을 붙이면 \\n이 그대로 유지
print(repr(pattern_good))     # '\\\\n' — 실제로 \\과 n, 정규표현식 엔진이 이를 \n 리터럴로 해석
```

이 이중 이스케이프 문제를 표로 정리하면:

| 찾고 싶은 것 | raw string 패턴 | 일반 문자열 패턴 |
|-------------|----------------|-----------------|
| 마침표 `.` | `r'\.'` | `'\\.'` |
| 백슬래시 `\` | `r'\\'` | `'\\\\'` |
| `\n` 리터럴 (2글자) | `r'\\n'` | `'\\\\n'` |
| 줄바꿈 문자 | `r'\n'` | `'\\n'` |

보시다시피, raw string을 사용하면 패턴이 훨씬 읽기 쉬워집니다.

**실전 권장 사항**: 정규표현식 패턴은 **항상 raw string(`r''`)으로** 작성하세요. 이것이 파이썬 정규표현식의 핵심 관례이며, 이 교재에서도 모든 패턴을 raw string으로 작성합니다.

```python
# ✅ 항상 이렇게
pattern = r'\d+\.\d+'  # 읽기 쉬움

# ❌ 이렇게 하지 마세요
pattern = '\\d+\\.\\d+'  # 읽기 어려움 (동일한 패턴)
```

### 상세 예시

> 🔍 **예시 2.5.1: 가격 패턴 매칭**
>
> **상황**: 텍스트에서 "$10.99"와 같은 가격 표시를 찾고 싶습니다. `$`와 `.` 모두 메타문자이므로 이스케이프가 필요합니다.
> ```python
> text = "Regular price: $10.99, Sale price: $7.50"
>
> # 메타문자 이스케이프 없이 (❌ 위험)
> print(re.findall(r'$10.99', text))  # [] — $가 끝 앵커로 해석됨!
>
> # 올바른 이스케이프 (✅)
> print(re.findall(r'\$..\...', text))  # ['$10.99', '$7.50']
> ```
> **핵심 포인트**: `\$`는 달러 기호 자체를, `\.`는 마침표 자체를 매칭합니다. `.`(이스케이프 없는)은 아무 문자 하나를 매칭합니다.

> 🔍 **예시 2.5.2: Windows 파일 경로 매칭**
>
> **상황**: Windows 파일 경로에서 백슬래시를 매칭해야 합니다.
> ```python
> path = r"C:\Users\Documents\file.txt"
>
> # raw string 패턴으로 백슬래시 매칭
> result = re.search(r'C:\\Users', path)
> print(result.group())  # 'C:\\Users'
>
> # 폴더 구분자(백슬래시) 개수 세기
> separators = re.findall(r'\\', path)
> print(f"백슬래시 개수: {len(separators)}")  # 백슬래시 개수: 3
> ```
> **핵심 포인트**: raw string에서 `r'\\'`는 정규표현식 엔진에게 "백슬래시 하나를 찾아라"라는 의미입니다. raw string이 없다면 `'\\\\\\\\'`(백슬래시 4개)를 써야 하므로, raw string의 편의성을 실감할 수 있습니다.

> 🔍 **예시 2.5.3: 수학 수식 매칭**
>
> **상황**: 텍스트에서 "3+5=8" 같은 간단한 덧셈 수식을 찾고 싶습니다. `+`는 메타문자입니다.
> ```python
> text = "계산: 3+5=8, 10+20=30"
>
> # +를 이스케이프해야 함
> results = re.findall(r'\d\+\d=\d', text)
> print(results)  # ['3+5=8']  — 10+20=30은 자릿수가 달라서 미매칭
> ```
> **핵심 포인트**: `\+`는 리터럴 더하기 기호를 매칭합니다. 이스케이프 없이 `+`를 쓰면 "앞 문자의 1회 이상 반복"이라는 완전히 다른 의미가 됩니다(Chapter 4에서 학습).

> ⚠️ **주의**: raw string을 사용하더라도 주의할 점이 있습니다.
> ```python
> # raw string에서도 따옴표 이스케이프는 파이썬이 처리
> pattern = r"She said \"hello\""  # 따옴표 이스케이프는 필요
>
> # raw string은 홀수 개의 백슬래시로 끝날 수 없음
> # path = r"C:\path\"  ← SyntaxError!
> # 해결: path = r"C:\path" + "\\"
> ```

> 🔬 **Deep Dive**: 파이썬의 이중 이스케이프 문제
>
> *이 부분은 심화 내용입니다. 처음 학습 시 건너뛰어도 됩니다.*
>
> 정규표현식에서 이스케이프가 복잡한 이유는 **두 단계의 해석**을 거치기 때문입니다:
>
> 1. **파이썬 문자열 파서**: 파이썬이 문자열 리터럴을 해석합니다
> 2. **정규표현식 엔진**: `re` 모듈이 패턴 문자열을 해석합니다
>
> 예를 들어, 텍스트에서 `\d`라는 두 글자(백슬래시 + d)를 찾고 싶다면:
>
> ```python
> text = r"패턴에서 \d는 숫자를 의미합니다"  # \d가 리터럴 두 글자로 들어있음
>
> # 일반 문자열로 패턴 작성 시:
> # '\\\\d' → 파이썬 해석 → '\\d' → 정규표현식 해석 → 리터럴 \d
> print(re.search('\\\\d', text).group())  # '\d'
>
> # raw string으로 패턴 작성 시:
> # r'\\d' → 파이썬 해석(무변환) → '\\d' → 정규표현식 해석 → 리터럴 \d
> print(re.search(r'\\d', text).group())   # '\d'
> ```
>
> | 단계 | 일반 문자열 `'\\\\d'` | raw string `r'\\d'` |
> |------|---------------------|---------------------|
> | 작성한 코드 | `'\\\\d'` | `r'\\d'` |
> | 파이썬 해석 후 | `\\d` (2글자) | `\\d` (2글자) |
> | 정규표현식 해석 | 리터럴 `\d` | 리터럴 `\d` |
>
> raw string을 사용하면 첫 번째 단계에서 변환이 없으므로, 훨씬 직관적으로 패턴을 작성할 수 있습니다.

### Section 요약

- `\`(백슬래시)를 메타문자 앞에 붙이면 특수 의미가 해제되어 **리터럴 문자**로 매칭된다
- `\n`, `\t` 등은 **특수 이스케이프 시퀀스**로, 줄바꿈이나 탭 같은 보이지 않는 문자를 표현한다
- **raw string(`r''`)**을 사용하면 파이썬 문자열 파서의 이스케이프 해석을 건너뛰어 패턴 작성이 간결해진다
- 정규표현식 패턴은 **항상 raw string으로 작성하는 것이 권장**된다

## 2.6 메타문자 종합 연습

> 💡 **한 줄 요약**: 지금까지 배운 메타문자들(`.`, `^`, `$`, `\A`, `\Z`, `|`, `\`)을 조합하여 실용적인 패턴을 구성해본다.

### 직관적 이해

요리를 배울 때 재료 하나하나를 익히는 것도 중요하지만, 여러 재료를 함께 사용하여 완성된 요리를 만드는 연습이 실력 향상에 훨씬 효과적입니다. 마찬가지로, 지금까지 배운 개별 메타문자들을 **조합**하여 사용하는 연습이 중요합니다. 이 Section에서는 여러 메타문자를 결합한 패턴을 살펴봅니다.

### 핵심 개념 설명: 메타문자 조합 전략

패턴을 조합할 때는 다음과 같은 전략을 사용합니다:

1. **매칭 대상을 명확히 정의**: "무엇을 찾고 싶은가?"를 먼저 한국어로 정리
2. **필요한 메타문자 선택**: 각 요구사항에 맞는 메타문자 결정
3. **점진적 구성**: 간단한 부분부터 시작하여 점차 복잡하게 확장
4. **테스트**: 매칭되어야 할 예시와 매칭되지 않아야 할 예시 모두 확인

**지금까지 배운 도구 정리**

| 도구 | 기호 | 역할 |
|------|------|------|
| 리터럴 | 일반 문자 | 자기 자신과 매칭 |
| 와일드카드 | `.` | 아무 문자 하나 (줄바꿈 제외) |
| 시작 앵커 | `^`, `\A` | 문자열 시작 위치 |
| 끝 앵커 | `$`, `\Z` | 문자열 끝 위치 |
| 선택 | `\|` | OR 논리 |
| 이스케이프 | `\` | 메타문자의 특수 의미 해제 |
| 그룹화 | `()` | 선택 범위 제한 (기초) |

### 상세 예시

> 🔍 **예시 2.6.1: 인사말 패턴**
>
> **상황**: 텍스트가 "Hello" 또는 "Hi"로 시작하는지 확인하고 싶습니다.
> ```python
> messages = [
>     "Hello, how are you?",
>     "Hi there!",
>     "hey what's up",
>     "Say Hello to everyone"
> ]
>
> for msg in messages:
>     result = re.search(r'^(Hello|Hi)', msg)
>     if result:
>         print(f"✅ 인사말로 시작: '{msg}' → {result.group()}")
>     else:
>         print(f"❌ 인사말 아님: '{msg}'")
> # ✅ 인사말로 시작: 'Hello, how are you?' → Hello
> # ✅ 인사말로 시작: 'Hi there!' → Hi
> # ❌ 인사말 아님: 'hey what's up'
> # ❌ 인사말 아님: 'Say Hello to everyone'
> ```
> **핵심 포인트**: `^`(시작 앵커) + `()`(그룹화) + `|`(선택)을 조합합니다. `^(Hello|Hi)`는 "문자열 시작이 Hello 또는 Hi"를 의미합니다.

> 🔍 **예시 2.6.2: 문장 끝 패턴**
>
> **상황**: 문장이 `.`, `!`, `?` 중 하나로 끝나는지 확인하고 싶습니다. 이 세 문자 중 `.`과 `?`는 메타문자이므로 이스케이프가 필요합니다.
> ```python
> sentences = [
>     "This is a statement.",
>     "Is this a question?",
>     "What an exclamation!",
>     "No punctuation here",
>     "Ends with comma,",
> ]
>
> for sent in sentences:
>     result = re.search(r'(\.|!|\?)$', sent)
>     if result:
>         print(f"✅ 올바른 문장 종결: '{sent}' → '{result.group()}'")
>     else:
>         print(f"❌ 종결부호 없음: '{sent}'")
> # ✅ 올바른 문장 종결: 'This is a statement.' → '.'
> # ✅ 올바른 문장 종결: 'Is this a question?' → '?'
> # ✅ 올바른 문장 종결: 'What an exclamation!' → '!'
> # ❌ 종결부호 없음: 'No punctuation here'
> # ❌ 종결부호 없음: 'Ends with comma,'
> ```
> **핵심 포인트**: `\.`(이스케이프된 마침표) + `\?`(이스케이프된 물음표) + `!`(메타문자가 아님) + `|`(선택) + `$`(끝 앵커)를 조합합니다.

> 🔍 **예시 2.6.3: 간단한 태그 매칭**
>
> **상황**: HTML에서 `<b>`, `<i>`, `<p>` 같은 한 글자짜리 태그를 찾고 싶습니다.
> ```python
> html = "<b>bold</b> and <i>italic</i> and <p>paragraph</p>"
>
> # <, 아무 문자 하나, > 순서로 매칭
> tags = re.findall(r'<.>', html)
> print(tags)  # ['<b>', '<i>', '<p>']
>
> # 닫는 태그도 포함하려면?
> all_tags = re.findall(r'<.>|</.>', html)
> print(all_tags)  # ['<b>', '<i>', '<p>']  — 아직 /를 포함한 두 글자를 매칭 못함!
>
> # 수정: 닫는 태그는 <, /, 아무 문자, > — 총 4글자
> all_tags = re.findall(r'<.>|<\/.>', html)
> print(all_tags)  # ['<b>', '</b>', '<i>', '</i>', '<p>', '</p>']
> ```
> **핵심 포인트**: `.`(와일드카드) + 리터럴(`<`, `>`, `/`) + `|`(선택)을 조합합니다. `/`는 정규표현식에서 메타문자가 아니므로 이스케이프 없이 사용할 수 있지만, 가독성을 위해 `\/`로 쓰기도 합니다 (어느 쪽이든 동일하게 동작합니다).

> 🔍 **예시 2.6.4: 점진적 패턴 구성 과정**
>
> **상황**: "파일명.확장자" 형식에서, 확장자가 정확히 세 글자인 경우만 매칭하고 싶습니다.
> ```python
> files = ["report.pdf", "data.csv", "image.jpeg", "note.py", "archive.tar"]
>
> # 1단계: 마침표 뒤에 아무 글자 세 개
> # 패턴: \.  →  마침표
> #       ...  →  아무 세 글자
> #       $    →  문자열 끝
> pattern = r'\....$'
>
> # 잠깐! \.까지 합치면 마침표 포함 4글자? 아니요.
> # \. 은 마침표 1개, ... 은 아무 문자 3개, $는 끝
> # 그러므로 "마침표 + 세글자 + 끝" = 확장자가 세 글자
>
> for f in files:
>     result = re.search(r'\....$', f)
>     if result:
>         print(f"✅ 3글자 확장자: {f} → '{result.group()}'")
>     else:
>         print(f"❌ 3글자 확장자 아님: {f}")
> # ✅ 3글자 확장자: report.pdf → '.pdf'
> # ✅ 3글자 확장자: data.csv → '.csv'
> # ❌ 3글자 확장자 아님: image.jpeg  (4글자)
> # ❌ 3글자 확장자 아님: note.py     (2글자)
> # ✅ 3글자 확장자: archive.tar → '.tar'
> ```
> **핵심 포인트**: 복잡해 보이는 패턴도 단계적으로 구성하면 이해하기 쉽습니다. `\.`(마침표) + `...`(세 글자) + `$`(끝)으로 분해하여 생각합니다.

> ⚠️ **주의**: 패턴을 조합할 때 가장 흔한 실수는 **메타문자 이스케이프를 잊는 것**입니다.
> ```python
> # 실수: 이메일에서 @와 . 이스케이프를 잊음
> re.search(r'user.name@company.com', 'userXname@companyYcom')
> # 매칭됨! — .이 아무 문자와 매칭되므로
>
> # 수정: 특수문자를 이스케이프
> re.search(r'user\.name@company\.com', 'userXname@companyYcom')
> # None — 정확한 마침표만 매칭
> ```
> 패턴을 작성할 때 항상 스스로에게 물어보세요: "이 문자가 메타문자인가? 리터럴로 쓰려는 것인가?"

### Section 요약

- 여러 메타문자를 조합하면 복잡한 매칭 조건을 표현할 수 있다
- 패턴 작성은 **요구사항 정의 → 메타문자 선택 → 점진적 구성 → 테스트** 순서를 따른다
- 실수를 줄이려면 항상 **매칭되어야 할 예시**와 **매칭되지 않아야 할 예시** 모두를 테스트한다
- 메타문자 이스케이프를 잊지 않도록 주의한다

## Chapter 핵심 요약 (Chapter Summary)

이번 Chapter에서는 정규표현식의 핵심 구성 요소인 **메타문자**를 학습했습니다. 리터럴 매칭이라는 기초 위에, 특별한 의미를 가진 메타문자들을 하나씩 쌓아올렸습니다.

- **Section 2.1 (리터럴 매칭)**: 정규표현식의 가장 기본 동작으로, 일반 문자는 자기 자신과 정확히 1:1로 매칭된다. 대소문자를 구분하며, 부분 문자열에서도 매칭된다.

- **Section 2.2 (마침표 `.`)**: 줄바꿈을 제외한 어떤 문자 하나와도 매칭되는 와일드카드이다. 진짜 마침표를 매칭하려면 `\.`로 이스케이프해야 한다.

- **Section 2.3 (앵커 `^`, `$`, `\A`, `\Z`)**: 문자가 아닌 *위치*를 매칭하는 메타문자이다. `^`은 시작, `$`은 끝을 지정하며, `\A`/`\Z`는 MULTILINE 모드에서도 항상 전체 문자열의 시작/끝만을 가리킨다.

- **Section 2.4 (선택 `|`)**: OR 논리를 패턴에 도입한다. 여러 선택지 중 하나와 매칭되면 성공이다. `|`의 범위를 제한하려면 `()`로 그룹화한다.

- **Section 2.5 (이스케이프와 raw string)**: `\`로 메타문자의 특수 의미를 해제하고, `r''`(raw string)으로 파이썬 문자열 이스케이프와의 충돌을 방지한다. 정규표현식 패턴은 항상 raw string으로 작성하는 것이 권장된다.

- **Section 2.6 (종합 연습)**: 개별 메타문자들을 조합하여 실용적인 패턴을 구성하는 연습을 했다. 점진적 패턴 구성과 테스트의 중요성을 확인했다.

> - a) `$100.00` → `$`가 끝 앵커로 해석되어 "문자열 끝이 100.00인 경우"를 의미. "100.00"이 뒤에 오지만 `$`는 "100.00" 앞이 아닌 문자열 끝에 있어야 하므로, 이 패턴은 제대로 동작하지 않음 ❌
> - b) `\$100\.00` → `\$`는 달러 기호, `100`은 리터럴, `\.`은 마침표, `00`은 리터럴. 정확히 매칭 ✅
> - c) `\$...\...` → `\$`(달러) + `...`(아무 3글자 = "100") + `\.`(마침표) + `..`(아무 2글자 = "00"). 매칭 ✅
> - d) `^\$100\.00$` → 문자열 전체가 정확히 "$100.00"인지 확인. 매칭 ✅
> </details>

>

> ```
>
> 테스트 케이스:
> ```python
> test_cases = [
>     "https://example.com",
>     "http://test.net",
>     "ftp://files.org",

> ```

> **참고**: 이 패턴에는 아직 배우지 않은 `.*`가 포함되어 있습니다. `.*`는 "아무 문자가 0개 이상 반복"을 의미합니다. (Chapter 4에서 상세 학습)
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 패턴을 부분별로 분해하여 각 부분의 역할을 파악합니다.
>
> **패턴 분해**:
> ```
> ^          → 문자열 시작
> (http|https) → "http" 또는 "https"
> ://        → 리터럴 "://"
> .*         → 아무 문자 0개 이상 (중간 내용 전체)
> \.         → 리터럴 마침표
> (com|net|org) → "com" 또는 "net" 또는 "org"
> $          → 문자열 끝
> ```
>
> **각 테스트 케이스 분석**:
>
> 1. `"https://example.com"` → ✅ 매칭
>    - `^(https)` ✅, `://` ✅, `.*` = `example`, `\.` = `.`, `(com)$` ✅
>
> 2. `"http://test.net"` → ✅ 매칭
>    - `^(http)` ✅, `://` ✅, `.*` = `test`, `\.` = `.`, `(net)$` ✅
>
> 3. `"ftp://files.org"` → ❌ 매칭 실패
>    - `^(http|https)` ❌ — "ftp"는 선택지에 없음
>
> 4. `"https://my.site.com"` → ✅ 매칭
>    - `^(https)` ✅, `://` ✅, `.*` = `my.site`, `\.` = 마지막 `.`, `(com)$` ✅
>    - `.*`가 탐욕적으로 매칭하여 "my.site"까지 소비하고, 마지막 `.com`만 남김
>
> 5. `"http://example.com/page"` → ❌ 매칭 실패
>    - `(com|net|org)$` ❌ — 문자열이 "page"로 끝남 ("com" 뒤에 "/page"가 더 있음)
>
> 6. `"https://example.community"` → ❌ 매칭 실패
>    - 마지막이 "community"인데, `(com|net|org)$` 조건에서 "community"는 "com"으로 시작하지만, `$`(끝 앵커) 때문에 "com" 뒤에 "munity"가 남아 매칭 실패
>
> **확장 생각**: 4번 케이스가 흥미롭습니다. `.*`는 "가능한 많이" 매칭하는 탐욕적 전략을 사용하기 때문에, `my.site`까지 먹어들어가다가 `\.(com|net|org)$`를 만족시키기 위해 일부를 돌려줍니다(백트래킹). 이런 탐욕적/비탐욕적 매칭과 백트래킹은 Chapter 4와 Chapter 15에서 자세히 다룹니다.

>
> # 소괄호 안: "매개변수" = 4글자
> parens = re.findall(r'\(.....\)', text)  # 잠깐, "매개변수"는 4글자? 아니면 3글자?
> # "매개변수"는 4글자입니다: 매, 개, 변, 수
> # 하지만 \(와 \) 자체는 이스케이프된 리터럴 괄호
>
> # 각각의 괄호 쌍을 개별 패턴으로
> p1 = re.findall(r'\(.....\)', text)  # 5글자? 다시 세어봅시다.
> # 아, "매개변수" = 매(1) 개(2) 변(3) 수(4) → 4글자
>
> result1 = re.findall(r'\(....\)', text)   # (매개변수) → \( + 4글자 + \)
> result2 = re.findall(r'\[...\]', text)    # [인덱스] → \[ + 3글자 + \]
> result3 = re.findall(r'\{.\}', text)      # {키} → \{ + 1글자 + \}
>
> print(result1)  # ['(매개변수)']
> print(result2)  # ['[인덱스]']
> print(result3)  # ['{키}']
>
> # 모두 합치기
> all_brackets = result1 + result2 + result3
> print(all_brackets)  # ['(매개변수)', '[인덱스]', '{키}']
> ```
>
> **한계 인식**: 이 방법은 괄호 안의 문자 수를 미리 알아야 한다는 큰 제약이 있습니다. 실전에서는 수량자(`*`, `+`)를 사용하여 `\(.*?\)`(Chapter 4)와 같이 작성합니다. 현재 단계에서는 이 한계를 인식하는 것 자체가 학습의 핵심입니다.
>
> **확장 생각**: 이 문제가 보여주는 것은 **수량자 없이는 "가변 길이"를 표현할 수 없다**는 정규표현식의 근본적인 필요성입니다. `.`은 정확히 1글자만 매칭하므로, 내용물의 길이를 모를 때는 무력합니다. 이것이 Chapter 4의 수량자가 존재하는 이유입니다.
> </details>
