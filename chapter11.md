## 11.1 파이썬과 유니코드

> 💡 **한 줄 요약**: 파이썬 3의 문자열은 기본적으로 유니코드이며, 정규표현식 엔진도 유니코드를 기본 모드로 동작합니다.

### 직관적 이해

전 세계에는 수천 개의 문자 체계가 있습니다. 영어의 알파벳 26자, 한국어의 수천 개 한글 음절, 일본어의 히라가나·카타카나·한자, 아랍어, 태국어… 이 모든 문자를 하나의 통일된 체계로 표현하려는 시도가 바로 **유니코드(Unicode)**입니다.

컴퓨터 초창기에는 ASCII라는 체계가 표준이었습니다. ASCII는 영문 알파벳, 숫자, 기본 특수문자 등 128개의 문자만 표현할 수 있었습니다. 이것으로는 한국어는 물론이고 서유럽 언어의 악센트 기호(é, ñ)조차 제대로 표현할 수 없었습니다.

유니코드는 이 문제를 해결하기 위해 탄생했습니다. 현재 유니코드 표준은 약 15만 자 이상의 문자를 포함하며, 한글, 한자, 이모지까지 모두 아우릅니다.

**왜 정규표현식에서 유니코드가 중요할까요?**

여러분이 Chapter 3에서 배운 `\w`(단어 문자)를 떠올려 봅시다. `\w`가 "영문 알파벳과 숫자, 밑줄"만 매칭한다면, 한국어 텍스트에서 "단어"를 찾는 건 불가능합니다. 그런데 파이썬 3에서 `\w`는 한글도 매칭합니다! 이것이 바로 유니코드 모드 덕분입니다.

### 핵심 개념 설명

> 📝 **유니코드(Unicode)**: 전 세계의 모든 문자를 하나의 통일된 번호 체계로 표현하기 위한 국제 표준. 각 문자에 고유한 **코드포인트(code point)**를 부여합니다.

> 📝 **코드포인트(Code Point)**: 유니코드 표준에서 각 문자에 부여된 고유한 번호. 보통 `U+XXXX` 형식으로 표기합니다. 예를 들어 `U+AC00`은 한글 '가', `U+0041`은 영문 대문자 'A'입니다.

> 📝 **UTF-8**: 유니코드 코드포인트를 실제 바이트로 인코딩하는 방식 중 하나. 파이썬 3의 소스 파일 기본 인코딩이며, 웹에서 가장 널리 사용됩니다.

**파이썬 3의 문자열과 유니코드**

파이썬 3에서 `str` 타입은 유니코드 문자열입니다. 이것은 파이썬 2와의 가장 큰 차이점 중 하나입니다.

```python
# 파이썬 3에서 문자열은 유니코드
text = "Hello, 안녕하세요! 🎉"
print(type(text))   # <class 'str'>
print(len(text))    # 15 (문자 수 기준)

# 각 문자의 유니코드 코드포인트 확인
print(ord('A'))     # 65 (U+0041)
print(ord('가'))    # 44032 (U+AC00)
print(hex(ord('가')))  # '0xac00'

# 코드포인트로 문자 생성
print(chr(44032))   # 가
print(chr(0xAC00))  # 가
```

**파이썬 정규표현식의 기본 유니코드 모드**

파이썬 3의 `re` 모듈은 **기본적으로 유니코드 모드**로 동작합니다. 이것은 아무런 플래그를 지정하지 않아도 `\w`, `\d`, `\s`, `\b` 같은 사전 정의 문자 클래스가 유니코드 전체 범위를 고려한다는 뜻입니다.

```python
import re

# 파이썬 3에서 \w는 유니코드 "단어 문자"를 매칭
result = re.findall(r'\w+', "Hello 안녕하세요 World 세계")
print(result)  # ['Hello', '안녕하세요', 'World', '세계']
# → 한글도 \w에 매칭됩니다!
```

> 🔗 **연결**: `\w`, `\d`, `\s`, `\b` 등의 사전 정의 문자 클래스는 Chapter 3에서 배웠습니다. 당시에는 주로 영문 맥락에서 다루었지만, 이 Chapter에서 유니코드 환경에서의 동작을 자세히 살펴봅니다.

### 상세 예시

> 🔍 **예시 11.1.1: 유니코드 문자의 코드포인트 확인**
>
> **상황**: 다양한 문자의 유니코드 코드포인트를 확인하고, 문자 간의 순서 관계를 이해합니다.
>
> ```python
> # 다양한 문자의 코드포인트 확인
> chars = ['A', 'Z', 'a', 'z', '0', '9', '가', '힣', 'ㄱ', 'ㅎ', 'ㅏ', 'ㅣ']
> for c in chars:
>     print(f"'{c}' → U+{ord(c):04X} (10진수: {ord(c)})")
> ```
>
> **출력**:
> ```
> 'A' → U+0041 (10진수: 65)
> 'Z' → U+005A (10진수: 90)
> 'a' → U+0061 (10진수: 97)
> 'z' → U+007A (10진수: 122)
> '0' → U+0030 (10진수: 48)
> '9' → U+0039 (10진수: 57)
> '가' → U+AC00 (10진수: 44032)
> '힣' → U+D7A3 (10진수: 55203)
> 'ㄱ' → U+3131 (10진수: 12593)
> 'ㅎ' → U+314E (10진수: 12622)
> 'ㅏ' → U+314F (10진수: 12623)
> 'ㅣ' → U+3163 (10진수: 12643)
> ```
>
> **핵심 포인트**: 유니코드에서 문자는 코드포인트 순서대로 정렬됩니다. 한글 음절 '가'(U+AC00)부터 '힣'(U+D7A3)까지는 연속된 범위를 형성하며, 이것이 `[가-힣]` 같은 범위 표현이 가능한 이유입니다.

> 🔍 **예시 11.1.2: 유니코드 모드의 기본 동작 확인**
>
> **상황**: 파이썬 3에서 정규표현식이 유니코드 문자를 어떻게 처리하는지 확인합니다.
>
> ```python
> import re
>
> text = "이름: 홍길동, Age: 25, メール: test@example.com"
>
> # \w+는 유니코드 단어 문자를 모두 매칭
> words = re.findall(r'\w+', text)
> print(words)
> # ['이름', '홍길동', 'Age', '25', 'メール', 'test', 'example', 'com']
>
> # 한글, 영문, 일본어 모두 \w로 매칭됨을 확인
> ```
>

> print(len("가"))                    # 1 (문자 수)

> print(re.findall(r'.', "가나다"))   # ['가', '나', '다'] — 3개의 문자
> ```

- 유니코드는 전 세계 문자를 하나의 체계로 표현하는 국제 표준이며, 각 문자에 고유한 코드포인트를 부여합니다
- 파이썬 3의 `str` 타입은 유니코드 문자열이며, `re` 모듈도 기본적으로 유니코드 모드로 동작합니다
- `ord()` 함수로 문자의 코드포인트를, `chr()` 함수로 코드포인트에 해당하는 문자를 얻을 수 있습니다
- 정규표현식은 바이트가 아닌 **문자** 단위로 동작합니다

## 11.2 유니코드 모드에서의 문자 클래스

> 💡 **한 줄 요약**: `\w`, `\d`, `\s`, `\b`는 유니코드 모드(기본값)와 ASCII 모드에서 매칭 범위가 크게 다릅니다.

### 직관적 이해

Chapter 3에서 `\w`는 "단어 문자"를 매칭한다고 배웠습니다. 그런데 "단어 문자"의 범위는 **어떤 모드로 동작하느냐**에 따라 완전히 달라집니다.

비유하자면, `\w`는 "출입증을 가진 사람"과 같습니다. ASCII 모드에서는 "영어 이름의 출입증만 인정"하는 좁은 기준이고, 유니코드 모드에서는 "전 세계 어떤 언어의 이름이든 출입증으로 인정"하는 넓은 기준입니다.

### 핵심 개념 설명

> 🔗 **연결**: Chapter 10에서 `re.ASCII` 플래그와 인라인 플래그 `(?a)`를 배웠습니다. 이 플래그가 `\w`, `\d`, `\s`, `\b`의 동작을 어떻게 변경하는지 이 Section에서 자세히 살펴봅니다.

**`\w`의 두 가지 얼굴**

| 모드 | `\w`의 매칭 범위 | 동등한 문자 클래스 |
|------|-----------------|-------------------|
| **유니코드 모드** (기본) | 유니코드의 모든 "단어 문자": 모든 언어의 글자, 숫자, 밑줄 | (수만 개의 문자) |
| **ASCII 모드** (`re.ASCII`) | `[a-zA-Z0-9_]` 만 | 63개 문자 |

```python
import re

text = "Hello_안녕_42_こんにちは_Ω"

# 유니코드 모드 (기본)
print(re.findall(r'\w+', text))
# ['Hello_안녕_42_こんにちは_Ω']
# → 전체가 하나의 "단어"로 매칭! (한글, 일본어, 그리스 문자 모두 \w에 포함)

# ASCII 모드
print(re.findall(r'\w+', text, re.ASCII))
# ['Hello_', '42_', '_']
# → 영문, 숫자, 밑줄만 매칭. 한글·일본어·그리스 문자는 \w에서 제외
```

**`\d`의 두 가지 얼굴**

이것은 많은 사람이 놀라는 부분입니다. 유니코드 모드에서 `\d`는 아라비아 숫자 `0-9`뿐 아니라 **전 세계의 다양한 숫자 표기**도 매칭합니다.

| 모드 | `\d`의 매칭 범위 |
|------|-----------------|
| **유니코드 모드** (기본) | 모든 유니코드 숫자: 0-9, ٠-٩(아랍 숫자), ०-९(데바나가리 숫자), ꯰-꯹ 등 |
| **ASCII 모드** (`re.ASCII`) | `[0-9]`만 |

```python
import re

# 다양한 숫자 표기가 포함된 텍스트
text = "가격: 100원, ١٢٣, ₹४५६"

# 유니코드 모드 (기본): 다양한 숫자 체계를 모두 매칭
print(re.findall(r'\d+', text))
# ['100', '١٢٣', '४५६']

# ASCII 모드: 0-9만 매칭
print(re.findall(r'\d+', text, re.ASCII))
# ['100']
```

> 💬 **참고**: 실무에서 "숫자"라 하면 보통 아라비아 숫자 0-9를 의미합니다. 유니코드 모드의 `\d`가 다른 문자 체계의 숫자까지 매칭하는 것은 예상치 못한 결과를 낳을 수 있으므로, **아라비아 숫자만 매칭하고 싶다면 `[0-9]`를 명시적으로 사용**하는 것이 안전합니다.

**`\s`의 두 가지 얼굴**

| 모드 | `\s`의 매칭 범위 |
|------|-----------------|
| **유니코드 모드** (기본) | ASCII 공백 + 유니코드 공백 문자(예: 줄바꿈 없는 공백 `\u00A0`, 전각 공백 `\u3000` 등) |
| **ASCII 모드** (`re.ASCII`) | `[ \t\n\r\f\v]`만 |

```python
import re

# 전각 공백(\u3000)이 포함된 텍스트
text = "이름:\u3000홍길동"  # "이름:　홍길동" (전각 공백)

# 유니코드 모드: 전각 공백도 \s로 매칭
print(re.findall(r'\S+', text))
# ['이름:', '홍길동'] — 전각 공백이 구분자로 인식됨

# ASCII 모드: 전각 공백은 \s로 매칭되지 않음
print(re.findall(r'\S+', text, re.ASCII))
# ['이름:\u3000홍길동'] — 전각 공백이 비공백으로 취급됨
```

**`\b`의 두 가지 얼굴**

> 🔗 **연결**: Chapter 3에서 `\b`는 "단어 문자와 비단어 문자의 경계"라고 배웠습니다.

`\b`의 동작은 `\w`의 정의에 의존합니다. `\w`의 범위가 달라지면, "단어의 경계"도 달라집니다.

```python
import re

text = "나는 student입니다"

# 유니코드 모드: "나는", "student입니다" 등이 하나의 단어로 인식될 수 있음
print(re.findall(r'\b\w+\b', text))
# ['나는', 'student입니다']
# → 한글도 \w에 포함되므로, 공백이 단어 경계가 됨

# ASCII 모드: 한글은 \w가 아니므로 영문 단어만 추출
print(re.findall(r'\b\w+\b', text, re.ASCII))
# ['student']
# → 한글은 비단어 문자이므로, 한글-영문 전환점이 단어 경계가 됨
```

**ASCII 모드 활성화 방법**

> 🔗 **연결**: Chapter 10에서 배운 플래그 적용 방법을 활용합니다.

```python
import re

text = "가격 1000원"

# 방법 1: 함수 인자로 플래그 전달
re.findall(r'\w+', text, re.ASCII)

# 방법 2: compile()에 플래그 전달
pattern = re.compile(r'\w+', re.ASCII)
pattern.findall(text)

# 방법 3: 인라인 플래그
re.findall(r'(?a)\w+', text)
```

> 🔍 **예시 11.2.2: 숫자 추출 시 모드 차이**
>

>

>

> 🔬 **Deep Dive**: `\w`가 매칭하는 유니코드 카테고리
>
> *이 부분은 심화 내용입니다. 처음 학습 시 건너뛰어도 됩니다.*
>
> 유니코드 표준에서 각 문자는 **카테고리(General Category)**가 지정되어 있습니다. 파이썬 3의 `\w`는 다음 유니코드 카테고리에 해당하는 문자를 매칭합니다:
>
> - `L` (Letter): 모든 언어의 글자 — 한글, 한자, 라틴, 키릴, 그리스 문자 등
> - `N` (Number): 모든 숫자 — 아라비아 숫자, 로마 숫자, 기타 숫자 기호
> - `_` (밑줄 문자)
> - 일부 연결 문자(`Pc` 카테고리 등)
>
> ```python
> import unicodedata
>
> # 문자의 유니코드 카테고리 확인
> for c in ['가', 'A', '1', '!', '_', '٧']:
>     print(f"'{c}' → 카테고리: {unicodedata.category(c)}, 이름: {unicodedata.name(c, 'N/A')}")
> ```
>
> ```
> '가' → 카테고리: Lo, 이름: HANGUL SYLLABLE GA
> 'A' → 카테고리: Lu, 이름: LATIN CAPITAL LETTER A
> '1' → 카테고리: Nd, 이름: DIGIT ONE
> '!' → 카테고리: Po, 이름: EXCLAMATION MARK
> '_' → 카테고리: Pc, 이름: LOW LINE
> '٧' → 카테고리: Nd, 이름: ARABIC-INDIC DIGIT SEVEN
> ```
>
> `Lo`는 "Letter, other"로 한글을 포함하고, `Lu`는 "Letter, uppercase"로 대문자를 의미합니다. 이 카테고리 정보는 Chapter 17에서 `regex` 모듈의 `\p{L}` 구문을 배울 때 다시 활용됩니다.

### 흔한 실수와 주의사항

> ⚠️ **주의**: `\d`와 `[0-9]`는 같지 않습니다
>
> 유니코드 모드에서 `\d`는 `[0-9]`보다 넓은 범위를 매칭합니다. 전화번호, 우편번호 등 **아라비아 숫자 0-9만** 필요한 경우에는 `\d` 대신 `[0-9]`를 사용하세요.
>
> ```python
> import re
> # \d는 유니코드 숫자까지 매칭
> print(re.match(r'^\d+$', '٧٨٩'))   # <re.Match object> — 매칭됨!
> # [0-9]는 아라비아 숫자만 매칭
> print(re.match(r'^[0-9]+$', '٧٨٩'))  # None — 매칭 안 됨
> ```

> ⚠️ **주의**: `re.ASCII` 플래그의 영향 범위
>
> `re.ASCII` 플래그는 `\w`, `\W`, `\d`, `\D`, `\s`, `\S`, `\b`, `\B` 모두에 영향을 줍니다. 패턴의 일부에만 ASCII 모드를 적용하고 싶다면, 인라인 플래그를 그룹 단위로 적용해야 합니다. 하지만 파이썬의 `re` 모듈에서는 그룹 범위 인라인 플래그가 제한적이므로, 보통은 패턴 전체에 적용합니다.

### Section 요약

- 유니코드 모드(기본)에서 `\w`는 전 세계 언어의 글자·숫자·밑줄을, ASCII 모드에서는 `[a-zA-Z0-9_]`만 매칭합니다
- `\d`는 유니코드 모드에서 아라비아 숫자 외의 숫자 문자도 매칭하므로, 0-9만 필요하면 `[0-9]`를 사용하세요
- `\s`는 유니코드 모드에서 전각 공백 등 추가 공백 문자도 매칭합니다
- `\b`의 동작은 `\w`의 정의에 의존하므로, 모드에 따라 단어 경계 위치가 달라집니다
- `re.ASCII` 플래그나 인라인 `(?a)`로 ASCII 모드를 활성화할 수 있습니다

## 11.3 한글 매칭 패턴

> 💡 **한 줄 요약**: 한글은 완성형 음절(`[가-힣]`)과 자모(`[ㄱ-ㅎㅏ-ㅣ]`)로 나뉘며, 각각의 유니코드 범위를 알아야 정확한 매칭이 가능합니다.

### 직관적 이해

한글은 세계에서 가장 과학적인 문자 체계 중 하나로 평가받습니다. 자음과 모음을 조합하여 음절 글자를 만드는 독특한 구조를 가지고 있는데, 이 구조가 유니코드에도 반영되어 있습니다.

유니코드에서 한글은 크게 두 가지 영역으로 나뉩니다:

1. **한글 호환 자모(Hangul Compatibility Jamo)**: 'ㄱ', 'ㄴ', 'ㅏ', 'ㅓ' 같이 낱개로 쓰이는 자모
2. **한글 음절(Hangul Syllables)**: '가', '나', '힣' 같이 조합된 완성형 음절

이 둘은 유니코드에서 **서로 다른 코드포인트 범위**에 위치합니다. 따라서 "모든 한글"을 매칭하려면, 두 범위를 모두 고려해야 합니다.

### 핵심 개념 설명

**한글 유니코드 블록 정리**

| 범위 | 유니코드 범위 | 설명 | 글자 수 | 정규표현식 표현 |
|------|-------------|------|---------|---------------|
| 한글 호환 자모 | U+3131 ~ U+3163 | ㄱ~ㅎ, ㅏ~ㅣ | 51개 | `[ㄱ-ㅎㅏ-ㅣ]` 또는 `[\u3131-\u3163]` |
| 한글 음절 | U+AC00 ~ U+D7A3 | 가~힣 (완성형) | 11,172개 | `[가-힣]` 또는 `[\uAC00-\uD7A3]` |

> 📝 **한글 음절(Hangul Syllable)**: 초성 + 중성 + (종성)이 조합된 완성형 글자. '가', '나', '다', '한', '글' 등이 여기에 해당합니다. 유니코드에서 '가'(U+AC00)부터 '힣'(U+D7A3)까지 연속 배치되어 있습니다.

> 📝 **한글 호환 자모(Hangul Compatibility Jamo)**: 낱개로 사용되는 자음과 모음. 'ㄱ', 'ㄴ', 'ㅁ', 'ㅏ', 'ㅓ' 등이 여기에 해당합니다. 키보드에서 자음/모음만 단독으로 입력했을 때 이 영역의 문자가 됩니다.

**완성형 음절 매칭: `[가-힣]`**

가장 흔하게 사용하는 한글 매칭 패턴입니다.

```python
import re

text = "Hello 안녕하세요 2025년 새해 복 많이 받으세요!"

# 한글 음절만 매칭
hangul_syllables = re.findall(r'[가-힣]+', text)
print(hangul_syllables)
# ['안녕하세요', '년', '새해', '복', '많이', '받으세요']
```

> 🔗 **연결**: Chapter 3에서 범위 표현 `[a-z]`를 배웠습니다. `[가-힣]`도 같은 원리입니다. 대괄호 안에서 하이픈(`-`)은 범위를 나타내며, 유니코드 코드포인트 순서에 따라 '가'(U+AC00)부터 '힣'(U+D7A3) 사이의 모든 문자를 매칭합니다.

**자모 매칭: `[ㄱ-ㅎㅏ-ㅣ]`**

때로는 낱개 자모도 매칭해야 합니다. 인터넷 글이나 채팅에서 "ㅋㅋㅋ", "ㅎㅎ", "ㅠㅠ" 같은 표현을 처리할 때 필요합니다.

```python
import re

text = "ㅋㅋㅋ 진짜 웃겨ㅎㅎ 너무 슬프다ㅠㅠ"

# 낱개 자모 매칭
jamo = re.findall(r'[ㄱ-ㅎㅏ-ㅣ]+', text)
print(jamo)
# ['ㅋㅋㅋ', 'ㅎㅎ', 'ㅠㅠ']

# 한글 음절만 매칭 (자모 제외)
syllables = re.findall(r'[가-힣]+', text)
print(syllables)
# ['진짜', '웃겨', '너무', '슬프다']
```

**모든 한글 매칭: 음절 + 자모**

한글 음절과 자모를 모두 포함하려면, 두 범위를 하나의 문자 클래스에 결합합니다.

```python
import re

text = "ㅋㅋㅋ 안녕하세요! Hello ㅎㅎ 반가워요"

# 모든 한글 (음절 + 자모)
all_hangul = re.findall(r'[가-힣ㄱ-ㅎㅏ-ㅣ]+', text)
print(all_hangul)
# ['ㅋㅋㅋ', '안녕하세요', 'ㅎㅎ', '반가워요']
```

**한글 음절의 내부 구조 (이해를 위한 배경 지식)**

한글 음절은 **초성(19개) × 중성(21개) × (종성(27개) + 받침 없음(1개))** = 19 × 21 × 28 = **11,172개**로 구성됩니다. 유니코드에서 이들은 체계적으로 배열되어 있어서, 코드포인트로부터 초성·중성·종성을 계산할 수 있습니다.

```python
# 한글 음절의 초성·중성·종성 분해 (참고용)
def decompose_hangul(char):
    """완성형 한글 음절을 초성, 중성, 종성으로 분해"""
    code = ord(char) - 0xAC00
    if code < 0 or code > 11171:
        return None
    
    # 초성(19), 중성(21), 종성(28)
    cho = code // (21 * 28)
    jung = (code % (21 * 28)) // 28
    jong = code % 28
    
    cho_list = list("ㄱㄲㄴㄷㄸㄹㅁㅂㅃㅅㅆㅇㅈㅉㅊㅋㅌㅍㅎ")
    jung_list = list("ㅏㅐㅑㅒㅓㅔㅕㅖㅗㅘㅙㅚㅛㅜㅝㅞㅟㅠㅡㅢㅣ")
    jong_list = [''] + list("ㄱㄲㄳㄴㄵㄶㄷㄹㄺㄻㄼㄽㄾㄿㅀㅁㅂㅄㅅㅆㅇㅈㅊㅋㅌㅍㅎ")
    
    return cho_list[cho], jung_list[jung], jong_list[jong]

print(decompose_hangul('한'))  # ('ㅎ', 'ㅏ', 'ㄴ')
print(decompose_hangul('글'))  # ('ㄱ', 'ㅡ', 'ㄹ')
print(decompose_hangul('가'))  # ('ㄱ', 'ㅏ', '')  — 받침 없음
```

> 💬 **참고**: 위의 분해 함수는 한글의 구조를 이해하기 위한 참고용입니다. 정규표현식에서 직접 초성·중성·종성별로 매칭하는 것은 `re` 모듈만으로는 어렵습니다. 이런 고급 한글 처리는 별도의 한글 처리 라이브러리를 활용합니다.

**특정 초성으로 시작하는 음절 매칭**

한글 음절이 유니코드에서 체계적으로 배열되어 있다는 점을 활용하면, 특정 초성으로 시작하는 음절의 범위를 계산할 수 있습니다.

```python
import re

# 'ㅎ'으로 시작하는 음절: '하'(U+D558) ~ '힣'(U+D7A3)
# 계산: 'ㅎ'은 초성 인덱스 18, 18 × 21 × 28 = 10584, 0xAC00 + 10584 = 0xD558
text = "한글 학습 화이팅 행복 나무 다리 하늘"
h_words = re.findall(r'[하-힣]+', text)
print(h_words)
# ['한', '학', '화이팅', '행', '하늘']
# → 'ㅎ' 초성으로 시작하는 모든 음절
```

> 🔍 **예시 11.3.2: 한글 + 영문 혼합 텍스트에서 한글 단어만 추출**
>
> **상황**: 한국어와 영어가 섞인 텍스트에서 한글 단어만 추출합니다.
>
> ```python
> import re
>
> text = """
> Python은 가장 인기 있는 programming language 중 하나입니다.
> 2025년 기준 developer survey에서 1위를 차지했습니다.
> AI와 machine learning 분야에서 특히 널리 사용됩니다.
> """
>
> # 한글 단어 추출 (한글 음절이 1개 이상 연속)
> korean_words = re.findall(r'[가-힣]+', text)
> print(korean_words)
> # ['은', '가장', '인기', '있는', '중', '하나입니다',
> #  '년', '기준', '에서', '위를', '차지했습니다',
> #  '와', '분야에서', '특히', '널리', '사용됩니다']
>
> # 2글자 이상의 한글 단어만 추출
> korean_words_2plus = re.findall(r'[가-힣]{2,}', text)
> print(korean_words_2plus)
> # ['가장', '인기', '있는', '하나입니다', '기준', '에서',
> #  '위를', '차지했습니다', '분야에서', '특히', '널리', '사용됩니다']
> ```
>
> **핵심 포인트**: `[가-힣]+`로 한글 연속 구간을 추출하고, `{2,}` 같은 수량자로 최소 길이를 지정할 수 있습니다.

> 🔍 **예시 11.3.3: 인터넷 용어의 자모 패턴 처리**
>
> **상황**: 채팅이나 SNS 텍스트에서 자모로 된 감탄사를 정리합니다.
>
> ```python
> import re
>
> messages = [
>     "ㅋㅋㅋㅋ 그거 진짜 웃기다",
>     "ㅎㅎㅎ 맞아요",
>     "ㅠㅠㅠㅠ 슬프다",
>     "ㅇㅇ 알겠어",
>     "안녕하세요!",
> ]
>
> for msg in messages:
>     # 자모 반복 (2개 이상 연속) 제거
>     cleaned = re.sub(r'[ㄱ-ㅎㅏ-ㅣ]{2,}', '', msg).strip()
>     print(f"원본: {msg}")
>     print(f"정리: {cleaned}")
>     print()
> ```
>
> **출력**:
> ```
> 원본: ㅋㅋㅋㅋ 그거 진짜 웃기다
> 정리: 그거 진짜 웃기다
>
> 원본: ㅎㅎㅎ 맞아요
> 정리: 맞아요
>
> 원본: ㅠㅠㅠㅠ 슬프다
> 정리: 슬프다
>
> 원본: ㅇㅇ 알겠어
> 정리: 알겠어
>
> 원본: 안녕하세요!
> 정리: 안녕하세요!
> ```
>
> **핵심 포인트**: `[ㄱ-ㅎㅏ-ㅣ]{2,}`는 2개 이상 연속된 자모를 매칭합니다. 이를 `re.sub()`으로 빈 문자열로 치환하면 자모 감탄사가 제거됩니다.

### 흔한 실수와 주의사항

> ⚠️ **주의**: `[가-힣]`과 `[ㄱ-ㅎㅏ-ㅣ]`은 다릅니다
>
> `[가-힣]`은 **완성형 음절**만 매칭하고, `[ㄱ-ㅎㅏ-ㅣ]`은 **낱개 자모**만 매칭합니다. 둘을 혼동하면 매칭 누락이 발생합니다.
>
> ```python
> import re
> print(re.findall(r'[가-힣]+', 'ㅋㅋㅋ'))   # [] — 빈 결과! 자모는 음절이 아님
> print(re.findall(r'[ㄱ-ㅎ]+', '안녕'))      # [] — 빈 결과! 음절은 자모가 아님
> ```

> ⚠️ **주의**: 한글 자모 범위 지정 시 순서에 주의
>
> 한글 호환 자모에서 자음(ㄱ~ㅎ)과 모음(ㅏ~ㅣ)은 코드포인트가 연속되지 않습니다. 따라서 `[ㄱ-ㅣ]`처럼 하나의 범위로 쓰면, 자음과 모음 사이에 있는 다른 문자까지 포함될 수 있습니다. **`[ㄱ-ㅎㅏ-ㅣ]`처럼 자음과 모음 범위를 분리하여 작성하는 것이 안전합니다.**
>
> 사실 U+3131(ㄱ)부터 U+3163(ㅣ)까지의 범위에는 자음(ㄱ~ㅎ)과 모음(ㅏ~ㅣ)이 순서대로 들어있어 `[ㄱ-ㅣ]`도 동작하지만, 의도를 명확히 하기 위해 `[ㄱ-ㅎㅏ-ㅣ]`로 분리하여 쓰는 것을 권장합니다.

### Section 요약

- 한글 완성형 음절은 `[가-힣]` (U+AC00~U+D7A3, 11,172자)로 매칭합니다
- 한글 낱개 자모는 `[ㄱ-ㅎㅏ-ㅣ]` (U+3131~U+3163)로 매칭합니다
- 모든 한글을 포함하려면 `[가-힣ㄱ-ㅎㅏ-ㅣ]`로 두 범위를 결합합니다
- 한글 음절은 초성(19) × 중성(21) × 종성(28) = 11,172개로 체계적으로 배열되어 있습니다
- 인터넷 한국어 텍스트에서는 자모 단독 사용(ㅋㅋ, ㅎㅎ)이 흔하므로, 자모 범위도 함께 고려해야 합니다

## 11.4 유니코드 코드포인트와 범위

> 💡 **한 줄 요약**: `\uXXXX` 이스케이프를 사용하면 코드포인트로 문자를 직접 지정할 수 있으며, 이를 통해 다양한 유니코드 블록과 범위를 정밀하게 매칭할 수 있습니다.

### 직관적 이해

앞서 `[가-힣]`이라는 패턴을 배웠습니다. 이것이 작동하는 이유는 '가'와 '힣'이 유니코드에서 연속된 범위를 차지하기 때문입니다. 그런데 만약 키보드에 없는 문자나, 매우 특정한 유니코드 범위를 매칭해야 한다면 어떻게 할까요?

이때 **유니코드 코드포인트 이스케이프**(`\uXXXX`)를 사용합니다. 코드포인트를 직접 지정함으로써, 어떤 유니코드 문자든 정확하게 참조할 수 있습니다.

### 핵심 개념 설명

**`\uXXXX` 이스케이프**

파이썬 문자열과 정규표현식에서 `\uXXXX`(X는 16진수 4자리)를 사용하면, 해당 코드포인트의 유니코드 문자를 지정할 수 있습니다.

```python
import re

# \uXXXX를 사용한 문자 지정
print('\u0041')   # A
print('\uAC00')   # 가
print('\uD7A3')   # 힣

# 정규표현식에서의 활용
# [가-힣]과 동일한 패턴
pattern1 = r'[가-힣]+'
pattern2 = r'[\uAC00-\uD7A3]+'

text = "Hello 안녕하세요"
print(re.findall(pattern1, text))  # ['안녕하세요']
print(re.findall(pattern2, text))  # ['안녕하세요']
```

> 💬 **참고**: `\uXXXX`는 4자리 16진수로 **기본 다국어 평면(BMP)** 범위(U+0000~U+FFFF)의 문자를 지정합니다. 이모지처럼 U+FFFF를 넘는 문자는 `\UXXXXXXXX`(8자리) 또는 파이썬 문자열에서 직접 문자를 사용해야 합니다.

**주요 유니코드 블록과 범위**

| 유니코드 블록 | 범위 | 정규표현식 | 설명 |
|-------------|------|-----------|------|
| 기본 라틴 | U+0000~U+007F | `[\u0000-\u007F]` | ASCII 문자 |
| 한글 호환 자모 | U+3131~U+3163 | `[\u3131-\u3163]` | ㄱ~ㅎ, ㅏ~ㅣ |
| 한글 음절 | U+AC00~U+D7A3 | `[\uAC00-\uD7A3]` | 가~힣 |
| CJK 통합 한자 | U+4E00~U+9FFF | `[\u4E00-\u9FFF]` | 한중일 공통 한자 |
| 히라가나 | U+3040~U+309F | `[\u3040-\u309F]` | 일본어 히라가나 |
| 카타카나 | U+30A0~U+30FF | `[\u30A0-\u30FF]` | 일본어 카타카나 |
| 전각 문자 | U+FF01~U+FF5E | `[\uFF01-\uFF5E]` | 전각 영숫자 및 기호 |

```python
import re

text = "Python은 좋은 언語(언어)다. 日本語のPython. 한자: 韓國"

# 한자만 추출
chinese_chars = re.findall(r'[\u4E00-\u9FFF]+', text)
print(chinese_chars)  # ['語', '日本語', '韓國']

# 히라가나만 추출
hiragana = re.findall(r'[\u3040-\u309F]+', text)
print(hiragana)  # ['の']

# 한글 음절만 추출
korean = re.findall(r'[\uAC00-\uD7A3]+', text)
print(korean)  # ['은', '좋은', '언', '언어', '다', '한자']
```

**BMP를 넘는 문자: 이모지 등**

이모지, 고대 문자 등 U+FFFF를 넘는 코드포인트의 문자는 **보조 다국어 평면(SMP, Supplementary Multilingual Plane)**에 위치합니다. 이런 문자는 `\UXXXXXXXX`(8자리)로 지정하거나, 파이썬 문자열에서 직접 입력합니다.

```python
import re

text = "축하합니다! 🎉🎊 Happy New Year 🥳"

# 이모지 범위 매칭 (일부 범위만 예시)
# 이모지는 여러 유니코드 블록에 걸쳐 있어, 완전한 매칭은 복잡합니다
emoji_pattern = re.compile(
    r'[\U0001F600-\U0001F64F'    # 이모티콘 (Emoticons)
    r'\U0001F300-\U0001F5FF'     # 기호 및 그림 (Misc Symbols and Pictographs)
    r'\U0001F680-\U0001F6FF'     # 교통 및 지도 (Transport and Map)
    r'\U0001F900-\U0001F9FF'     # 보조 이모티콘 (Supplemental Emoticons)
    r'\U0001FA00-\U0001FA6F'     # 체스 기호
    r'\U0001FA70-\U0001FAFF'     # 기호 확장
    r'\U00002702-\U000027B0'     # 딩뱃(Dingbats)
    r']+'
)

emojis = emoji_pattern.findall(text)
print(emojis)  # ['🎉🎊', '🥳']
```

> 💬 **참고**: 이모지 매칭은 유니코드 표준이 업데이트될 때마다 새로운 이모지가 추가되므로, 완전한 이모지 매칭 패턴은 상당히 복잡합니다. 실무에서는 `emoji` 같은 전용 라이브러리를 사용하는 것이 더 실용적입니다. 또한 Chapter 17에서 배울 `regex` 모듈의 유니코드 카테고리 매칭(`\p{Emoji}`)을 사용하면 훨씬 간편합니다.

### 상세 예시

> 🔍 **예시 11.4.1: 특수 유니코드 문자 제거**
>
> **상황**: 텍스트에서 전각 문자를 반각 문자로 변환합니다.
>
> ```python
> import re
>
> text = "Ｈｅｌｌｏ，Ｗｏｒｌｄ！ 전각문자 테스트"
>
> # 전각 영문자 → 반각 변환
> def fullwidth_to_halfwidth(match):
>     char = match.group()
>     # 전각 문자(U+FF01~U+FF5E)를 반각(U+0021~U+007E)으로 변환
>     code = ord(char) - 0xFEE0
>     return chr(code)
>
> result = re.sub(r'[\uFF01-\uFF5E]', fullwidth_to_halfwidth, text)
> print(f"원본: {text}")
> print(f"변환: {result}")
> ```
>
> **출력**:
> ```
> 원본: Ｈｅｌｌｏ，Ｗｏｒｌｄ！ 전각문자 테스트
> 변환: Hello,World! 전각문자 테스트
> ```
>
> **핵심 포인트**: 전각 문자의 코드포인트에서 `0xFEE0`을 빼면 대응하는 반각 문자의 코드포인트가 됩니다. `re.sub()`의 함수 치환 기능과 유니코드 코드포인트 범위를 결합한 활용 예입니다.

> 🔍 **예시 11.4.2: CJK(중국·일본·한국) 문자 감지**
>
> **상황**: 텍스트에 CJK 문자가 포함되어 있는지 확인하고, 포함된 언어를 대략적으로 판별합니다.
>
> ```python
> import re
>
> def detect_cjk(text):
>     """텍스트에 포함된 CJK 문자 유형을 감지"""
>     results = []
>     
>     if re.search(r'[\uAC00-\uD7A3]', text):
>         results.append("한글")
>     if re.search(r'[\u4E00-\u9FFF]', text):
>         results.append("한자(CJK)")
>     if re.search(r'[\u3040-\u309F]', text):
>         results.append("히라가나")
>     if re.search(r'[\u30A0-\u30FF]', text):
>         results.append("카타카나")
>     
>     return results if results else ["CJK 문자 없음"]
>
> test_texts = [
>     "안녕하세요 Hello",
>     "日本語テスト",
>     "한자 漢字 テスト",
>     "Pure English text",
> ]
>
> for text in test_texts:
>     print(f"'{text}' → {detect_cjk(text)}")
> ```
>
> **출력**:
> ```
> '안녕하세요 Hello' → ['한글']
> '日本語テスト' → ['한자(CJK)', '카타카나']
> '한자 漢字 テスト' → ['한글', '한자(CJK)', '카타카나']
> 'Pure English text' → ['CJK 문자 없음']
> ```
>
> **핵심 포인트**: 유니코드 블록 범위를 활용하면, 텍스트에 포함된 문자 체계를 판별할 수 있습니다. 다만 이것은 대략적인 판별이며, 한자는 중국어·일본어·한국어에서 모두 사용되므로 한자만으로 정확한 언어 판별은 어렵습니다.

### 흔한 실수와 주의사항

> ⚠️ **주의**: `\u` vs `\U` 대소문자 구분
>
> - `\uXXXX` (소문자 u): 4자리 16진수 (U+0000~U+FFFF, BMP)
> - `\UXXXXXXXX` (대문자 U): 8자리 16진수 (U+0000~U+10FFFF, 전체 유니코드)
>
> 이모지 등 BMP 밖의 문자를 지정할 때 `\u`를 사용하면 에러가 발생합니다.
>
> ```python
> # 잘못된 예
> # pattern = r'[\u1F600-\u1F64F]'  # 오류! \u는 4자리만 지원
> 
> # 올바른 예
> pattern = r'[\U0001F600-\U0001F64F]'  # \U는 8자리
> ```

> ⚠️ **주의**: 정규표현식의 `\uXXXX`는 파이썬 문자열 리터럴의 `\uXXXX`와 같은 결과
>
> raw string(`r''`) 안에서도 `\uXXXX`는 **파이썬 문자열 처리 단계에서 해석**됩니다. 따라서 `r'\uAC00'`이라고 쓰면, 파이썬이 이를 '가' 문자로 해석한 뒤 정규표현식 엔진에 전달합니다.
>
> ```python
> import re
> # 다음 두 패턴은 동일합니다
> print(re.findall(r'[\uAC00-\uD7A3]+', '안녕하세요'))  # ['안녕하세요']
> print(re.findall('[가-힣]+', '안녕하세요'))              # ['안녕하세요']
> ```

### Section 요약

- `\uXXXX`(4자리)로 BMP 범위의 유니코드 문자를 코드포인트로 지정할 수 있습니다
- `\UXXXXXXXX`(8자리)로 BMP 밖의 유니코드 문자(이모지 등)를 지정할 수 있습니다
- 주요 유니코드 블록(한글, CJK 한자, 히라가나, 카타카나 등)의 범위를 알면 다양한 문자 체계를 정밀하게 매칭할 수 있습니다
- 이모지 매칭은 범위가 넓고 복잡하므로, 실무에서는 전용 라이브러리나 확장 모듈을 권장합니다

## 11.5 다국어 텍스트 처리 패턴

> 💡 **한 줄 요약**: 다국어 텍스트를 처리할 때는 유니코드 모드의 기본 동작을 이해하고, 각 언어의 특성에 맞는 패턴을 설계해야 합니다.

### 직관적 이해

현대의 텍스트 데이터는 여러 언어가 혼재하는 경우가 매우 흔합니다. 한국어 웹사이트에 영어, 한자, 이모지가 섞여 있고, 국제 서비스의 로그에는 여러 나라의 문자가 혼재합니다. 이런 다국어 환경에서 정규표현식을 사용할 때, 한 가지 언어만 고려한 패턴은 예상치 못한 결과를 낳을 수 있습니다.

이 Section에서는 다국어 텍스트를 처리할 때 알아두어야 할 패턴과 주의사항을 정리합니다.

### 핵심 개념 설명

**다국어 텍스트에서 `\w`와 `\b`의 동작**

유니코드 모드에서 `\w`는 거의 모든 언어의 글자를 매칭하므로, 여러 언어가 공백 없이 이어져 있을 때 하나의 "단어"로 인식됩니다.

```python
import re

text = "Python은좋다"

# 유니코드 모드: 한글+영문이 하나의 \w+ 토큰
print(re.findall(r'\w+', text))
# ['Python은좋다']  — 전체가 하나의 "단어"!

# 한글과 영문을 분리하려면 문자 클래스를 명시적으로 사용
print(re.findall(r'[a-zA-Z]+|[가-힣]+', text))
# ['Python', '은좋다']
```

**언어별 분리 패턴**

다국어 텍스트에서 각 언어의 토큰을 분리하는 패턴:

```python
import re

text = "Python은 最高의 language다! 🎉"

# 언어별 토큰 분리
pattern = re.compile(
    r'[a-zA-Z]+'           # 영어
    r'|[가-힣]+'           # 한글
    r'|[\u4E00-\u9FFF]+'   # 한자
    r'|[0-9]+'             # 숫자
)

tokens = pattern.findall(text)
print(tokens)
# ['Python', '은', '最高', '의', 'language', '다']
```

**전각/반각 문자 혼재 처리**

일본어나 중국어 텍스트, 또는 오래된 한국어 데이터에서는 전각 문자가 혼재하는 경우가 있습니다.

```python
import re

# 전각 숫자와 반각 숫자가 혼재
text = "가격: ３，５００원 (3,500원)"

# 전각 숫자: ０-９ (U+FF10~U+FF19)
# 반각 숫자: 0-9 (U+0030~U+0039)
all_digits = re.findall(r'[0-9\uFF10-\uFF19]+', text)
print(all_digits)  # ['３', '５００', '3', '500']

# 전각 숫자를 반각으로 정규화한 후 처리하는 것이 일반적
def normalize_fullwidth_digits(text):
    """전각 숫자를 반각으로 변환"""
    def convert(match):
        return chr(ord(match.group()) - 0xFEE0)
    return re.sub(r'[\uFF10-\uFF19]', convert, text)

normalized = normalize_fullwidth_digits(text)
print(normalized)  # "가격: 3，500원 (3,500원)"
```

**줄바꿈과 공백의 다국어 처리**

> 🔗 **연결**: Chapter 10에서 배운 `re.DOTALL` 플래그는 `.`이 줄바꿈도 매칭하도록 변경합니다. 다국어 텍스트에서는 다양한 종류의 줄바꿈과 공백이 존재할 수 있습니다.

```python
import re

# 다양한 공백 문자
spaces = {
    '일반 공백': ' ',              # U+0020
    '줄바꿈 없는 공백': '\u00A0',  # U+00A0 (NBSP)
    '전각 공백': '\u3000',         # U+3000
    '탭': '\t',                   # U+0009
}

text = "단어A\u00A0단어B\u3000단어C 단어D"

# 유니코드 \s: 모든 유니코드 공백을 매칭
print(re.split(r'\s+', text))
# ['단어A', '단어B', '단어C', '단어D']

# ASCII \s: 일반 공백, 탭, 줄바꿈만 매칭
print(re.split(r'(?a)\s+', text))
# ['단어A\xa0단어B\u3000단어C', '단어D']
# → NBSP와 전각 공백이 구분자로 인식되지 않음!
```

> 🔍 **예시 11.5.2: 한국어+영어 혼합 텍스트의 단어 분리**
>
> **상황**: IT 기술 블로그처럼 한국어와 영어가 빈번히 혼용되는 텍스트에서 의미 있는 토큰을 추출합니다.

> import re
>
> text = "Machine Learning은 AI의 핵심 분야로, deep learning과 NLP가 인기입니다."

> 🔬 **Deep Dive**: 유니코드 정규화(Normalization)와 정규표현식
>
> *이 부분은 심화 내용입니다. 처음 학습 시 건너뛰어도 됩니다.*
>
> 유니코드에서 같은 문자가 다른 코드포인트 시퀀스로 표현될 수 있습니다. 예를 들어, 한글 '가'는 완성형 U+AC00 하나의 코드포인트일 수도 있고, 조합형으로 초성 'ㄱ'(U+1100) + 중성 'ㅏ'(U+1161)의 두 코드포인트로 표현될 수도 있습니다.
>
> ```python
> import unicodedata
>
> # NFC: 완성형 (Canonical Decomposition, followed by Canonical Composition)
> nfc = unicodedata.normalize('NFC', '가')
> print(f"NFC: '{nfc}', 길이: {len(nfc)}, 코드포인트: U+{ord(nfc):04X}")
> # NFC: '가', 길이: 1, 코드포인트: U+AC00
>
> # NFD: 분해형 (Canonical Decomposition)
> nfd = unicodedata.normalize('NFD', '가')
> print(f"NFD: '{nfd}', 길이: {len(nfd)}")
> # NFD: '가', 길이: 2 (초성 ㄱ + 중성 ㅏ로 분해됨)
> print(f"코드포인트: U+{ord(nfd[0]):04X} U+{ord(nfd[1]):04X}")
> # 코드포인트: U+1100 U+1161
> ```
>
> 정규표현식은 코드포인트 수준에서 동작하므로, NFC와 NFD 형태의 같은 문자가 다르게 매칭될 수 있습니다. 따라서 **정규표현식을 적용하기 전에 `unicodedata.normalize('NFC', text)`로 정규화**하는 것이 안전합니다.
>
> ```python
> import re
> import unicodedata
>
> # 정규화 전: NFD 형태의 '가'는 [가-힣]에 매칭되지 않을 수 있음
> nfd_ga = unicodedata.normalize('NFD', '가')
> print(re.match(r'[가-힣]', nfd_ga))  # None!
>
> # 정규화 후: NFC 형태로 변환하면 정상 매칭
> nfc_ga = unicodedata.normalize('NFC', nfd_ga)
> print(re.match(r'[가-힣]', nfc_ga))  # <re.Match object>
> ```
>
> macOS에서 생성된 파일명이나, 웹에서 수집한 텍스트에서 NFD 형태의 한글이 종종 발견됩니다. 텍스트 처리 전에 NFC 정규화를 하는 것을 습관화하세요.

### 흔한 실수와 주의사항

> ⚠️ **주의**: `\w+`로 한국어 "단어"를 추출할 수 없습니다
>
> 한국어는 영어와 달리 단어 사이에 반드시 공백이 있지 않고, 조사(은/는/이/가 등)가 명사에 붙습니다. `\w+`는 "안녕하세요"를 하나의 토큰으로 추출하지만, "안녕" + "하" + "세요"의 형태소로 분리하지는 못합니다. 형태소 분석이 필요하면 `konlpy` 같은 한국어 NLP 라이브러리를 활용하세요.

> ⚠️ **주의**: 맥(macOS)에서의 한글 파일명 문제
>
> macOS의 파일 시스템은 한글 파일명을 NFD(분해형)로 저장합니다. 이 때문에 파일명에 대해 `[가-힣]` 패턴을 적용하면 매칭이 실패할 수 있습니다. 파일명 처리 시에는 반드시 `unicodedata.normalize('NFC', filename)`으로 정규화 후 패턴을 적용하세요.

> ⚠️ **주의**: 이모지는 하나의 "문자"가 아닐 수 있습니다
>
> 일부 이모지는 여러 유니코드 코드포인트가 결합된 **이모지 시퀀스**입니다. 예를 들어, 가족 이모지(👨‍👩‍👧‍👦)는 여러 개의 코드포인트와 결합 문자(ZWJ, Zero Width Joiner)로 구성됩니다. 단순한 `.` 하나가 이런 복합 이모지 전체를 매칭하지 못할 수 있습니다.
>
> ```python
> family = '👨‍👩‍👧‍👦'
> print(len(family))  # 11 (여러 코드포인트로 구성)
> 
> import re
> print(re.findall(r'.', family))  # 여러 개의 조각으로 분리됨
> ```

### Section 요약

- 유니코드 모드의 `\w`는 한영 혼합 텍스트에서 언어 경계를 구분하지 못하므로, 언어별 분리가 필요하면 명시적 문자 클래스(`[a-zA-Z]+|[가-힣]+`)를 사용합니다
- 전각/반각 문자가 혼재하는 데이터는 정규화(반각으로 통일) 후 처리하는 것이 좋습니다
- 유니코드 정규화(NFC/NFD)가 정규표현식 매칭 결과에 영향을 줄 수 있으므로, 처리 전에 NFC 정규화를 수행하세요
- 이모지는 복합 코드포인트로 구성될 수 있어 단순 패턴으로는 완전한 매칭이 어렵습니다
- 한국어 형태소 분석은 정규표현식의 범위를 넘어서므로, 전문 NLP 라이브러리를 병행하세요

>
> ```python
> text = "Ｈｅｌｌｏ　Ｗｏｒｌｄ　１２３"
> # 기대 결과: "Hello World 123"
> ```
>
> <details>
> <summary>모범 답안</summary>
>
> **접근 방법**: 전각 문자(U+FF01~U+FF5E)의 코드포인트에서 0xFEE0을 빼면 대응하는 반각 문자가 됩니다. 전각 공백(U+3000)은 반각 공백(U+0020)으로 별도 처리합니다.
>
> **풀이 과정**:
> ```python
> import re
>
> def normalize_fullwidth(text):
>     """전각 문자를 반각으로 변환"""
>     # 1. 전각 영숫자 및 기호 (U+FF01~U+FF5E) → 반각 (U+0021~U+007E)
>     def fw_to_hw(match):
>         return chr(ord(match.group()) - 0xFEE0)
>     
>     result = re.sub(r'[\uFF01-\uFF5E]', fw_to_hw, text)
>     
>     # 2. 전각 공백 (U+3000) → 반각 공백 (U+0020)
>     result = result.replace('\u3000', ' ')
>     
>     return result

> print(normalize_fullwidth(text))  # "Hello World 123"

> text2 = "전화번호：０３−１２３４−５６７８"
> print(normalize_fullwidth(text2))  # "전화번호:03-1234-5678"
> ```
>
> **정답**: `re.sub(r'[\uFF01-\uFF5E]', 변환함수, text)`와 전각 공백 치환의 조합

> **보충 설명**: 전각-반각 변환은 0xFEE0의 오프셋 관계가 있어 계산이 간단합니다. 이 패턴은 일본어, 중국어 텍스트 처리에서도 자주 사용됩니다.

>
> **접근 방법**: 세 가지 자모 패턴을 순서대로 치환합니다.
>
> **풀이 과정**:
> ```python
> import re
>
> def convert_jamo_to_emoji(text):
>     """자모 반복 표현을 이모지로 변환"""
>     # ㅋ 2개 이상 → 😂
>     text = re.sub(r'ㅋ{2,}', '😂', text)
>     # ㅎ 2개 이상 → 😊
>     text = re.sub(r'ㅎ{2,}', '😊', text)
>     # ㅠ 또는 ㅜ 2개 이상 → 😢
>     text = re.sub(r'[ㅠㅜ]{2,}', '😢', text)
>     return text
>
> text = "ㅋㅋㅋㅋ 진짜 웃겨 ㅎㅎ 근데 좀 슬프기도 ㅠㅠㅠ"
> print(convert_jamo_to_emoji(text))
> # "😂 진짜 웃겨 😊 근데 좀 슬프기도 😢"
>
> # 추가 테스트
> text2 = "ㅋㅋ 좋아 ㅜㅜ 아쉽다 ㅎㅎㅎ"
> print(convert_jamo_to_emoji(text2))
> # "😂 좋아 😢 아쉽다 😊"
> ```
>
> **정답**: `re.sub(r'ㅋ{2,}', '😂', text)` 등의 패턴을 순차 적용
>
> **보충 설명**: `[ㅠㅜ]{2,}`처럼 문자 클래스를 사용하면 'ㅠ'와 'ㅜ'가 혼재해도 매칭됩니다 (예: "ㅜㅠㅜ"). 패턴의 적용 순서에 따라 결과가 달라질 수 있으므로, 의도에 맞게 순서를 조정해야 합니다.
> </details>

## D3. Problem (심화 문제) — 깊이 있는 사고와 창의적 문제 해결

> **Problem 11.1**: 다음 요구사항을 만족하는 **다국어 텍스트 분석기** 함수를 작성하세요.
>
> 입력: 여러 언어가 혼합된 텍스트
> 출력: 딕셔너리 형태로 다음 정보를 반환
> - `'korean'`: 한글 문자 수 (음절 + 자모)
> - `'english'`: 영문 알파벳 수
> - `'digits'`: 아라비아 숫자(0-9) 수
> - `'chinese'`: 한자(CJK) 수
> - `'spaces'`: 공백 문자 수
> - `'other'`: 그 외 문자 수
> - `'total'`: 전체 문자 수
> - `'dominant'`: 가장 많은 문자 유형 (공백과 기타 제외)
>
> 테스트:
> ```python
> text = "Python은 最高의 언어! 2025년 화이팅 🎉"
> result = analyze_multilingual(text)
> ```
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 각 문자 유형에 대한 문자 클래스를 정의하고, `findall()`로 해당 유형의 문자를 모두 찾아 개수를 셉니다.
>
> **풀이 전략**:
> 1. 각 유형별 패턴을 정의합니다
> 2. `findall()`로 각 유형에 해당하는 문자를 추출합니다
> 3. 결과를 딕셔너리로 정리합니다
>
> **상세 풀이**:
> ```python
> import re
>
> def analyze_multilingual(text):
>     """다국어 텍스트 분석"""
>     counts = {
>         'korean': len(re.findall(r'[가-힣ㄱ-ㅎㅏ-ㅣ]', text)),
>         'english': len(re.findall(r'[a-zA-Z]', text)),
>         'digits': len(re.findall(r'[0-9]', text)),
>         'chinese': len(re.findall(r'[\u4E00-\u9FFF]', text)),
>         'spaces': len(re.findall(r'\s', text)),
>     }
>     
>     counts['total'] = len(text)
>     counts['other'] = counts['total'] - sum(counts[k] for k in 
>                        ['korean', 'english', 'digits', 'chinese', 'spaces'])
>     
>     # 가장 많은 문자 유형 판별 (공백과 기타 제외)
>     lang_counts = {k: v for k, v in counts.items() 
>                    if k not in ('spaces', 'other', 'total') and v > 0}
>     counts['dominant'] = max(lang_counts, key=lang_counts.get) if lang_counts else 'none'
>     
>     return counts
>
> # 테스트
> text = "Python은 最高의 언어! 2025년 화이팅 🎉"
> result = analyze_multilingual(text)
> for key, value in result.items():
>     print(f"  {key}: {value}")
> ```
>
> **출력 예시**:
> ```
>   korean: 7
>   english: 6
>   digits: 4
>   chinese: 2
>   spaces: 5
>   other: 3
>   total: 27
>   dominant: korean
> ```
>
> **확장 생각**: 이 분석기를 확장하여 일본어(히라가나+카타카나)도 분류하거나, 문장 단위 분석, 텍스트의 주요 언어 자동 판별 등으로 발전시킬 수 있습니다. 실무에서는 `chardet`이나 `langdetect` 같은 라이브러리가 더 정확한 언어 판별을 제공합니다.
> </details>

> **Problem 11.2**: 한국어 텍스트에서 **초성 검색** 기능을 구현하세요.
>
> 한글 음절의 초성을 분석하여, 사용자가 초성만 입력하면 해당 초성 패턴에 맞는 단어를 찾는 기능입니다.
>
> 예시:
> - 초성 "ㅎㄱ" → "한글", "학교", "환경" 등 매칭
> - 초성 "ㅈㄱ" → "정규", "전국", "제가" 등 매칭
>
> ```python
> words = ["한글", "학교", "환경", "정규", "전국", "제가", "파이썬", "프로그램", "안녕"]
> search_chosung("ㅎㄱ", words)  # → ["한글", "학교", "환경"]
> search_chosung("ㅍㅇ", words)  # → ["파이썬"]
> ```
>
> **힌트**: 한글 음절의 코드포인트에서 초성을 계산하는 공식: `초성_인덱스 = (ord(char) - 0xAC00) // (21 * 28)`. 각 초성에 해당하는 음절 범위를 동적으로 생성하여 정규표현식 패턴을 만드세요.
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: 초성 하나에 해당하는 한글 음절의 범위를 계산하고, 이를 정규표현식 문자 클래스로 변환하여 패턴을 동적으로 생성합니다.
>
> **풀이 전략**:
> 1. 초성 리스트에서 입력된 자모의 인덱스를 찾습니다
> 2. 해당 초성에 대응하는 음절 범위(시작~끝)를 계산합니다
> 3. 음절 범위를 `[시작-끝]` 형태의 문자 클래스로 변환합니다
> 4. 전체 초성 패턴을 연결하여 정규표현식을 만듭니다
>
> **상세 풀이**:
> ```python
> import re
>
> # 초성 목록 (유니코드 순서)
> CHOSUNG_LIST = list("ㄱㄲㄴㄷㄸㄹㅁㅂㅃㅅㅆㅇㅈㅉㅊㅋㅌㅍㅎ")
>
> def chosung_to_range(cho):
>     """초성 하나에 대응하는 한글 음절 범위를 반환"""
>     if cho not in CHOSUNG_LIST:
>         return None
>     
>     idx = CHOSUNG_LIST.index(cho)
>     # 각 초성에 해당하는 음절: 21(중성) × 28(종성) = 588개
>     start = 0xAC00 + idx * 21 * 28
>     end = start + 21 * 28 - 1
>     return chr(start), chr(end)
>
> def search_chosung(chosung_query, word_list):
>     """초성 검색: 초성 패턴에 맞는 단어를 찾아 반환"""
>     # 초성 하나하나를 음절 범위 패턴으로 변환
>     pattern_parts = []
>     for cho in chosung_query:
>         range_pair = chosung_to_range(cho)
>         if range_pair is None:
>             return []  # 유효하지 않은 초성
>         start, end = range_pair
>         pattern_parts.append(f'[{start}-{end}]')
>     
>     # 패턴 조합: 각 음절 위치에 해당 초성 범위를 매칭
>     pattern = ''.join(pattern_parts)
>     regex = re.compile(pattern)
>     
>     # 단어 목록에서 패턴의 시작 부분에 매칭되는 단어 필터링
>     return [word for word in word_list if regex.match(word)]
>
> # 테스트
> words = ["한글", "학교", "환경", "정규", "전국", "제가", "파이썬", "프로그램", "안녕"]
>
> print(search_chosung("ㅎㄱ", words))   # ['한글', '학교', '환경']
> print(search_chosung("ㅍㅇ", words))   # ['파이썬']
> print(search_chosung("ㅈㄱ", words))   # ['정규', '전국', '제가']
> print(search_chosung("ㅍㄹ", words))   # ['프로그램']
> print(search_chosung("ㅇㄴ", words))   # ['안녕']
>
> # 생성된 패턴 확인
> cho = 'ㅎ'
> start, end = chosung_to_range(cho)
> print(f"'{cho}' 초성 범위: '{start}'(U+{ord(start):04X}) ~ '{end}'(U+{ord(end):04X})")
> # 'ㅎ' 초성 범위: '하'(U+D558) ~ '힣'(U+D7A3)
> ```
>
> **확장 생각**: 이 기법은 한국어 검색 엔진, 자동완성, 연락처 검색 등에서 실제로 사용됩니다. 더 발전시키면 초성+중성 검색("하ㄴ" → "한글", "한국"), 부분 매칭("ㅎ" → 초성이 ㅎ인 모든 단어), 정렬 기능 등을 추가할 수 있습니다. 또한 이중 초성(ㄲ, ㄸ 등)의 처리도 고려해야 합니다.
> </details>

> **Problem 11.3**: 유니코드 정규화(NFC/NFD)가 정규표현식 매칭에 미치는 영향을 실험하고, **NFC 정규화 후 한글을 안전하게 처리하는 래퍼 함수**를 작성하세요.
>
> 요구사항:
> 1. NFC/NFD 형태의 한글에 대해 `[가-힣]` 패턴의 매칭 결과 차이를 확인하세요
> 2. 텍스트를 NFC로 정규화한 후 정규표현식을 적용하는 래퍼 함수 `safe_findall(pattern, text)`를 작성하세요
> 3. 래퍼 함수가 NFC/NFD 입력 모두에 대해 동일한 결과를 보장함을 검증하세요
>
> <details>
> <summary>풀이 가이드</summary>
>
> **핵심 아이디어**: NFD 형태의 한글은 `[가-힣]` 패턴에 매칭되지 않습니다. NFC 정규화를 선행하면 이 문제를 해결할 수 있습니다.
>
> **풀이 전략**:
> 1. 같은 한글 텍스트를 NFC/NFD로 각각 변환합니다
> 2. 두 형태에 대해 `[가-힣]` 패턴을 적용하여 차이를 확인합니다
> 3. NFC 정규화를 내장한 래퍼 함수를 만듭니다
>
> **상세 풀이**:
> ```python
> import re
> import unicodedata
>
> # 1. NFC/NFD 차이 확인
> text = "안녕하세요"
>
> nfc_text = unicodedata.normalize('NFC', text)
> nfd_text = unicodedata.normalize('NFD', text)
>
> print(f"NFC 길이: {len(nfc_text)}")  # 5
> print(f"NFD 길이: {len(nfd_text)}")  # 15 (각 음절이 초성+중성+(종성)으로 분해)
>
> print(f"NFC 매칭: {re.findall(r'[가-힣]+', nfc_text)}")  # ['안녕하세요']
> print(f"NFD 매칭: {re.findall(r'[가-힣]+', nfd_text)}")  # [] — 매칭 실패!
>
> # 2. 안전한 래퍼 함수
> def safe_findall(pattern, text, flags=0):
>     """NFC 정규화 후 findall을 수행하는 안전한 래퍼"""
>     normalized = unicodedata.normalize('NFC', text)
>     return re.findall(pattern, normalized, flags)
>
> def safe_search(pattern, text, flags=0):
>     """NFC 정규화 후 search를 수행하는 안전한 래퍼"""
>     normalized = unicodedata.normalize('NFC', text)
>     return re.search(pattern, normalized, flags)
>
> def safe_sub(pattern, repl, text, **kwargs):
>     """NFC 정규화 후 sub을 수행하는 안전한 래퍼"""
>     normalized = unicodedata.normalize('NFC', text)
>     return re.sub(pattern, repl, normalized, **kwargs)
>
> # 3. 검증
> test_texts = [
>     unicodedata.normalize('NFC', "한글 테스트"),
>     unicodedata.normalize('NFD', "한글 테스트"),
>     "한글 테스트",  # 파이썬 리터럴은 보통 NFC
> ]
>
> for t in test_texts:
>     form = "NFC" if t == unicodedata.normalize('NFC', t) else "NFD"
>     result = safe_findall(r'[가-힣]+', t)
>     print(f"{form} 입력 → safe_findall 결과: {result}")
> ```
>
> **출력**:
> ```
> NFC 길이: 5
> NFD 길이: 15
> NFC 매칭: ['안녕하세요']
> NFD 매칭: []
> NFC 입력 → safe_findall 결과: ['한글', '테스트']
> NFD 입력 → safe_findall 결과: ['한글', '테스트']
> NFC 입력 → safe_findall 결과: ['한글', '테스트']
> ```
>
> **확장 생각**: 실무에서는 데이터 처리 파이프라인의 초기 단계에서 텍스트를 NFC로 정규화하는 것이 일반적입니다. 이렇게 하면 이후 모든 정규표현식 처리에서 일관된 결과를 보장합니다. 또한 클래스로 감싸서 `SafeRegex` 같은 유틸리티를 만들면 프로젝트 전체에서 재사용할 수 있습니다.
